# DNS Management

## Provider and Configuration

DNS is managed through **Route 53** with all records defined in Terraform. No DNS records are created manually through the AWS console.

## Record Types

| Record | Type | TTL | Purpose |
|--------|------|-----|---------|
| `api.example.com` | A (alias) | - | Points to CloudFront distribution |
| `app.example.com` | A (alias) | - | Points to CloudFront distribution |
| `internal.example.com` | A (alias) | - | Points to internal ALB |
| `_acme-challenge.example.com` | CNAME | 300 | Let's Encrypt DNS validation |

## Health Checks and Failover

Route 53 health checks monitor the primary region endpoint every 30 seconds. If the primary region fails three consecutive checks, DNS failover routes traffic to the standby region within 60 seconds.

```hcl
resource "aws_route53_health_check" "primary" {
  fqdn              = "api-primary.example.com"
  port              = 443
  type              = "HTTPS"
  resource_path     = "/healthz"
  failure_threshold = 3
  request_interval  = 30
}
```

## DNS TTLs

Production DNS records use low TTLs (60 seconds) to enable fast failover. This is a deliberate tradeoff — higher TTLs would reduce DNS query volume but slow down incident response. The additional DNS query cost is negligible compared to the risk of extended outages.

See [[arch-patterns]] for the multi-region architecture that DNS failover supports.
