# Grafana Dashboard Standards

## Organization

Dashboards are organized into folders by domain:

- **Infrastructure** — cluster health, node utilization, network throughput
- **Application** — per-service latency, error rates, throughput
- **Business** — user-facing KPIs, conversion funnels, feature adoption
- **On-Call** — the four golden signals per service, alerting status

Every dashboard is **provisioned from Git** via Grafana's provisioning API. No dashboard is created manually in the UI — if it's not in the `grafana-dashboards` repo, it gets deleted on the next sync.

## Dashboard Design Rules

1. Every dashboard must have a **time range selector** and **environment variable** at the top
2. Use **consistent color coding**: green for healthy, yellow for warning, red for critical
3. Include a **documentation panel** at the top explaining what the dashboard shows and when to use it
4. Latency panels should show p50, p95, and p99 — not averages
5. All panels must have **meaningful units** (requests/sec, not just numbers)

## The Four Golden Signals Dashboard

Each service gets an auto-generated dashboard showing the four golden signals from the Google SRE book:

- **Latency** — request duration histogram (p50/p95/p99)
- **Traffic** — requests per second
- **Errors** — 5xx rate as percentage of total
- **Saturation** — CPU and memory utilization vs limits

This dashboard is the first thing an on-call engineer checks during an incident.
