# Backup and Disaster Recovery

## Backup Strategy

### Database Backups

PostgreSQL backups use a three-tier approach:

- **Continuous WAL archiving** to S3 via pgBackRest — enables point-in-time recovery to any second within the retention window
- **Daily full backups** at 02:00 UTC — complete database snapshot
- **Weekly offsite copy** to a different AWS region (us-west-2) — protects against regional outages

Retention: 30 days of WAL archives, 90 days of daily backups, 1 year of weekly offsite copies.

### Object Storage

S3 buckets use **versioning** with lifecycle rules. Deleted objects are retained for 30 days before permanent removal. Cross-region replication is enabled for the primary data bucket.

### Kubernetes Resources

All cluster configuration lives in Git (GitOps). The cluster itself is ephemeral — if destroyed, it can be recreated from Terraform + ArgoCD in under 30 minutes.

## Recovery Time Objectives

| Component | RTO | RPO |
|-----------|-----|-----|
| API Server | 5 minutes | 0 (stateless) |
| Database | 30 minutes | < 1 minute (WAL) |
| Search Index | 1 hour | Rebuildable from DB |
| Full Environment | 2 hours | < 1 minute |

## DR Drills

We run a **disaster recovery drill quarterly**. The drill restores a production-equivalent environment from backups in the secondary region and runs the full integration test suite against it. Results are documented and gaps become action items.
