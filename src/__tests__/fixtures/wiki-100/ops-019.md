# Container Registry

## Registry Choice

We use **Amazon ECR** as our private container registry. Each service has its own repository within the registry, named to match the service identifier.

## Image Tagging Strategy

Images are tagged with multiple identifiers:

- **Git SHA** (`abc1234`) — immutable, maps 1:1 to a commit
- **Branch name** (`main`, `feature-xyz`) — mutable, latest build from that branch
- **SemVer** (`v1.2.3`) — for release-tagged images
- **`latest`** — always points to the most recent `main` build

Production deployments always reference the Git SHA tag, never `latest` or a branch tag. This ensures reproducibility and makes rollbacks trivial.

## Lifecycle Policies

ECR lifecycle policies automatically clean up old images:

```json
{
  "rules": [
    {
      "rulePriority": 1,
      "description": "Keep last 50 tagged images",
      "selection": {
        "tagStatus": "tagged",
        "countType": "imageCountMoreThan",
        "countNumber": 50
      },
      "action": { "type": "expire" }
    },
    {
      "rulePriority": 2,
      "description": "Remove untagged images after 7 days",
      "selection": {
        "tagStatus": "untagged",
        "countType": "sinceImagePushed",
        "countUnit": "days",
        "countNumber": 7
      },
      "action": { "type": "expire" }
    }
  ]
}
```

## Image Scanning

ECR enhanced scanning runs on every push. Images with critical CVEs are blocked from deployment by an OPA Gatekeeper policy in the cluster.
