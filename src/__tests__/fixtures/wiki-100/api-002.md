# Rate Limiting Patterns

Rate limiting protects backend services from abuse, ensures fair usage across tenants, and prevents cascading failures during traffic spikes. A well-designed rate limiter is invisible to well-behaved clients and informative to those hitting limits.

## Token Bucket Algorithm

The **token bucket** is the most widely deployed algorithm. A bucket holds up to N tokens, refills at a fixed rate R per second, and each request consumes one token. When the bucket is empty, requests are rejected with `429 Too Many Requests`.

```typescript
interface TokenBucket {
  capacity: number;
  tokens: number;
  refillRate: number; // tokens per second
  lastRefill: number; // timestamp ms
}

function tryConsume(bucket: TokenBucket): boolean {
  const now = Date.now();
  const elapsed = (now - bucket.lastRefill) / 1000;
  bucket.tokens = Math.min(bucket.capacity, bucket.tokens + elapsed * bucket.refillRate);
  bucket.lastRefill = now;

  if (bucket.tokens >= 1) {
    bucket.tokens -= 1;
    return true;
  }
  return false;
}
```

## Sliding Window Log

The **sliding window log** tracks each request timestamp in a sorted set. To check the limit, count entries within the last N seconds. This is precise but memory-intensive for high-throughput endpoints.

Redis `ZRANGEBYSCORE` makes this straightforward:

```
ZADD rate:{user_id} {timestamp} {request_id}
ZREMRANGEBYSCORE rate:{user_id} 0 {window_start}
ZCARD rate:{user_id}
```

## Response Headers

Always return rate limit metadata in response headers:

- `X-RateLimit-Limit` — maximum requests per window
- `X-RateLimit-Remaining` — requests left in current window
- `X-RateLimit-Reset` — UTC epoch seconds when the window resets
- `Retry-After` — seconds to wait (only on 429 responses)

## Multi-Tier Limits

Production APIs typically enforce multiple tiers simultaneously:

- **Per-IP**: 100 req/min (anonymous abuse prevention)
- **Per-user**: 1000 req/min (authenticated fair usage)
- **Per-endpoint**: varies (expensive operations get tighter limits)
- **Global**: circuit breaker at 80% backend capacity
