# Helm Chart Conventions

Helm is our **package manager for Kubernetes** manifests. Every service ships with a Helm chart in its repository under `charts/<service-name>/`.

## Chart Structure

```
charts/api-server/
  Chart.yaml
  values.yaml
  values-staging.yaml
  values-production.yaml
  templates/
    deployment.yaml
    service.yaml
    hpa.yaml
    ingress.yaml
    configmap.yaml
    _helpers.tpl
```

## Values Hierarchy

Values are merged in order of precedence (last wins):

- `values.yaml` — defaults applicable to all environments
- `values-<env>.yaml` — environment-specific overrides
- CLI `--set` flags — used only for CI-injected secrets references

## Deployment Template Standards

Every deployment template must include:

- **Resource requests and limits** — no pod runs without defined CPU/memory bounds
- **Liveness and readiness probes** — health check endpoints at `/healthz` and `/readyz`
- **Pod anti-affinity** — spread replicas across nodes for HA
- **Security context** — `runAsNonRoot: true`, `readOnlyRootFilesystem: true`

## Versioning

Chart versions follow **SemVer**. The `appVersion` in `Chart.yaml` matches the Docker image tag. We use `helm diff` in CI to preview changes before applying them via ArgoCD.
