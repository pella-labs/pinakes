# Encryption at Rest

All data at rest is encrypted. This page documents the encryption strategy for each storage layer.

## Database Encryption

PostgreSQL uses **Transparent Data Encryption** (TDE) via the cloud provider's managed encryption. The encryption key is an AWS KMS CMK with automatic annual rotation.

For sensitive columns (PII, payment data), we apply **application-layer encryption** using the Vault Transit engine before writing to the database. This provides defense in depth — even a database backup leak does not expose plaintext PII.

```typescript
async function encryptField(plaintext: string): Promise<string> {
  const response = await vault.write('transit/encrypt/pii-key', {
    plaintext: Buffer.from(plaintext).toString('base64'),
  });
  return response.data.ciphertext; // vault:v1:base64...
}

async function decryptField(ciphertext: string): Promise<string> {
  const response = await vault.write('transit/decrypt/pii-key', {
    ciphertext,
  });
  return Buffer.from(response.data.plaintext, 'base64').toString();
}
```

## Object Storage

S3 buckets use **SSE-KMS** with a dedicated CMK per environment. Bucket policies enforce encryption — unencrypted PutObject requests are rejected.

## File System Encryption

EBS volumes use AES-256 encryption with AWS-managed keys. This is the baseline; it does not protect against privileged access within the instance.

## Key Management

All encryption keys are managed through AWS KMS or Vault Transit. No encryption key is ever stored in application code or config. See [[sec-009]] for the full secrets management architecture and sec-019 for key rotation.
