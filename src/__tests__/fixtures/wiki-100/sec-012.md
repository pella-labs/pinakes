# OpenID Connect Integration

OpenID Connect (OIDC) is our preferred SSO protocol for new integrations. It builds on OAuth2 (see [[sec-001]]) and adds an identity layer via the **ID Token**.

## Why OIDC Over SAML

- Simpler protocol — JSON instead of XML
- Better mobile support
- Native OAuth2 compatibility — reuse existing token infrastructure
- Smaller message sizes
- Active ecosystem with modern libraries

We still support SAML for legacy enterprise IdPs (see sec-011) but recommend OIDC for all new integrations.

## Discovery

We publish an OIDC discovery document at:

```
https://auth.example.com/.well-known/openid-configuration
```

This includes endpoints for authorization, token, userinfo, JWKS, and supported scopes/claims.

## Supported Scopes

- `openid` — required, returns `sub` claim
- `profile` — name, picture, updated_at
- `email` — email, email_verified
- `groups` — custom scope returning organization memberships

## ID Token Validation

The ID Token is a JWT signed with our RS256 key. Clients MUST:

1. Verify the signature using our JWKS endpoint
2. Check `iss` matches `https://auth.example.com`
3. Check `aud` contains the client ID
4. Check `exp` is in the future
5. Check `nonce` matches the one sent in the authorization request (if applicable)

See [[sec-002]] for general JWT validation rules.
