# Prefetching and Preloading Strategies

## Resource Hints

HTML provides several resource hints to inform the browser about upcoming resource needs:

```html
<!-- DNS lookup ahead of time -->
<link rel="dns-prefetch" href="https://api.example.com" />

<!-- Full connection setup (DNS + TCP + TLS) -->
<link rel="preconnect" href="https://api.example.com" />

<!-- Download a critical resource early -->
<link rel="preload" href="/fonts/inter.woff2" as="font" type="font/woff2" crossorigin />

<!-- Likely next navigation — prefetch the page -->
<link rel="prefetch" href="/dashboard" />

<!-- High-confidence next page — prerender it entirely -->
<link rel="prerender" href="/dashboard" />
```

## Speculative Loading

The **Speculation Rules API** provides programmatic control over prefetching and prerendering:

```html
<script type="speculationrules">
{
  "prerender": [{
    "where": { "href_matches": "/product/*" },
    "eagerness": "moderate"
  }],
  "prefetch": [{
    "where": { "href_matches": "/blog/*" },
    "eagerness": "conservative"
  }]
}
</script>
```

## Framework Integration

Next.js `<Link>` prefetches linked routes automatically on viewport intersection. Disable with `prefetch={false}` for rarely-visited pages.

## Best Practices

- Preconnect to critical third-party origins (CDN, API, fonts)
- Preload resources needed in the first 3 seconds of page load
- Prefetch likely next navigations based on user behavior data
- Don't over-prefetch on metered connections
