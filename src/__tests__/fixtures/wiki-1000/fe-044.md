# Component Libraries Comparison

## Headless vs Styled

**Headless libraries** provide behavior and accessibility without styling:
- Radix UI
- Headless UI
- React Aria (Adobe)
- Ark UI

**Styled libraries** include both behavior and visual design:
- Material UI (MUI)
- Ant Design
- Chakra UI
- Mantine

## Choosing the Right Approach

Use **headless** when:
- You have a custom design system
- You need full control over styling
- Bundle size matters

Use **styled** when:
- Speed of development is the priority
- The built-in design is acceptable
- Your team lacks dedicated designers

## shadcn/ui

**shadcn/ui** takes a unique approach: instead of installing a dependency, you copy component source code into your project. Components are built on Radix UI and styled with Tailwind CSS. This means:

- Full ownership of the code
- No version update breakage
- Easy to customize
- No additional dependencies

## Evaluation Criteria

When choosing a component library, evaluate:
- Accessibility compliance (WCAG 2.1 AA)
- Bundle size impact
- TypeScript support quality
- Documentation and community
- Customization escape hatches
