# Backup Strategies

## The 3-2-1 Rule

Follow the **3-2-1 backup rule**: keep 3 copies of data, on 2 different media types, with 1 copy offsite. In cloud terms, this means production data, a same-region backup, and a cross-region backup.

## Database Backups

### Automated Snapshots

RDS and Aurora take **automated snapshots** daily with configurable retention (we use 30 days). These are full point-in-time recovery capable.

### Logical Backups

Run `pg_dump` nightly for a portable backup that isn't tied to the RDS engine version. Store in S3 with lifecycle rules moving to Glacier after 30 days.

```bash
#!/bin/bash
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
pg_dump -h $DB_HOST -U $DB_USER -d $DB_NAME \
  | gzip \
  | aws s3 cp - "s3://backups/db/${TIMESTAMP}.sql.gz"
```

## Object Storage

S3 objects use **versioning** plus **cross-region replication**. Deleted objects are recoverable for 30 days via versioning. Critical buckets have **Object Lock** enabled to prevent accidental or malicious deletion.

## Backup Verification

Backups that aren't tested are hypothetical. Monthly, restore a database backup to a test environment and verify data integrity. Automate this with a CI job.
