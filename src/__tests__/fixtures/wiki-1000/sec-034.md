# Secure Cookie Configuration

Cookies carry authentication state and other sensitive data. Misconfigured cookies are a common vector for session hijacking and CSRF attacks.

## Essential Attributes

Every authentication cookie must include:

```
Set-Cookie: session=abc123; Secure; HttpOnly; SameSite=Lax; Path=/; Max-Age=3600
```

### Secure
Only send the cookie over HTTPS. Without this, cookies are transmitted in plaintext on HTTP connections.

### HttpOnly
Prevent JavaScript from reading the cookie. This mitigates XSS-based session theft (see [[sec-009]]).

### SameSite
Controls cross-site cookie behavior:

- **Strict**: cookie is never sent cross-site. Most secure but can break legitimate flows (e.g., following a link from email).
- **Lax**: cookie is sent on top-level navigations (GET) but not on cross-site POST. Good default.
- **None**: cookie is always sent cross-site. Requires `Secure` flag. Only use when cross-origin access is genuinely needed.

### Path
Restrict the cookie to specific URL paths. Use `/` for session cookies, more specific paths for feature-specific cookies.

### Max-Age / Expires
Set appropriate lifetimes. Authentication sessions should expire. Avoid indefinite cookies.

## Cookie Prefixes

Modern browsers support **cookie prefixes** for additional security:

- `__Secure-` prefix: requires `Secure` attribute
- `__Host-` prefix: requires `Secure`, must have `Path=/`, must not have `Domain`
