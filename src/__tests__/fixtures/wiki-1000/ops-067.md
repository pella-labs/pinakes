# AWS EKS Cluster Setup

## Cluster Architecture

Our **EKS clusters** run with managed node groups across three availability zones for high availability.

## Terraform Configuration

```hcl
module "eks" {
  source  = "terraform-aws-modules/eks/aws"
  version = "~> 20.0"

  cluster_name    = "production"
  cluster_version = "1.29"

  vpc_id     = module.vpc.vpc_id
  subnet_ids = module.vpc.private_subnet_ids

  cluster_endpoint_public_access = true
  cluster_endpoint_private_access = true

  eks_managed_node_groups = {
    general = {
      instance_types = ["m6i.xlarge"]
      min_size       = 3
      max_size       = 20
      desired_size   = 5

      labels = {
        workload = "general"
      }
    }

    spot = {
      instance_types = ["m5.xlarge", "m5a.xlarge", "m6i.xlarge"]
      capacity_type  = "SPOT"
      min_size       = 0
      max_size       = 30
      desired_size   = 3

      labels = {
        workload = "batch"
      }
      taints = [{
        key    = "spot"
        value  = "true"
        effect = "NO_SCHEDULE"
      }]
    }
  }
}
```

## Add-Ons

Essential EKS add-ons installed via Terraform:

- **VPC CNI** — pod networking
- **CoreDNS** — DNS resolution
- **kube-proxy** — service networking
- **EBS CSI Driver** — persistent volumes

See [[ops-004]] for K8s fundamentals and [[ops-024]] for spot instance strategy.
