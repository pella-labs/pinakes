# API Testing Strategy

## Test Pyramid for APIs

```
         /  E2E  \        ← 5-10 critical paths
        /  Contract \      ← every service boundary
       / Integration  \    ← every endpoint
      /    Unit Tests   \  ← business logic
```

## Contract Testing

Use **Pact** or **schema validation** to ensure services agree on the API shape:

1. Consumer writes a contract (expected request/response)
2. Provider verifies it can satisfy the contract
3. Contracts live in a broker, versioned per service

## Integration Testing

Test each endpoint with a real database (not mocked):

```typescript
describe('POST /orders', () => {
  it('creates an order', async () => {
    const res = await request(app)
      .post('/orders')
      .set('Authorization', `Bearer ${testToken}`)
      .send({ customerId: 'cust_1', items: [{ productId: 'p_1', quantity: 2 }] });

    expect(res.status).toBe(201);
    expect(res.body.id).toMatch(/^ord_/);
    expect(res.body.items).toHaveLength(1);
  });

  it('returns 400 for missing customerId', async () => {
    const res = await request(app)
      .post('/orders')
      .set('Authorization', `Bearer ${testToken}`)
      .send({ items: [] });

    expect(res.status).toBe(400);
    expect(res.body.error.code).toBe('VALIDATION_ERROR');
  });
});
```

## Load Testing

Use **k6** or **autocannon** to establish baseline performance. Run on every release to catch regressions. Test with realistic data volumes.
