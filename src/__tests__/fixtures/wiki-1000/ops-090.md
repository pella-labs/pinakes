# Infrastructure Cost Allocation

## Tagging Strategy

Consistent **resource tagging** is the foundation of cost allocation. Without tags, you can't attribute costs to teams, services, or environments.

## Required Tags

Every resource must have:

| Tag | Purpose | Example |
|-----|---------|---------|
| `team` | Owning team | platform |
| `service` | Service name | api-server |
| `environment` | Deployment env | production |
| `cost-center` | Finance code | CC-1234 |
| `managed-by` | How it's managed | terraform |

## Enforcement

Use **AWS Organizations SCPs** to prevent resource creation without required tags:

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Deny",
    "Action": "ec2:RunInstances",
    "Resource": "arn:aws:ec2:*:*:instance/*",
    "Condition": {
      "Null": {
        "aws:RequestTag/team": "true"
      }
    }
  }]
}
```

## Cost Allocation Reports

AWS **Cost and Usage Reports** break down spending by tag. Export to S3 and analyze with Athena for custom reports. Share weekly cost summaries with team leads.

## Showback vs Chargeback

**Showback** reports costs to teams for awareness. **Chargeback** actually bills the team's budget. Start with showback; it drives cost-conscious behavior without the overhead of internal billing.
