# React Context vs Props vs State Management

## Decision Framework

The right approach depends on how widely state is shared and how frequently it changes.

## Props (Direct Passing)

Best for state used by 1-3 components in a direct parent-child chain.

**Advantages**: explicit data flow, easy to trace, no overhead
**Drawback**: prop drilling through many intermediate components

## Context

Best for state that many components across the tree need but that changes infrequently.

**Good candidates**: theme, locale, auth status, feature flags
**Bad candidates**: form input values, search results, frequently updating data

The re-render cost of context is proportional to the number of consumers. Split contexts to minimize blast radius.

## External State Management

Best for complex, frequently-changing state shared across many components.

**Zustand**: when you need simplicity and minimal boilerplate
**Redux Toolkit**: when you need middleware, time-travel debugging, and a large ecosystem
**Jotai**: when you have many independent atoms of state

## The Gradient

```
Local state → Props → Context → External store
Simple ←————————————————————————→ Complex
Few consumers ←———————————→ Many consumers
Rare updates ←————————————→ Frequent updates
```

Start with the simplest option. Move to the next level only when you feel concrete pain.
