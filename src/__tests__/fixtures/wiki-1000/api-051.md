# Bulk Operations API

## Design Options

### Option A: Batch Endpoint

Single endpoint, multiple operations in one request. See [[api-017]] for details.

### Option B: Streaming Upload

Client streams NDJSON, server processes line by line:

```
POST /bulk/import
Content-Type: application/x-ndjson

{"action":"create","data":{"name":"Alice"}}
{"action":"create","data":{"name":"Bob"}}
{"action":"update","id":"u_001","data":{"name":"Carol"}}
```

### Option C: Async Job

For large operations (>1000 items):

1. `POST /bulk/jobs` — submit the job, get a job ID
2. `GET /bulk/jobs/{id}` — poll for status
3. `GET /bulk/jobs/{id}/results` — download results when complete

## Progress Reporting

For async jobs, include progress in the status response:

```json
{
  "id": "job_abc",
  "status": "processing",
  "progress": { "total": 5000, "processed": 2340, "errors": 12 },
  "started_at": "2025-11-03T10:00:00Z"
}
```

## Error Handling

Individual item errors should not fail the batch. Return per-item results with individual status codes. Provide a summary at the top: total, succeeded, failed.
