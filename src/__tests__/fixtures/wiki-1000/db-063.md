# Bulk Data Loading

## The Problem

Inserting rows one at a time is slow. Network round-trips, transaction overhead, and index maintenance add up. For millions of rows, you need bulk loading strategies.

## PostgreSQL COPY

The fastest way to load data into PostgreSQL:

```sql
COPY users (id, name, email, created_at)
FROM '/tmp/users.csv'
WITH (FORMAT csv, HEADER true);
```

From a Node.js application:

```typescript
import { pipeline } from 'stream/promises';
import { from as copyFrom } from 'pg-copy-streams';

const stream = client.query(copyFrom('COPY users FROM STDIN CSV HEADER'));
await pipeline(fs.createReadStream('users.csv'), stream);
```

## SQLite Bulk Insert

Wrap in a single transaction. Without this, each INSERT is its own transaction (includes fsync):

```typescript
const insert = db.prepare('INSERT INTO users VALUES (?, ?, ?)');
const insertMany = db.transaction((rows) => {
  for (const row of rows) insert.run(...row);
});
insertMany(allRows);  // thousands of rows, one transaction
```

## Optimization Tricks

- Disable indexes during load, rebuild after
- Increase `maintenance_work_mem` (PostgreSQL)
- Use `UNLOGGED` tables for staging data
- Disable triggers during bulk operations
- Use parallel loading (split file, load from multiple connections)
