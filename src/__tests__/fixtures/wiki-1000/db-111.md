# SQLite Virtual Tables

## What Are Virtual Tables?

**Virtual tables** in SQLite look like regular tables but are backed by custom code. They enable querying external data sources, implementing specialized search, or creating computed tables.

## FTS5

The most common virtual table: full-text search (see [[db-035]]).

```sql
CREATE VIRTUAL TABLE docs_fts USING fts5(title, body);
```

## sqlite-vec

Vector similarity search:

```sql
CREATE VIRTUAL TABLE embeddings USING vec0(
  embedding float[384]
);
```

## CSV Virtual Table

Query CSV files directly:

```sql
CREATE VIRTUAL TABLE temp.csv_data USING csv(filename='/tmp/data.csv', header=yes);
SELECT * FROM csv_data WHERE column1 = 'value';
```

## Custom Virtual Tables

You can implement custom virtual tables in C. The API requires implementing:
- `xCreate` / `xConnect` — table creation
- `xBestIndex` — query planning
- `xFilter` — row iteration
- `xColumn` — column access

## Use Cases

Virtual tables are used internally by SQLite for EXPLAIN, pragma functions, and the dbstat table. Third-party extensions use them for R-tree indexes, JSON functions, and more.

For SQLite extension loading, see [[db-090]].
