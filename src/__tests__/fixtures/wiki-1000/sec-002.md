# OAuth2 Client Credentials Grant

The **client credentials grant** is designed for machine-to-machine communication where no user context is needed. The client authenticates directly with the authorization server using its own credentials.

## When to Use

- Service-to-service API calls
- Background jobs and cron tasks
- Internal microservice communication
- CLI tools authenticating as a service account

## Security Considerations

The client secret must be treated with the same care as a password. Rotate credentials on a regular schedule. Consider using **mutual TLS** (see [[sec-010]]) instead of shared secrets for higher-assurance environments.

```bash
curl -X POST https://auth.example.com/token \
  -d grant_type=client_credentials \
  -d client_id=my-service \
  -d client_secret=$CLIENT_SECRET \
  -d scope="read:events write:events"
```

## Token Lifetime

Client credentials tokens should have shorter lifetimes than user tokens — typically 15-30 minutes. The overhead of re-requesting is minimal for automated clients and limits the blast radius of a leaked token.
