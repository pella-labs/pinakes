# WAF Configuration

## Web Application Firewall

A **WAF** (Web Application Firewall) inspects HTTP traffic and blocks malicious requests before they reach your application.

## AWS WAF Rules

We use AWS WAF attached to our CloudFront distributions and ALBs:

- **AWS Managed Rules**: core rule set, SQL injection, XSS, known bad inputs
- **Rate limiting**: block IPs exceeding 2000 requests per 5 minutes
- **Geo blocking**: restrict access by country if required
- **Custom rules**: block specific patterns relevant to our application

## Rule Priority

WAF rules are evaluated in priority order. Structure them as:

1. Allow trusted IPs (office, VPN, monitoring)
2. Block known bad actors (IP reputation lists)
3. Rate limiting rules
4. AWS Managed Rules (SQL injection, XSS)
5. Custom application rules
6. Default allow

## Logging and Monitoring

Log all WAF decisions to S3 for analysis. Monitor blocked request rates in Grafana. Spike in blocks might indicate an attack — or a false positive affecting legitimate users.

## False Positives

WAF rules can block legitimate traffic. When adding new rules, start in **count mode** (log but don't block) to identify false positives before switching to block mode.
