# API Gateway Authentication Patterns

## Token Relay

Gateway validates the JWT and passes it to backends unchanged. Each backend can inspect claims.

## Token Exchange

Gateway validates the external token and issues an **internal token** with additional context (resolved permissions, tenant info). Backends trust the gateway-issued token.

## mTLS Between Services

After the gateway handles external auth, internal service-to-service calls use mutual TLS. No tokens needed — identity is the certificate.

## Auth Caching

JWT verification involves RSA/ECDSA crypto. Cache the verified result (keyed by token hash) for the token's remaining TTL. Reduces CPU by 80% on high-traffic APIs.

```typescript
const authCache = new LRU<string, AuthContext>({ max: 10000, ttl: 300_000 });

async function cachedVerify(token: string): Promise<AuthContext> {
  const hash = crypto.createHash('sha256').update(token).digest('hex');
  const cached = authCache.get(hash);
  if (cached) return cached;
  
  const ctx = await verifyJwt(token);
  authCache.set(hash, ctx);
  return ctx;
}
```
