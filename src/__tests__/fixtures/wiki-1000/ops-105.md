# Kubernetes etcd Operations

## What is etcd

**etcd** is the key-value store that backs the Kubernetes API server. It stores all cluster state: pods, services, secrets, config maps, and custom resources.

## etcd Health

On managed Kubernetes (EKS, GKE), etcd is managed by the cloud provider. On self-managed clusters, monitor these metrics:

- **Leader elections**: frequent elections indicate network instability
- **Disk latency**: etcd requires low-latency SSDs (< 10ms p99)
- **Database size**: grows with cluster size, compact regularly
- **Snapshot interval**: every 5 minutes for backup

## Backup and Restore

```bash
# Create snapshot
etcdctl snapshot save /backup/etcd-snapshot.db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/etcd/ca.crt \
  --cert=/etc/etcd/server.crt \
  --key=/etc/etcd/server.key

# Restore snapshot
etcdctl snapshot restore /backup/etcd-snapshot.db \
  --data-dir=/var/lib/etcd-restore
```

## Performance Tuning

- Use dedicated SSDs for etcd data directory
- Separate etcd network from pod network
- Limit the number of objects per namespace
- Enable compaction to reclaim disk space

Most teams on managed Kubernetes never touch etcd directly, but understanding it helps debug cluster-level issues.
