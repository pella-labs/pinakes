# TypeScript Strict Mode for React

## Recommended tsconfig

```json
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true,
    "forceConsistentCasingInFileNames": true,
    "jsx": "react-jsx",
    "module": "esnext",
    "moduleResolution": "bundler",
    "target": "es2022",
    "lib": ["dom", "dom.iterable", "es2022"]
  }
}
```

## Key Strict Flags

**noUncheckedIndexedAccess** forces handling of potentially undefined array/object access:

```typescript
const items: string[] = ['a', 'b'];
const first = items[0]; // string | undefined (must check before using)
```

**exactOptionalPropertyTypes** distinguishes between missing and explicitly undefined properties:

```typescript
interface Config {
  theme?: 'light' | 'dark';
}

// Error: cannot assign undefined when the type is 'light' | 'dark'
const config: Config = { theme: undefined };
```

## React-Specific Types

Use `React.ComponentPropsWithRef` for wrapping native elements:

```tsx
type InputProps = React.ComponentPropsWithRef<'input'> & {
  label: string;
};
```

Use `React.ReactNode` for children, `React.ElementType` for polymorphic components.

See [[fe-035]] for TypeScript patterns in React.
