# gRPC Interceptors

The gRPC equivalent of middleware.

## Unary Interceptor

```typescript
function loggingInterceptor(
  call: ServerUnaryCall<any, any>,
  callback: sendUnaryData<any>,
  next: () => void
) {
  const start = Date.now();
  const method = call.getPath();

  next();

  call.on('finish', () => {
    logger.info({
      method,
      duration_ms: Date.now() - start,
      status: call.cancelled ? 'cancelled' : 'ok',
    });
  });
}
```

## Common Interceptors

- **Auth** — extract metadata, verify token, attach to context
- **Logging** — method, duration, status
- **Metrics** — counters, histograms
- **Error mapping** — convert exceptions to gRPC status codes
- **Retry** — client-side interceptor for retryable errors
- **Deadline propagation** — pass remaining budget to downstream calls

## Client vs Server Interceptors

Server interceptors process incoming requests. Client interceptors process outgoing requests. Both form a chain, executed in registration order.

For error handling specifics, see [[api-036]].
