# HIPAA Security Requirements

**HIPAA** (Health Insurance Portability and Accountability Act) mandates safeguards for **Protected Health Information** (PHI). Any system that stores, processes, or transmits PHI must comply with the Security Rule.

## Safeguard Categories

### Administrative Safeguards
- Risk analysis and management
- Workforce training
- Access management policies
- Contingency planning
- Business Associate Agreements (BAAs)

### Physical Safeguards
- Facility access controls
- Workstation use and security
- Device and media controls

### Technical Safeguards
- Access controls (unique user IDs, emergency access, automatic logoff)
- Audit controls (see [[sec-014]])
- Integrity controls (data validation)
- Transmission security (encryption in transit — see [[sec-017]])

## Encryption Requirements

HIPAA doesn't mandate specific algorithms but references NIST standards:

- **At rest**: AES-256 (see [[sec-025]])
- **In transit**: TLS 1.2+ (see [[sec-017]])
- **Key management**: documented procedures for key lifecycle

## Cloud Considerations

When using cloud services for PHI:

- Ensure the provider signs a **Business Associate Agreement**
- Use dedicated instances or verified multi-tenant isolation
- Enable cloud-native encryption and audit logging
- Verify data residency requirements
