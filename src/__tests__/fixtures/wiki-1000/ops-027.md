# Log Aggregation

## Architecture

All application logs flow through a centralized pipeline:

1. Applications write structured JSON to stdout
2. **Fluent Bit** collects logs from each node
3. Logs are forwarded to **Loki** (or CloudWatch Logs)
4. **Grafana** provides the query and dashboard layer

## Structured Logging

Every log line must be structured JSON with standard fields:

```json
{
  "level": "info",
  "ts": "2025-12-01T10:30:00Z",
  "msg": "request completed",
  "service": "api-server",
  "trace_id": "abc123",
  "method": "GET",
  "path": "/api/users",
  "status": 200,
  "duration_ms": 42
}
```

## Log Levels

- **error** — something failed and needs attention
- **warn** — something unexpected but handled
- **info** — significant business events
- **debug** — development troubleshooting (disabled in prod)

## Retention

- Hot storage (queryable): 14 days
- Warm storage (slower queries): 90 days
- Cold storage (archive): 1 year
- Compliance archive: 7 years (encrypted in S3 Glacier)

Never log secrets, PII, or authentication tokens. Use redaction filters in the log pipeline.
