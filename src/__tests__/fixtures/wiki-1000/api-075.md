# Content-Type Negotiation for APIs

## Request Content Types

Validate `Content-Type` on every mutating request:

```typescript
function requireJson(req: Request, res: Response, next: NextFunction) {
  if (!req.is('application/json')) {
    return res.status(415).json({
      error: {
        code: 'UNSUPPORTED_MEDIA_TYPE',
        message: `Expected application/json, got ${req.headers['content-type']}`,
      }
    });
  }
  next();
}
```

## Response Content Types

Default to `application/json`. Support alternatives via `Accept` header:

- `application/json` — standard
- `text/csv` — for data export
- `application/xml` — legacy integrations (avoid adding new)
- `application/octet-stream` — binary downloads

## Vendor Media Types

Custom content types for API-specific formats:

```
Content-Type: application/vnd.myapp.order.v2+json
```

Encodes both the resource type and version. See [[api-003]] for versioning via media types and [[api-013]] for negotiation parsing.
