# Atomic State Management with Jotai

## Core Concept

**Jotai** takes an atomic approach to React state management. Each piece of state is an **atom**—a small, independent unit that components can subscribe to individually. This avoids unnecessary re-renders because components only update when the specific atoms they use change.

## Basic Usage

```typescript
import { atom, useAtom } from 'jotai';

const countAtom = atom(0);
const doubledAtom = atom((get) => get(countAtom) * 2);

function Counter() {
  const [count, setCount] = useAtom(countAtom);
  const [doubled] = useAtom(doubledAtom);

  return (
    <div>
      <p>Count: {count}, Doubled: {doubled}</p>
      <button onClick={() => setCount((c) => c + 1)}>Increment</button>
    </div>
  );
}
```

## Derived Atoms

Derived atoms compute values from other atoms. They are lazy—computed only when read—and memoized automatically. This is similar to **selectors** in Recoil or **computed** in MobX.

## Async Atoms

Jotai supports async atoms natively. An atom can return a promise, and Jotai integrates with React Suspense to handle loading states.

## Comparison with Zustand

While [[fe-002]] covers broader state management, the key difference is:

- **Zustand**: single store, selector-based subscriptions
- **Jotai**: many atoms, component-level subscriptions

Choose Jotai when you have many independent pieces of state that different components consume. Choose Zustand when state is more centralized with clear update patterns.
