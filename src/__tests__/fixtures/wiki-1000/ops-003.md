# GitHub Actions Workflows

## Repository Structure

Our GitHub Actions workflows live in `.github/workflows/`. Each workflow has a single responsibility.

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version-file: '.nvmrc'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm test
      - run: pnpm lint
```

## Caching Strategy

We cache `node_modules` via pnpm's built-in cache support in `actions/setup-node`. Docker layers are cached using `docker/build-push-action` with GitHub Actions cache backend.

## Secrets Management

Secrets are stored in GitHub's encrypted secrets store and injected as environment variables. Never echo secrets in logs. Use **OIDC federation** for cloud provider auth instead of long-lived credentials where possible.

## Reusable Workflows

Common patterns are extracted into **reusable workflows** under `.github/workflows/reusable-*.yml`. Teams call them with `uses: ./.github/workflows/reusable-deploy.yml`.
