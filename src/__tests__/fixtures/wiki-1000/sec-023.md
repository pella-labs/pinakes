# GDPR Technical Requirements

The **General Data Protection Regulation** imposes specific technical requirements on any system processing personal data of EU residents.

## Core Technical Obligations

### Data Minimization
Only collect and process personal data that is strictly necessary for the stated purpose. Review your database schema — if a column exists "just in case," you may be violating this principle.

### Right to Erasure
Users can request deletion of their personal data. Your system must be able to:

- Identify all personal data for a given user across all stores
- Delete or anonymize that data
- Handle cascading deletes in related tables
- Remove data from backups within a reasonable timeframe

### Data Portability
Users can request their data in a **machine-readable format** (typically JSON or CSV). Build an export endpoint that produces a complete data package.

### Consent Management
Track explicit consent for each processing purpose. Consent must be:

- Freely given
- Specific to a purpose
- Informed (user knows what they're consenting to)
- Revocable at any time

## Technical Implementation

- Tag all personal data fields in your schema
- Implement a **data subject access request** (DSAR) pipeline
- Maintain a **Record of Processing Activities** (ROPA)
- Apply **pseudonymization** where possible

See [[sec-024]] for HIPAA requirements and [[sec-022]] for SOC2 overlap.
