# API Response Envelopes

## Consistent Wrapping

Every response follows the same shape:

```json
{
  "data": { ... },
  "meta": {
    "request_id": "req_abc",
    "timestamp": "2025-11-03T14:30:00Z",
    "version": "2.1.0"
  }
}
```

For lists:

```json
{
  "data": [...],
  "meta": { "request_id": "req_abc", "timestamp": "..." },
  "pagination": {
    "next_cursor": "...",
    "has_more": true,
    "total_count": 1234
  }
}
```

For errors:

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Order not found",
    "request_id": "req_abc"
  }
}
```

## Why Envelopes

- Consistent parsing logic on the client
- Metadata always available without special headers
- Room for pagination without polluting the data
- Error shape is always the same

## Trade-off

Envelopes add bytes. For high-frequency internal APIs, skip the envelope and use headers for metadata.
