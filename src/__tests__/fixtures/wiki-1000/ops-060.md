# GitOps Directory Structure

## Repository Layout

Our GitOps repository follows a consistent structure that ArgoCD traverses automatically:

```
k8s-manifests/
  base/
    namespaces.yaml
    rbac.yaml
    network-policies.yaml
  infrastructure/
    cert-manager/
    external-secrets/
    ingress-controller/
    monitoring/
  services/
    api-server/
      base/
        deployment.yaml
        service.yaml
        ingress.yaml
      overlays/
        dev/
          kustomization.yaml
        staging/
          kustomization.yaml
        production/
          kustomization.yaml
    web-app/
      base/
      overlays/
  clusters/
    us-east-1/
      kustomization.yaml
    eu-west-1/
      kustomization.yaml
```

## Kustomize Overlays

Each environment uses **Kustomize overlays** to customize the base manifests. This keeps DRY principles while allowing environment-specific configuration like replica counts, resource limits, and image tags.

## ArgoCD App of Apps

We use the **App of Apps pattern** where a root ArgoCD Application points to the `clusters/` directory, which in turn contains Application manifests for each service. This provides a single point of entry for cluster reconciliation.
