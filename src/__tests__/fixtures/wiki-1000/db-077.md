# PostgreSQL Logical Replication

## Physical vs Logical

**Physical replication** streams WAL bytes — the replica is a byte-for-byte copy. **Logical replication** streams logical changes (inserts, updates, deletes) decoded from the WAL.

Logical replication allows:
- Selective replication (specific tables)
- Different indexes on publisher and subscriber
- Cross-version replication
- Data transformation during replication

## Setting Up

```sql
-- On the publisher
ALTER SYSTEM SET wal_level = logical;
CREATE PUBLICATION my_pub FOR TABLE users, orders;

-- On the subscriber
CREATE SUBSCRIPTION my_sub
  CONNECTION 'host=publisher.example.com dbname=myapp'
  PUBLICATION my_pub;
```

## Use Cases

- Migrating between PostgreSQL versions with zero downtime
- Feeding data to analytics databases
- Selective replication for microservices
- Implementing CDC without external tools (see [[db-026]])

## Limitations

- DDL changes are not replicated (schema changes must be applied manually)
- Sequences are not replicated
- Large objects (LOBs) are not supported
- Conflicts must be handled manually on the subscriber
