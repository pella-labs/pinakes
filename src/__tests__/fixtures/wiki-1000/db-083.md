# Redis Transactions

## MULTI/EXEC

Redis transactions use `MULTI` to start a transaction block and `EXEC` to execute all queued commands atomically:

```
MULTI
SET user:1:name "Alice"
INCR user:1:login_count
EXPIRE user:1:session 3600
EXEC
```

All commands between MULTI and EXEC are queued and executed as a single unit. No other client's commands are interleaved.

## Limitations

Redis transactions are NOT like SQL transactions:
- No rollback — if a command fails, the others still execute
- No isolation levels — other clients see intermediate states during EXEC
- WATCH provides optimistic locking but is not automatic

## WATCH for Optimistic Locking

```
WATCH user:1:balance
GET user:1:balance           -- returns "100"
MULTI
SET user:1:balance 90        -- deduct 10
EXEC
-- Returns nil if another client modified user:1:balance
```

## Lua Scripts

For complex atomic operations, Lua scripts are more powerful:

```lua
-- Atomic rate limiter
local current = redis.call('GET', KEYS[1])
if current and tonumber(current) >= tonumber(ARGV[1]) then
  return 0  -- rate limited
end
redis.call('INCR', KEYS[1])
redis.call('EXPIRE', KEYS[1], ARGV[2])
return 1  -- allowed
```
