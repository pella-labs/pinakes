# Cross-Origin Isolation

**Cross-origin isolation** is a browser security feature that provides stronger process isolation for your web application, enabling access to powerful APIs like `SharedArrayBuffer`.

## What It Does

Cross-origin isolation ensures your page runs in its own process, isolated from other origins. This prevents speculative execution attacks like Spectre.

## Required Headers

```
Cross-Origin-Embedder-Policy: require-corp
Cross-Origin-Opener-Policy: same-origin
```

### COEP (Cross-Origin-Embedder-Policy)

`require-corp` means every resource loaded by your page must explicitly opt in via:
- Being same-origin
- Having a `Cross-Origin-Resource-Policy` header
- Being served with appropriate CORS headers

### COOP (Cross-Origin-Opener-Policy)

`same-origin` ensures your page doesn't share a browsing context group with cross-origin pages. `window.opener` references from cross-origin pages become null.

## Migration Challenges

Enabling cross-origin isolation can break:
- Third-party scripts that don't set CORP headers
- Embedded iframes from other origins
- Payment widgets and social media embeds

Audit all cross-origin resources before enabling.

## Checking Isolation Status

```javascript
if (crossOriginIsolated) {
  // SharedArrayBuffer and high-resolution timers are available
}
```

See [[sec-007]] for CORS configuration and [[sec-008]] for CSP, which interact with these headers.
