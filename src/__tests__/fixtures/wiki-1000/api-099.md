# API Integration Patterns

## Synchronous Request-Response

Simplest pattern. Client sends request, waits for response.

**When:** immediate feedback needed, low latency acceptable.

## Async Request-Acknowledge

Client sends request, gets immediate `202 Accepted` with a job ID. Polls or receives webhook when done.

**When:** long-running operations (report generation, data processing).

```json
POST /reports/generate
→ 202 Accepted
{ "job_id": "job_123", "status_url": "/jobs/job_123" }
```

## Event-Driven

Services emit events. Interested services subscribe. No direct coupling.

**When:** loosely coupled systems, multiple consumers for the same event.

## Choreography vs Orchestration

**Choreography:** each service reacts to events independently. No central coordinator. Simple but hard to track the flow.

**Orchestration:** a central service (saga orchestrator) drives the workflow. Calls each service in sequence. Easier to monitor, single point of failure.

## Anti-Corruption Layer

When integrating with a third-party API, wrap it in an **adapter** that translates their domain model to yours. Don't let external API shapes leak into your domain.

```typescript
class StripeAdapter {
  async createCharge(amount: Money): Promise<PaymentResult> {
    const stripeResult = await stripe.charges.create({
      amount: amount.cents,
      currency: amount.currency,
    });
    return mapToPaymentResult(stripeResult);
  }
}
```
