# Database Sharding with Citus

## What Is Citus?

**Citus** is a PostgreSQL extension that transforms a single PostgreSQL database into a distributed database. It handles sharding, distributed queries, and rebalancing transparently.

## Distribution

```sql
-- Distribute tables by tenant_id
SELECT create_distributed_table('orders', 'tenant_id');
SELECT create_distributed_table('order_items', 'tenant_id');

-- Reference tables (small, replicated to all nodes)
SELECT create_reference_table('products');
```

## Co-located Joins

When tables share the same distribution column, joins between them execute locally on each shard — no data shuffling:

```sql
-- Fast: both distributed by tenant_id
SELECT o.id, oi.product_id
FROM orders o
JOIN order_items oi ON o.id = oi.order_id AND o.tenant_id = oi.tenant_id
WHERE o.tenant_id = 42;
```

## When to Use Citus

Citus shines for multi-tenant SaaS where all queries include a tenant_id filter. It's also good for real-time analytics on large datasets.
