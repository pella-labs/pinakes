# Headless CMS Integration

## What Is a Headless CMS

A **headless CMS** provides content management and an API but no frontend. Your application fetches content via REST or GraphQL and renders it however you choose.

## Popular Options

- **Sanity**: real-time collaboration, GROQ query language, generous free tier
- **Contentful**: enterprise-grade, GraphQL API, strong ecosystem
- **Strapi**: open-source, self-hosted, customizable
- **Keystatic**: file-based, Git-backed, works with Astro and Next.js

## Integration Pattern

```typescript
// lib/cms.ts
export async function getArticle(slug: string) {
  const response = await fetch(
    `https://cdn.sanity.io/v2021-03-25/data/query/production?query=*[_type == "article" && slug.current == $slug][0]`,
    {
      headers: { Authorization: `Bearer ${process.env.SANITY_TOKEN}` },
      next: { revalidate: 60 },
    }
  );
  return response.json();
}
```

## Content Modeling

Structure content as composable blocks rather than rigid page templates. A page is a sequence of content blocks (hero, text, image grid, CTA) that editors can assemble freely.

## Preview Mode

Implement draft preview mode so editors can see unpublished changes. Next.js provides `draftMode()` for this. The preview fetches from the draft API instead of the published CDN.
