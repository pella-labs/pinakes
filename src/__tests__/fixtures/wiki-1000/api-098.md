# Graceful API Shutdown

When deploying, you need to drain in-flight requests without dropping them.

## Sequence

1. Stop accepting new connections
2. Wait for in-flight requests to complete (with timeout)
3. Close database connections
4. Exit

```typescript
const server = app.listen(8080);

process.on('SIGTERM', async () => {
  logger.info('SIGTERM received, starting graceful shutdown');

  // Stop accepting new connections
  server.close(async () => {
    logger.info('All connections drained');
    await db.close();
    process.exit(0);
  });

  // Force exit after timeout
  setTimeout(() => {
    logger.warn('Forced shutdown after timeout');
    process.exit(1);
  }, 30000);
});
```

## Kubernetes Integration

Configure the pod's `terminationGracePeriodSeconds` to match your drain timeout:

```yaml
spec:
  terminationGracePeriodSeconds: 35  # slightly longer than app timeout
  containers:
    - name: api
      lifecycle:
        preStop:
          exec:
            command: ["sleep", "5"]  # wait for LB to remove pod
```

The 5-second `preStop` sleep gives the load balancer time to stop routing traffic before the app starts draining.

## WebSocket Shutdown

For WebSocket connections, send a close frame with a "server shutting down" reason. Give clients 10 seconds to reconnect to another server.
