# Cloud Cost Optimization

## FinOps Principles

**FinOps** is the practice of bringing financial accountability to cloud spending. It's not about cutting costs — it's about getting maximum value from cloud investment.

## Quick Wins

1. **Right-size instances**: most workloads are over-provisioned. Use AWS Compute Optimizer recommendations.
2. **Reserved Instances / Savings Plans**: commit to 1-year terms for predictable workloads. Savings of 30-60%.
3. **Delete unused resources**: unattached EBS volumes, idle load balancers, stopped instances with EBS.
4. **Spot instances** for fault-tolerant workloads (see [[ops-024]]).
5. **S3 lifecycle policies**: move infrequently accessed data to cheaper storage classes.

## Kubernetes Cost Optimization

- Set appropriate **resource requests and limits** (see [[ops-034]])
- Use **Karpenter** for efficient node provisioning
- Implement **cluster autoscaler** to remove underutilized nodes
- Schedule non-production workloads to scale down at night and weekends

## Tagging Strategy

Tag all resources with:
- `team` — who owns it
- `environment` — dev/staging/prod
- `service` — which application
- `cost-center` — for financial reporting

Untagged resources are flagged in weekly cost reviews.

## Monitoring

Use **AWS Cost Explorer** and **Kubecost** for visibility. Set budget alerts at 80% and 100% thresholds. Review costs weekly in the engineering standup.
