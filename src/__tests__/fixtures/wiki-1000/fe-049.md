# Edge Runtime in Next.js

## What Is the Edge Runtime

The **Edge Runtime** is a lightweight JavaScript environment that runs at CDN edge locations. It starts faster and has lower latency than Node.js but supports a restricted API surface.

## Middleware

Next.js Middleware runs on the Edge Runtime before every request:

```typescript
// middleware.ts
import { NextRequest, NextResponse } from 'next/server';

export function middleware(request: NextRequest) {
  const country = request.geo?.country ?? 'US';
  const locale = request.cookies.get('locale')?.value ?? 'en';

  if (request.nextUrl.pathname === '/') {
    return NextResponse.rewrite(
      new URL(`/${locale}/home`, request.url)
    );
  }

  return NextResponse.next();
}

export const config = {
  matcher: ['/((?!api|_next|static).*)'],
};
```

## Use Cases

- Authentication and authorization checks
- A/B testing and feature flags
- Geolocation-based routing
- Bot detection and rate limiting
- Request/response header manipulation

## Limitations

The Edge Runtime does not support:
- Node.js built-in modules (`fs`, `path`, `crypto` beyond Web Crypto)
- Native npm packages (anything using C++ bindings)
- Long-running processes

Keep middleware fast—it runs on every matching request. Heavy logic should be deferred to serverless functions or the origin server.
