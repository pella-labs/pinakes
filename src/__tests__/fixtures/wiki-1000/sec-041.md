# Clickjacking Prevention

**Clickjacking** tricks users into clicking hidden elements by overlaying transparent iframes on legitimate-looking pages. The user thinks they're clicking a button on the visible page, but they're actually interacting with the hidden frame.

## Defense Mechanisms

### X-Frame-Options Header

```
X-Frame-Options: DENY
```

Options: `DENY` (no framing), `SAMEORIGIN` (same origin only). The `ALLOW-FROM` directive is deprecated and not supported by modern browsers.

### CSP frame-ancestors

The modern replacement for X-Frame-Options:

```
Content-Security-Policy: frame-ancestors 'none'
```

Or to allow same-origin framing:

```
Content-Security-Policy: frame-ancestors 'self'
```

This is more flexible and supports multiple origins. Use both `X-Frame-Options` and `frame-ancestors` for backward compatibility.

### JavaScript Frame-Busting

As a fallback for very old browsers:

```javascript
if (self !== top) {
  top.location = self.location;
}
```

This is easily bypassed by sandbox attributes and should not be relied upon as the sole defense.

See [[sec-008]] for the broader CSP configuration and [[sec-033]] for the complete security headers checklist.
