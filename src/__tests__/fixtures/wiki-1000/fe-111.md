# React Strict Mode

## What It Does

**React Strict Mode** is a development-only tool that highlights potential problems. It intentionally double-invokes certain functions to surface side effects in the render phase.

## Behaviors in Development

- **Double-rendering**: components render twice to detect impure renders
- **Double-effects**: effects run, cleanup, and re-run to verify cleanup works
- **Deprecated API warnings**: flags usage of legacy lifecycle methods and string refs

## Why Double Effects Matter

If your effect doesn't clean up properly, double invocation exposes the bug:

```tsx
// Bug: no cleanup, creates two subscriptions
useEffect(() => {
  const sub = eventBus.subscribe('update', handler);
  // Missing: return () => sub.unsubscribe();
}, []);
```

In Strict Mode, this creates two subscriptions but only one is cleaned up, making the bug visible.

## Common False Alarms

- Network requests fire twice (use AbortController cleanup)
- Animations trigger twice (use cleanup to reset)
- Console logs appear doubled (expected behavior)

## Enabling

```tsx
<StrictMode>
  <App />
</StrictMode>
```

Strict Mode has zero production impact. It compiles away entirely. Leave it enabled.
