# Helm Charts

## Why Helm

**Helm** is the package manager for Kubernetes. It templates K8s manifests, manages releases, and enables version-controlled deployments. Without it, managing dozens of YAML files per service becomes unwieldy.

## Chart Structure

```
mychart/
  Chart.yaml          # metadata
  values.yaml         # default config
  values-staging.yaml # env override
  values-prod.yaml    # env override
  templates/
    deployment.yaml
    service.yaml
    ingress.yaml
    hpa.yaml
    _helpers.tpl
```

## Values Hierarchy

Helm merges values in order:

1. `values.yaml` (defaults)
2. `-f values-staging.yaml` (environment overlay)
3. `--set image.tag=abc123` (CLI overrides)

## Templating Tips

- Use `{{ include "mychart.fullname" . }}` for consistent naming
- Define common labels in `_helpers.tpl`
- Use `{{ .Values.image.tag | default .Chart.AppVersion }}` for sensible defaults
- Always quote strings in YAML: `{{ .Values.env | quote }}`

## Helm Hooks

Use **hooks** for database migrations (`pre-upgrade`), smoke tests (`post-install`), and cleanup (`pre-delete`). Be careful with hook weight ordering.
