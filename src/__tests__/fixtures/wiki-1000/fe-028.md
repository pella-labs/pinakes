# React Error Boundaries

## Catching Render Errors

**Error boundaries** are React components that catch JavaScript errors in their child component tree, log them, and display a fallback UI.

```tsx
class ErrorBoundary extends React.Component<
  { children: React.ReactNode; fallback: React.ReactNode },
  { hasError: boolean }
> {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  componentDidCatch(error: Error, errorInfo: React.ErrorInfo) {
    reportError(error, errorInfo);
  }

  render() {
    if (this.state.hasError) return this.props.fallback;
    return this.props.children;
  }
}
```

## Granular Boundaries

Place error boundaries strategically:
- Around the entire app (last resort)
- Around route-level content
- Around individual widgets that might fail independently

A failing chat widget should not crash the entire dashboard.

## React 19 Improvements

React 19 adds better error boundary APIs and integrates with the `use` hook for handling async errors.

## What Error Boundaries Do Not Catch

- Event handler errors (use try/catch)
- Async errors outside the render cycle
- Server-side rendering errors
- Errors in the error boundary itself
