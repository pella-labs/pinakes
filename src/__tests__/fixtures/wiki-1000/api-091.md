# GraphQL Caching Strategies

## HTTP Caching (Limited)

GraphQL typically uses POST, which isn't cached by CDNs. Workarounds:

- **GET for queries**: encode the query in the URL (limited by URL length)
- **Persisted queries** ([[api-056]]): use the hash as a cache key
- **CDN support**: Cloudflare and Fastly support GraphQL-aware caching

## Application-Level Caching

### Response Cache

Cache entire query results keyed by normalized query + variables:

```typescript
const cacheKey = sha256(JSON.stringify({
  query: normalizeQuery(document),
  variables,
}));
```

### Field-Level Cache

Use `@cacheControl` directive per field:

```graphql
type Product @cacheControl(maxAge: 300) {
  id: ID!
  name: String!
  price: Money! @cacheControl(maxAge: 60)
  inventory: Int! @cacheControl(maxAge: 0)
}
```

The response `max-age` is the minimum of all resolved fields.

## Dataloader as Cache

DataLoader caches within a single request. For cross-request caching, use Redis-backed loaders. See [[api-039]] for the DataLoader pattern.
