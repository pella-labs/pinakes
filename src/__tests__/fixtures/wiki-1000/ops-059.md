# Container Orchestration Comparison

## Kubernetes vs ECS vs Nomad

Choosing a **container orchestrator** is a significant infrastructure decision. Here's how the main options compare.

### Kubernetes

The industry standard. Massive ecosystem, runs anywhere, highly extensible. But complex to operate, steep learning curve, and overkill for small teams.

### ECS (Elastic Container Service)

AWS-native, simpler than Kubernetes. Fargate removes node management entirely. Limited ecosystem compared to K8s, AWS lock-in.

### Nomad

HashiCorp's orchestrator. Simpler than K8s, supports non-container workloads (Java, binaries). Smaller community, fewer integrations.

## Decision Framework

Choose **ECS/Fargate** if:
- AWS-only, small team, want simplicity
- Running fewer than 20 services
- Don't need the K8s ecosystem

Choose **Kubernetes** if:
- Multi-cloud or hybrid
- Large team with K8s expertise
- Need the ecosystem (Istio, ArgoCD, etc.)
- Running 20+ services

Choose **Nomad** if:
- Mixed workloads (containers + VMs + raw binaries)
- Already invested in HashiCorp stack
- Want something simpler than K8s but more flexible than ECS

We chose Kubernetes for the ecosystem and multi-cloud flexibility. See [[ops-004]] for K8s fundamentals.
