# Terraform Destroy and Lifecycle

## Resource Lifecycle

Control how Terraform manages resource creation and destruction:

```hcl
resource "aws_instance" "web" {
  ami           = var.ami_id
  instance_type = "t3.medium"

  lifecycle {
    create_before_destroy = true   # Create replacement before destroying old
    prevent_destroy       = true   # Block terraform destroy
    ignore_changes        = [tags] # Don't track tag changes
  }
}
```

## Targeted Destroy

Destroy specific resources without affecting others:

```bash
terraform destroy -target=aws_instance.temp_worker
```

Use sparingly — it can leave state inconsistent. Prefer removing the resource block and running `apply`.

## Moved Blocks

When refactoring, use `moved` blocks to avoid destroy/recreate:

```hcl
moved {
  from = aws_instance.web
  to   = module.compute.aws_instance.web
}
```

## Decommissioning

When removing infrastructure permanently:

1. Remove DNS records first (let TTL expire)
2. Remove from load balancer targets
3. Take final backups
4. Run `terraform destroy` on the specific resources
5. Clean up state file references
6. Remove the Terraform code
7. Document the decommission in a PR

See [[ops-006]] for Terraform fundamentals.
