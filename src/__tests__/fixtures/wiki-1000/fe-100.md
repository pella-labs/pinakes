# CI/CD for Frontend Monorepos

## Affected-Only Builds

In a monorepo, rebuilding everything on every commit wastes time. Use affected detection to build and test only what changed.

## Turborepo Remote Caching

```yaml
# .github/workflows/ci.yml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm turbo build --filter='...[HEAD^1]'
        env:
          TURBO_TOKEN: ${{ secrets.TURBO_TOKEN }}
          TURBO_TEAM: ${{ secrets.TURBO_TEAM }}
```

The `--filter='...[HEAD^1]'` flag builds only packages that changed since the previous commit, including their dependents.

## Parallel Test Execution

Split test suites across CI runners:

```yaml
strategy:
  matrix:
    shard: [1, 2, 3, 4]
steps:
  - run: pnpm playwright test --shard=${{ matrix.shard }}/4
```

## Deploy Previews

Deploy a preview environment for every PR. Each package gets its own preview URL. Use URL rewrites to compose micro-frontends in the preview.

## Cache Strategy

- **pnpm store**: cached between runs
- **Turbo cache**: shared across all branches via remote cache
- **Playwright browsers**: cached via actions/cache
