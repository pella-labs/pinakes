# PostgreSQL Roles and Permissions

## Role Hierarchy

PostgreSQL uses **roles** for both users and groups. A role can be a member of other roles, inheriting their privileges.

```sql
-- Create roles
CREATE ROLE readonly;
CREATE ROLE readwrite;
CREATE ROLE app_user WITH LOGIN PASSWORD 'secure_pass';

-- Grant privileges
GRANT USAGE ON SCHEMA public TO readonly;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly;
GRANT INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO readwrite;

-- Role inheritance
GRANT readonly TO readwrite;
GRANT readwrite TO app_user;
```

## Default Privileges

Set defaults for future objects:

```sql
ALTER DEFAULT PRIVILEGES IN SCHEMA public
GRANT SELECT ON TABLES TO readonly;

ALTER DEFAULT PRIVILEGES IN SCHEMA public
GRANT INSERT, UPDATE, DELETE ON TABLES TO readwrite;
```

## Schema Isolation

Use schemas to separate concerns:

```sql
CREATE SCHEMA analytics;
GRANT USAGE ON SCHEMA analytics TO data_team;
GRANT SELECT ON ALL TABLES IN SCHEMA analytics TO data_team;
-- data_team can read analytics but not public tables
```

For row-level access control, see [[db-061]] on RLS.
