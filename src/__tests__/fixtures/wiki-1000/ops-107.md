# AWS ElastiCache Operations

## Redis vs Memcached

- **Redis**: data structures, persistence, replication, Lua scripting. Use for sessions, queues, leaderboards.
- **Memcached**: simple key-value, multi-threaded. Use for simple caching where persistence isn't needed.

We use Redis for everything because the operational overhead of maintaining two caching systems isn't worth the marginal performance gain of Memcached.

## Cluster Mode

Enable **cluster mode** for horizontal scaling. Data is sharded across multiple nodes. Each shard has a primary and replicas.

## Configuration

```hcl
resource "aws_elasticache_replication_group" "redis" {
  replication_group_id       = "production-redis"
  description                = "Production Redis cluster"
  node_type                  = "cache.r6g.large"
  num_cache_clusters         = 3
  automatic_failover_enabled = true
  multi_az_enabled           = true
  at_rest_encryption_enabled = true
  transit_encryption_enabled = true
  engine_version             = "7.0"

  parameter_group_name = aws_elasticache_parameter_group.redis.name
  subnet_group_name    = aws_elasticache_subnet_group.redis.name
}
```

## Monitoring

Watch these metrics:
- **Memory usage**: alert at 80% used memory
- **Evictions**: non-zero means you need more memory
- **Cache hit ratio**: should be > 90%
- **Replication lag**: should be < 1 second

See [[ops-035]] for database operations.
