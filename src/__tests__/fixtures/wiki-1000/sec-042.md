# Subresource Integrity

**Subresource Integrity** (SRI) ensures that files fetched from CDNs or third-party origins haven't been tampered with. The browser verifies a cryptographic hash before executing the resource.

## How It Works

```html
<script src="https://cdn.example.com/lib.js"
        integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/uxy9rx7HNQlGYl1kPzQho1wx4JwY8w"
        crossorigin="anonymous"></script>
```

The browser downloads the file, computes its SHA-384 hash, and compares it to the `integrity` attribute. If they don't match, the browser refuses to execute the script.

## When to Use SRI

- Any script or stylesheet loaded from a CDN
- Third-party analytics or widget scripts
- Any resource where you don't control the hosting

## Generating Hashes

```bash
cat lib.js | openssl dgst -sha384 -binary | openssl base64 -A
```

Or use the `srihash.org` tool. Most package managers and build tools can generate SRI hashes automatically.

## Limitations

SRI only works for `<script>` and `<link>` elements. It doesn't cover images, fonts, or other resource types. For broader protection, combine with a strict [[sec-008]] Content Security Policy.
