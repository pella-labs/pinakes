# API Tenant Data Isolation

## Query-Level Enforcement

Every database query must include the tenant filter:

```typescript
class TenantRepository {
  constructor(private tenantId: string) {}

  async findOrders(filters: OrderFilters): Promise<Order[]> {
    return db.query(
      `SELECT * FROM orders 
       WHERE tenant_id = $1 AND status = $2 
       ORDER BY created_at DESC LIMIT $3`,
      [this.tenantId, filters.status, filters.limit]
    );
  }
}
```

## Row-Level Security (RLS)

PostgreSQL can enforce tenant isolation at the database level:

```sql
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;

CREATE POLICY tenant_isolation ON orders
  USING (tenant_id = current_setting('app.tenant_id'));

-- Set per-connection
SET app.tenant_id = 'tenant_abc';
```

## Testing Isolation

3. Search results don't leak across tenants
4. Aggregate queries (counts, sums) are tenant-scoped
