# API Connection Pooling

## Why Pool

Creating a TCP + TLS connection takes ~50ms. For APIs calling databases or downstream services, this adds up.

## HTTP Connection Pool

```typescript
import { Agent } from 'http';

const agent = new Agent({
  keepAlive: true,
  maxSockets: 50,          // per host
  maxFreeSockets: 10,
  timeout: 60000,
  keepAliveMsecs: 30000,
});

const response = await fetch('http://downstream/api', { agent });
```

## Database Connection Pool

```typescript
const pool = new Pool({
  max: 20,                  // max connections
  min: 5,                   // min idle connections
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 5000,
});
```

## Pool Sizing

**Formula**: connections = (core_count * 2) + effective_spindle_count

For most APIs: 10-20 connections per instance. Too many connections cause context switching overhead at the database. Too few cause request queuing.

## Monitoring

Track:
- Pool utilization (active / max)
- Wait time (how long requests wait for a connection)
- Connection errors
- Connection age (recycle long-lived connections to prevent leaks)

Alert if pool utilization exceeds 80% — it's a capacity signal.
