# Module Federation for Micro-Frontends

## How It Works

**Module Federation** in Webpack 5 allows JavaScript applications to dynamically load code from other applications at runtime. Each application can expose modules and consume modules from others.

## Configuration

```javascript
// Host application
new ModuleFederationPlugin({
  name: 'host',
  remotes: {
    dashboard: 'dashboard@https://cdn.example.com/dashboard/remoteEntry.js',
    settings: 'settings@https://cdn.example.com/settings/remoteEntry.js',
  },
  shared: {
    react: { singleton: true, requiredVersion: '^18.0.0' },
    'react-dom': { singleton: true, requiredVersion: '^18.0.0' },
  },
});

// Remote (dashboard) application
new ModuleFederationPlugin({
  name: 'dashboard',
  filename: 'remoteEntry.js',
  exposes: {
    './Widget': './src/DashboardWidget',
  },
  shared: {
    react: { singleton: true },
    'react-dom': { singleton: true },
  },
});
```

## Consuming Remote Modules

```tsx
const DashboardWidget = React.lazy(() => import('dashboard/Widget'));

function App() {
  return (
    <Suspense fallback={<Spinner />}>
      <DashboardWidget />
    </Suspense>
  );
}
```

## Versioning and Shared Dependencies

The `shared` configuration prevents loading multiple copies of React. If versions are incompatible, Module Federation falls back to loading separate copies.
