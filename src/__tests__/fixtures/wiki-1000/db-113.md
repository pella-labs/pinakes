# PostgreSQL Generated Columns

## Stored Generated Columns

A **generated column** is computed from other columns in the same row. PostgreSQL supports stored generated columns (computed on write, stored on disk):

```sql
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  price_cents INT NOT NULL,
  tax_rate DECIMAL NOT NULL DEFAULT 0.08,
  total_cents INT GENERATED ALWAYS AS (price_cents + floor(price_cents * tax_rate)::int) STORED
);
```

## Use Cases

- Full name from first + last name
- Computed hashes or checksums
- Normalized values for indexing
- Search vectors

```sql
CREATE TABLE articles (
  id SERIAL PRIMARY KEY,
  title TEXT NOT NULL,
  body TEXT NOT NULL,
  search_vector tsvector GENERATED ALWAYS AS (
    setweight(to_tsvector('english', title), 'A') ||
    setweight(to_tsvector('english', body), 'B')
  ) STORED
);

CREATE INDEX idx_search ON articles USING GIN (search_vector);
```

## Limitations

- Can only reference columns in the same row (not other tables)
- Cannot be written to directly
- PostgreSQL only supports STORED (not VIRTUAL/computed-on-read)
- The expression must be immutable (no `now()`, no sequence references)

For full-text search indexing, see [[db-033]].
