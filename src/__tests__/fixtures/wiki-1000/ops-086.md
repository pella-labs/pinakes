# Kubernetes Pod Priority and Preemption

## Pod Priority

**Pod Priority** ensures critical workloads get scheduled even when the cluster is full. Higher-priority pods can preempt lower-priority ones.

## Priority Classes

```yaml
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: critical
value: 1000000
globalDefault: false
description: "For critical production services"
---
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: default
value: 0
globalDefault: true
description: "Default priority for all pods"
---
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: batch
value: -100
globalDefault: false
preemptionPolicy: Never
description: "Low-priority batch jobs that never preempt"
```

## Using Priority

```yaml
spec:
  priorityClassName: critical
  containers:
    - name: api
      image: api:latest
```

## Preemption Behavior

When a high-priority pod can't be scheduled, the scheduler finds lower-priority pods that can be evicted to make room. Preempted pods get a graceful termination period.

Set `preemptionPolicy: Never` on batch jobs so they wait for capacity instead of evicting other pods.

See [[ops-041]] for scheduling and [[ops-014]] for autoscaling.
