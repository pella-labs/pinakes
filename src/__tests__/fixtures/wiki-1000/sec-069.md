# Security Testing in CI

Integrating **security testing** into CI/CD ensures vulnerabilities are caught before they reach production. Shift security left without slowing down delivery.

## Static Analysis (SAST)

Scan source code for vulnerability patterns:

- **Semgrep**: language-agnostic, customizable rules
- **ESLint security plugins**: for JavaScript/TypeScript
- **Bandit**: for Python
- **gosec**: for Go

Run SAST on every pull request. Focus on high-confidence rules to minimize false positives.

## Dependency Scanning

Check for known vulnerabilities in dependencies:

```yaml
# GitHub Actions example
- name: Security audit
  run: npm audit --audit-level=high
```

Fail the build on high/critical vulnerabilities. See [[sec-055]] for dependency management strategy.

## Secret Detection

Scan for accidentally committed secrets:

```yaml
- name: Secret scan
  uses: trufflesecurity/trufflehog@main
  with:
    extra_args: --only-verified
```

Run on every push, not just PRs.

## Dynamic Analysis (DAST)

Run against deployed staging environments:

- **OWASP ZAP**: open-source web scanner
- **Nuclei**: template-based vulnerability scanner

DAST is slower and noisier than SAST, so run it nightly or on release branches rather than every PR.

## Container Scanning

Scan built images before pushing to registry:

```yaml
- name: Scan image
  run: trivy image --severity HIGH,CRITICAL my-app:latest
```
