# Infrastructure Drift Detection

## What is Drift

**Infrastructure drift** occurs when the actual state of cloud resources diverges from the desired state defined in code. This happens through manual changes, out-of-band automation, or failed applies.

## Detection Methods

### Terraform Plan

Run `terraform plan` regularly (daily in CI) to detect drift:

```bash
terraform plan -detailed-exitcode
# Exit code 0: no changes
# Exit code 1: error
# Exit code 2: changes detected (drift)
```

### AWS Config Rules

**AWS Config** continuously monitors resource configuration and flags non-compliant resources. Use it as a backup to Terraform-based drift detection.

### Custom Scripts

For resources not managed by Terraform, write custom scripts that compare expected state (from a config file) with actual state (from API calls).

## Remediation

When drift is detected:

1. Determine if the manual change was intentional
2. If yes, import the change into Terraform
3. If no, run `terraform apply` to restore desired state
4. Investigate how the drift occurred and prevent recurrence

## Prevention

- Restrict console access with IAM (read-only for most users)
- Use SCPs to block manual changes to managed resources
- Alert on CloudTrail events that modify Terraform-managed resources

See [[ops-006]] for Terraform and [[ops-037]] for infrastructure testing.
