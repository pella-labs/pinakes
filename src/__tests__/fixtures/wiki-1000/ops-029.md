# Service Mesh with Istio

## What is a Service Mesh

A **service mesh** manages service-to-service communication. It handles traffic management, security (mTLS), and observability without changing application code.

## Istio Architecture

Istio consists of:

- **Envoy sidecars** — proxy containers injected alongside each application pod
- **Istiod** — control plane that configures proxies
- **Ingress/Egress gateways** — manage traffic entering and leaving the mesh

## Traffic Management

Istio enables sophisticated traffic routing:

```yaml
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: reviews
spec:
  hosts:
    - reviews
  http:
    - match:
        - headers:
            x-canary:
              exact: "true"
      route:
        - destination:
            host: reviews
            subset: v2
    - route:
        - destination:
            host: reviews
            subset: v1
```

## mTLS

Istio provides **mutual TLS** between all services in the mesh automatically. This encrypts all inter-service traffic and provides identity-based authentication.

## When to Use a Service Mesh

A service mesh adds complexity. It's worth it when you have many services that need consistent security, observability, and traffic management policies. For fewer than 10 services, the overhead may not be justified.
