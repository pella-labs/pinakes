# CSS Cascade Layers

## What Are Cascade Layers

**@layer** gives you explicit control over the CSS cascade. You define the order of layers, and styles in later layers win over earlier ones regardless of specificity.

```css
@layer reset, base, components, utilities;

@layer reset {
  *, *::before, *::after {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
  }
}

@layer base {
  body {
    font-family: system-ui, sans-serif;
    line-height: 1.6;
    color: #111827;
  }
}

@layer components {
  .button {
    padding: 8px 16px;
    border-radius: 4px;
    background: #3b82f6;
    color: white;
  }
}

@layer utilities {
  .mt-4 { margin-top: 1rem; }
  .hidden { display: none; }
}
```

## Why They Matter

Without layers, a utility class like `.mt-4` (specificity 0,1,0) loses to a component class `.card .header` (specificity 0,2,0). With layers, utilities always win because their layer comes last.

## Integration with Tailwind

Tailwind v3.1+ uses `@layer` internally. The `base`, `components`, and `utilities` layers correspond to Tailwind's layer concept.

## Third-Party CSS

Wrap third-party styles in a low-priority layer to prevent them from overriding your styles:

```css
@layer vendor {
  @import 'third-party-lib.css';
}
```
