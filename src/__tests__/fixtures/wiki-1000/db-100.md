# Data Warehouse Architecture

## Reference Architecture

A modern data warehouse follows a layered pattern:

### Raw Layer (Bronze)
Exact copies of source data. No transformations. Stored in a data lake (S3, GCS) or raw schema.

### Cleaned Layer (Silver)
Deduplication, type casting, null handling. Consistent naming conventions.

### Business Layer (Gold)
Aggregated, joined, and modeled data ready for analytics. Star or snowflake schema (see [[db-036]]).

## dbt for Transformations

**dbt** (data build tool) manages the Silver → Gold transformations as SQL models:

```sql
-- models/gold/daily_revenue.sql
{{ config(materialized='table') }}

SELECT
  date_trunc('day', o.created_at) AS date,
  p.category,
  sum(o.amount) AS revenue,
  count(distinct o.customer_id) AS unique_customers
FROM {{ ref('silver_orders') }} o
JOIN {{ ref('silver_products') }} p ON o.product_id = p.id
WHERE o.status = 'completed'
GROUP BY 1, 2
```

## Key Principles

- **Immutable raw data**: never modify the raw layer
- **Idempotent transformations**: rerunning produces the same result
- **Data lineage**: track which sources feed each model
- **Freshness monitoring**: alert when data is stale
