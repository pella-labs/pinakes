# PostgreSQL LISTEN/NOTIFY

## Real-Time Notifications

PostgreSQL's **LISTEN/NOTIFY** mechanism enables real-time event notifications between database sessions. No polling required.

## How It Works

```sql
-- Session 1: Listen for events
LISTEN new_orders;

-- Session 2: Send a notification
NOTIFY new_orders, '{"order_id": 1234, "total": 99.99}';
```

## Using with Node.js

```typescript
import { Client } from 'pg';

const client = new Client({ connectionString: process.env.DATABASE_URL });
await client.connect();

await client.query('LISTEN new_orders');

client.on('notification', (msg) => {
  const payload = JSON.parse(msg.payload!);
  console.log('New order:', payload.order_id);
});
```

## Trigger Integration

Combine with triggers for automatic notifications on data changes:

```sql
CREATE OR REPLACE FUNCTION notify_order_created()
RETURNS TRIGGER AS $$
BEGIN
  PERFORM pg_notify('new_orders', json_build_object(
    'order_id', NEW.id,
    'total', NEW.total
  )::text);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER order_created_trigger
AFTER INSERT ON orders
FOR EACH ROW EXECUTE FUNCTION notify_order_created();
```

## Limitations

- Payload limited to 8000 bytes
- Notifications are lost if no one is listening
- Requires a dedicated connection (can't use with PgBouncer in transaction mode — see [[db-045]])
