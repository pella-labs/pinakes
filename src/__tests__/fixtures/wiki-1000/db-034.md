# Database Backup Strategies

## Types of Backups

### Logical Backups
Export data as SQL statements or structured formats. Tools: `pg_dump`, `mysqldump`, `mongodump`.

Pros: portable, human-readable, selective (per-table)
Cons: slow for large databases, restore is slow

### Physical Backups
Copy the raw data files. Tools: `pg_basebackup`, Percona XtraBackup, filesystem snapshots.

Pros: fast backup and restore, consistent point-in-time
Cons: tied to the same database version, all-or-nothing

### Continuous Archiving (PITR)
Combine a base backup with archived WAL segments. Enables restoring to any point in time — not just when the backup was taken.

```bash
# PostgreSQL continuous archiving
archive_mode = on
archive_command = 'cp %p /backup/wal/%f'
```

## Backup Testing

A backup that hasn't been tested is not a backup. Schedule regular restore tests:

- Restore to a standby and run validation queries
- Verify row counts and checksums
- Time the restore process

## Retention Policy

Keep daily backups for 7 days, weekly for 4 weeks, monthly for 12 months. Adjust based on compliance requirements and storage costs. For large databases, consider incremental backups with tools like pgBackRest.
