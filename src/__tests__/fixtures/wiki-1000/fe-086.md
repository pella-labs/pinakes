# SWR vs TanStack Query

## SWR by Vercel

**SWR** (stale-while-revalidate) is a lightweight data fetching library focused on simplicity:

```typescript
import useSWR from 'swr';

function Profile() {
  const { data, error, isLoading } = useSWR('/api/user', fetcher);
  if (isLoading) return <Spinner />;
  if (error) return <Error />;
  return <div>{data.name}</div>;
}
```

## TanStack Query

**TanStack Query** is more feature-rich with explicit cache management:

```typescript
import { useQuery } from '@tanstack/react-query';

function Profile() {
  const { data, error, isLoading } = useQuery({
    queryKey: ['user'],
    queryFn: fetchUser,
    staleTime: 5 * 60 * 1000,
  });
}
```

## Comparison

| Feature | SWR | TanStack Query |
|---------|-----|----------------|
| Bundle size | ~4KB | ~12KB |
| Mutations | Basic | Full (with optimistic) |
| Devtools | Limited | Excellent |
| Infinite queries | Yes | Yes |
| Suspense | Yes | Yes |
| Offline | Basic | Advanced |
| Dependent queries | Yes | Yes |

## When to Choose

- **SWR**: simple data fetching, Vercel/Next.js ecosystem, minimal overhead
- **TanStack Query**: complex cache invalidation, mutations, optimistic updates, offline support

See [[fe-030]] for TanStack Query deep dive and [[fe-024]] for optimistic updates.
