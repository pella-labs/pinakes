# Cursor-Based Pagination Implementation

## Encoding the Cursor

The cursor is an opaque string. Internally it encodes the sort position.

```typescript
interface CursorPayload {
  id: string;
  sortValue: string | number;
  sortField: string;
}

function encodeCursor(payload: CursorPayload): string {
  return Buffer.from(JSON.stringify(payload)).toString('base64url');
}

function decodeCursor(cursor: string): CursorPayload {
  return JSON.parse(Buffer.from(cursor, 'base64url').toString());
}
```

## Query Construction

```sql
-- Forward pagination: get 25 items after cursor
SELECT id, name, created_at
FROM products
WHERE (created_at, id) > (:cursor_created_at, :cursor_id)
ORDER BY created_at ASC, id ASC
LIMIT 26; -- fetch one extra to determine has_more
```

Fetch `limit + 1` rows. If you get the extra row, `has_more = true` and you strip it from results.

## Backward Pagination

For `last` / `before`, reverse the sort and comparison:

```sql
WHERE (created_at, id) < (:cursor_created_at, :cursor_id)
ORDER BY created_at DESC, id DESC
LIMIT 26;
```

Then reverse the results in application code to maintain ascending order.

## Edge Cases

- Empty collection: return `{ edges: [], pageInfo: { hasNextPage: false, endCursor: null } }`
- Cursor points to deleted item: use `>=` and deduplicate, or return an error
- Sort column has NULLs: use `NULLS LAST` and handle in cursor encoding
