# GraphQL Query Whitelisting

## Threat Model

Public GraphQL APIs accept arbitrary queries. A malicious query can:

- Request deeply nested data (DoS via N+1)
- Use `__schema` introspection to map the entire API
- Craft expensive queries that bypass simple depth limits

## Allowlist Approach

Only permit queries that exist in a pre-registered allowlist:

```typescript
const allowedQueries = new Map<string, string>([
  ['abc123', 'query GetOrder($id: ID!) { order(id: $id) { id status total } }'],
  ['def456', 'query ListOrders($first: Int) { orders(first: $first) { edges { node { id } } } }'],
]);

function validateQuery(hash: string): boolean {
  return allowedQueries.has(hash);
}
```

## Build-Time Registration

Client apps generate the query map at build time:
1. Extract all GraphQL operations from the frontend codebase
2. Hash each query (SHA-256)
3. Upload the map to the server as part of deployment

## Introspection

Disable `__schema` and `__type` in production. Only enable in development/staging. This prevents API discovery by unauthorized parties.
