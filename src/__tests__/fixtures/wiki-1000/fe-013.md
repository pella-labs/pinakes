# Svelte: Compile-Time Reactivity

## No Virtual DOM

**Svelte** takes a fundamentally different approach from React and Vue. Instead of shipping a runtime that diffs a virtual DOM, Svelte compiles your components into efficient imperative code that directly updates the DOM.

## Reactivity Through Assignment

In Svelte, reactivity is triggered by assignment. There is no `setState` or `.value`—just assign to a variable and the DOM updates.

```svelte
<script>
  let count = 0;
  $: doubled = count * 2;

  function increment() {
    count += 1;
  }
</script>

<button on:click={increment}>
  {count} (doubled: {doubled})
</button>
```

The `$:` label marks a **reactive declaration**. Svelte's compiler analyzes dependencies at build time and generates precise update instructions.

## SvelteKit

**SvelteKit** is the official application framework for Svelte, analogous to Next.js for React. It provides file-based routing, SSR, and adapters for different deployment targets.

## Bundle Size

Because Svelte compiles away the framework, the baseline bundle size is significantly smaller than React or Vue. A simple Svelte app might ship 5-10KB of JavaScript compared to 40-50KB for a React equivalent.

## Tradeoffs

Svelte's compile-time approach means less runtime overhead but larger per-component output. For very large applications with hundreds of components, the total bundle can approach React's size. The sweet spot is small-to-medium applications where initial load time matters.
