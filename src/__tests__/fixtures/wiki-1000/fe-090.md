# Web Font Optimization

## The Problem

Custom fonts block text rendering until they download. This causes **Flash of Invisible Text** (FOIT) or **Flash of Unstyled Text** (FOUT).

## font-display

Control how fonts load with the `font-display` descriptor:

```css
@font-face {
  font-family: 'Inter';
  src: url('/fonts/inter.woff2') format('woff2');
  font-display: swap; /* Show fallback immediately, swap when loaded */
}
```

- `swap`: show fallback, swap when loaded (recommended for body text)
- `optional`: use font only if it loads very quickly (recommended for non-essential fonts)
- `fallback`: brief invisible period, then fallback
- `block`: invisible text until font loads (avoid)

## Preloading

Preload fonts that appear above the fold:

```html
<link rel="preload" href="/fonts/inter.woff2" as="font" type="font/woff2" crossorigin />
```

## Subsetting

Most fonts include characters for every language. Subset to include only the characters you need:

```css
@font-face {
  font-family: 'Inter';
  src: url('/fonts/inter-latin.woff2') format('woff2');
  unicode-range: U+0000-00FF, U+0131, U+0152-0153;
}
```

## Variable Fonts

A single **variable font** file replaces multiple weight/style files. One file for all weights reduces requests and total download size.

## Size Budgets

- Body font: under 50KB (woff2, subset)
- Heading font: under 30KB
- Total fonts: under 100KB
