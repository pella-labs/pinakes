# Kubernetes Ephemeral Containers

## Debugging Running Pods

**Ephemeral containers** let you debug a running pod without restarting it. This is essential for production debugging where you can't add debug tools to the production image.

```bash
kubectl debug -it pod/api-server-abc123 \
  --image=busybox:1.36 \
  --target=api-server
```

This attaches a new container to the existing pod, sharing the process namespace of the target container.

## Use Cases

- Inspect the filesystem of a distroless container
- Run network debugging tools (curl, dig, tcpdump)
- Attach a profiler to a running process
- Check environment variables and mounted secrets

## Node Debugging

Debug node-level issues by creating a privileged debugging pod:

```bash
kubectl debug node/ip-10-0-1-42 -it --image=ubuntu
```

This creates a pod with host filesystem access at `/host`.

## Limitations

Ephemeral containers cannot be removed once added — they stay until the pod is deleted. They also cannot have ports, readiness probes, or resource limits.

See [[ops-077]] for general troubleshooting.
