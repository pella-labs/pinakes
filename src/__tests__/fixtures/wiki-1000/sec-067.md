# Mobile Application Security

Mobile apps face unique security threats including device theft, reverse engineering, and insecure local storage.

## Secure Storage

- **iOS**: use the Keychain for sensitive data
- **Android**: use the Android Keystore system
- **Never** use SharedPreferences (Android) or UserDefaults (iOS) for secrets
- Encrypt local databases with SQLCipher

## Network Security

- Use TLS for all connections (see [[sec-017]])
- Consider certificate pinning (see [[sec-010]])
- Implement network security config (Android) or App Transport Security (iOS)
- Don't trust user-installed CAs in production builds

## Code Protection

- Enable code obfuscation (ProGuard/R8 for Android, bitcode for iOS)
- Detect rooted/jailbroken devices
- Implement runtime integrity checks
- Use anti-tampering measures for sensitive applications

## Authentication

- Use biometric authentication for local unlocking
- Store refresh tokens in secure storage (see [[sec-032]])
- Implement session timeout
- Support remote session revocation

## Data Leakage Prevention

- Clear sensitive data from memory when backgrounding
- Disable screenshots for sensitive screens
- Don't log sensitive data to system logs
- Validate data shared with other apps via intents/deep links
