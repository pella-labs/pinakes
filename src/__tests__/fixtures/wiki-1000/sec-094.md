# Secure Backup and Recovery

**Backup security** is often overlooked, but backups contain the same sensitive data as production — they need the same protection.

## Backup Encryption

Encrypt all backups:

- Use strong encryption (AES-256-GCM)
- Manage encryption keys separately from backups (see [[sec-048]])
- Store key material in a KMS, not alongside the backups
- Test decryption regularly — encrypted backups you can't decrypt are useless

## Access Control

- Restrict backup access to a minimal set of operators
- Use separate credentials for backup operations
- Audit all backup access (see [[sec-014]])
- Implement break-glass procedures for emergency restores

## Backup Testing

Regularly test the full restore process:

- Monthly restore to a non-production environment
- Verify data integrity after restore
- Measure restore time against your RTO (Recovery Time Objective)
- Document and maintain restore procedures

## Retention and Disposal

- Define retention periods based on compliance requirements
- Implement automated backup rotation
- Securely delete expired backups (cryptographic erasure for encrypted backups)
- Document the disposal process for audit

## Cloud Backup Considerations

- Enable versioning on cloud storage (prevents accidental or malicious deletion)
- Use Object Lock / immutable storage for compliance
- Cross-region replication for disaster recovery
- Monitor for unexpected bulk deletions or modifications
