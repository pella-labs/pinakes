# Hydration and Its Pitfalls

## What Is Hydration

**Hydration** is the process of attaching event handlers and client-side state to server-rendered HTML. React walks the DOM tree, matches it to the component tree, and makes it interactive.

## Hydration Mismatches

A mismatch occurs when the server-rendered HTML differs from what React produces on the client. Common causes:

- Using `Date.now()` or `Math.random()` during render
- Accessing `window` or `localStorage` during server render
- Different rendering based on user agent or screen size
- Browser extensions modifying the DOM before hydration

React 18 handles mismatches more gracefully but still warns in development.

## Solutions

Use `useEffect` for client-only logic:

```tsx
function TimeDisplay() {
  const [time, setTime] = useState<string | null>(null);

  useEffect(() => {
    setTime(new Date().toLocaleTimeString());
  }, []);

  if (!time) return <span>Loading...</span>;
  return <span>{time}</span>;
}
```

Or use the `suppressHydrationWarning` prop for intentional mismatches like timestamps.

## Partial Hydration

Not every component needs to hydrate. **Islands architecture** (used by Astro and Fresh) hydrates only interactive components, leaving the rest as static HTML. This reduces the JavaScript needed to make a page interactive. See [[fe-011]] for React Server Components as an alternative.
