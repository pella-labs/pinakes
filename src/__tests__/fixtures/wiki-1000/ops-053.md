# AWS IAM Best Practices

## Principle of Least Privilege

Every IAM entity should have the **minimum permissions** needed to perform its function. Start with zero permissions and add only what's required.

## Key Practices

- Never use the root account for daily operations
- Enable **MFA** on all human accounts
- Use **IAM roles** for services, not access keys
- Rotate any long-lived credentials every 90 days
- Use **OIDC federation** for CI/CD (GitHub Actions, GitLab) instead of static keys

## Service-Linked Roles

For EKS workloads, use **IRSA** (IAM Roles for Service Accounts) to give pods specific IAM permissions:

```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: s3-reader
  annotations:
    eks.amazonaws.com/role-arn: arn:aws:iam::123456789:role/s3-reader
```

## Policy Structure

Write focused, specific policies. Avoid `*` in actions and resources. Use **conditions** to restrict further:

```json
{
  "Effect": "Allow",
  "Action": "s3:GetObject",
  "Resource": "arn:aws:s3:::my-bucket/data/*",
  "Condition": {
    "StringEquals": {
      "aws:PrincipalTag/team": "backend"
    }
  }
}
```

## Auditing

Use **IAM Access Analyzer** to find unused permissions and **CloudTrail** to audit all API calls. Review permissions quarterly.
