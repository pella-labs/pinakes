# PostgreSQL Fundamentals

## Overview

PostgreSQL is an advanced **open-source relational database** that has become the default choice for many modern applications. Its extensibility, standards compliance, and robust feature set make it suitable for everything from small web apps to enterprise data warehouses.

## Key Features

- **MVCC** (Multi-Version Concurrency Control) for high-concurrency workloads
- Full **ACID compliance** with configurable isolation levels
- Rich type system including JSON, arrays, hstore, and custom types
- Extensible via procedural languages (PL/pgSQL, PL/Python, PL/v8)
- Advanced indexing: B-tree, GIN, GiST, SP-GiST, BRIN, and hash

## Connection Management

Every PostgreSQL connection spawns a backend process, which is relatively heavyweight compared to thread-based databases. This makes **connection pooling** essential for production deployments. See [[db-045]] for PgBouncer configuration patterns.

```sql
-- Check current connections
SELECT count(*) FROM pg_stat_activity;

-- View connection state breakdown
SELECT state, count(*)
FROM pg_stat_activity
GROUP BY state;
```

## WAL and Durability

The **Write-Ahead Log** is PostgreSQL's crash recovery mechanism. Every change is written to WAL before being applied to data files. This is conceptually similar to how [[db-003]] describes SQLite's WAL mode, though the implementation differs significantly.

Understanding WAL is critical for tuning replication lag and checkpoint behavior in production systems.
