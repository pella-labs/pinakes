# Kubernetes Troubleshooting

## Common Issues and Debugging Steps

### Pod Stuck in Pending

The scheduler can't place the pod. Check:

```bash
kubectl describe pod <name> -n <namespace>
kubectl get events --field-selector reason=FailedScheduling
```

Causes: insufficient resources, node affinity mismatch, taint without toleration, PVC not bound.

### Pod in CrashLoopBackOff

The container starts and crashes repeatedly. Check:

```bash
kubectl logs <pod> -n <ns> --previous
kubectl describe pod <pod> -n <ns>
```

Causes: application error, missing config/secret, failed health check, OOMKilled.

### ImagePullBackOff

Can't pull the container image. Check:

```bash
kubectl describe pod <pod> -n <ns> | grep -A5 Events
```

Causes: wrong image tag, registry auth failure, image doesn't exist, network policy blocking registry access.

### Service Not Routing Traffic

```bash
kubectl get endpoints <service> -n <ns>
kubectl get pods -l <selector-labels> -n <ns>
```

Causes: label selector mismatch, readiness probe failing, network policy blocking traffic.

## General Debug Toolkit

```bash
kubectl get pods -n <ns> -o wide          # Pod status and node
kubectl top pods -n <ns>                   # Resource usage
kubectl exec -it <pod> -- sh               # Shell into pod
kubectl port-forward svc/<svc> 8080:80     # Local port forward
```

Related: [[ops-063]] for probes, [[ops-031]] for network policies.
