# AWS RDS Operations

## RDS Best Practices

Running **RDS** in production requires attention to performance, security, and cost.

## Instance Sizing

Start small and scale up. Use **Performance Insights** to identify bottlenecks:

- High CPU → upgrade instance class
- High I/O → switch to io2 storage or increase IOPS
- High connections → add PgBouncer
- Slow queries → optimize queries or add read replicas

## Parameter Groups

Tune database parameters through custom **parameter groups**:

```hcl
resource "aws_db_parameter_group" "postgres" {
  family = "postgres16"
  name   = "production-postgres16"

  parameter {
    name  = "shared_buffers"
    value = "{DBInstanceClassMemory/4}"
  }

  parameter {
    name  = "max_connections"
    value = "400"
  }

  parameter {
    name  = "log_min_duration_statement"
    value = "1000"
  }
}
```

## Multi-AZ

Always enable **Multi-AZ** for production databases. The standby instance provides automatic failover in 60-120 seconds if the primary fails.

## Maintenance

- Enable auto minor version upgrades
- Schedule maintenance windows during low traffic
- Test major version upgrades in staging first

See [[ops-035]] for database operations, [[ops-022]] for backup strategies.
