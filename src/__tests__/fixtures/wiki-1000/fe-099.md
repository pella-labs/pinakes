# Responsive Navigation Patterns

## Common Approaches

### Hamburger Menu

The standard mobile navigation pattern. A menu icon toggles a full-screen or sidebar navigation panel.

### Bottom Navigation Bar

Common in mobile-first apps. Fixed at the bottom with 3-5 primary actions. Reachable with one thumb.

### Collapsible Sidebar

Desktop sidebar that collapses to icons on smaller screens or via a toggle.

## Accessible Hamburger Implementation

```tsx
function MobileNav() {
  const [isOpen, setIsOpen] = useState(false);
  const navRef = useFocusTrap(isOpen);

  return (
    <>
      <button
        aria-expanded={isOpen}
        aria-controls="mobile-nav"
        onClick={() => setIsOpen(!isOpen)}
      >
        <span className="sr-only">{isOpen ? 'Close' : 'Open'} menu</span>
        <MenuIcon />
      </button>

      {isOpen && (
        <nav id="mobile-nav" ref={navRef} role="navigation">
          <NavLinks onNavigate={() => setIsOpen(false)} />
        </nav>
      )}
    </>
  );
}
```

## Responsive Breakpoint Strategy

- **Mobile** (<768px): hamburger or bottom nav
- **Tablet** (768-1024px): collapsible sidebar or compact top nav
- **Desktop** (>1024px): full sidebar or expanded top nav

See [[fe-015]] for responsive design strategies and [[fe-042]] for the focus trap hook.
