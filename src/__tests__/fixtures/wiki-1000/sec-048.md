# Envelope Encryption

**Envelope encryption** is a key management pattern where data is encrypted with a data key, and the data key is encrypted with a master key. This approach solves the practical challenges of encrypting large volumes of data.

## How It Works

1. Generate a unique **Data Encryption Key** (DEK) for each piece of data
2. Encrypt the data with the DEK using a symmetric algorithm (AES-256-GCM)
3. Encrypt the DEK with the **Key Encryption Key** (KEK) / master key
4. Store the encrypted data alongside the encrypted DEK
5. Discard the plaintext DEK from memory

## Benefits

- **Key rotation is cheap**: rotating the KEK only requires re-encrypting DEKs, not all data
- **Performance**: data encryption uses fast symmetric crypto
- **Isolation**: each piece of data has its own key, limiting blast radius
- **KMS integration**: the KEK lives in a hardware security module or cloud KMS

## Cloud KMS Integration

```typescript
// AWS KMS envelope encryption
const { Plaintext: dek, CiphertextBlob: encryptedDek } =
  await kms.generateDataKey({ KeyId: masterKeyId, KeySpec: 'AES_256' });

// Encrypt data with DEK
const encrypted = encrypt(data, dek);

// Store: { encryptedDek, encrypted }
// Discard: dek (plaintext)
```

## Key Hierarchy

Large systems use a three-tier hierarchy:

- **Root key**: stored in HSM, never leaves hardware
- **Key Encryption Key**: encrypted by root key, used to wrap DEKs
- **Data Encryption Key**: encrypted by KEK, used to encrypt actual data

See [[sec-016]] for rotation strategies and [[sec-025]] for encryption at rest patterns.
