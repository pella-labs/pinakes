# PostgreSQL Advisory Locks

## What Are Advisory Locks?

**Advisory locks** are application-controlled locks managed by PostgreSQL. Unlike row or table locks, they don't lock any actual database object — they're arbitrary locks identified by integer keys.

## Session-Level Locks

```sql
-- Acquire lock (blocks until available)
SELECT pg_advisory_lock(12345);

-- Try to acquire (non-blocking, returns true/false)
SELECT pg_try_advisory_lock(12345);

-- Release
SELECT pg_advisory_unlock(12345);
```

## Transaction-Level Locks

Released automatically at commit/rollback:

```sql
BEGIN;
SELECT pg_advisory_xact_lock(12345);
-- do work...
COMMIT;  -- lock released automatically
```

## Use Cases

- **Cron job deduplication**: prevent multiple instances from running the same job
- **Rate limiting**: limit concurrent access to a shared resource
- **Schema migration locking**: ensure only one migration runs at a time
- **Application-level mutexes**: coordinate between multiple app servers

```typescript
// Ensure only one worker processes a tenant at a time
const lockId = hashTenantId(tenantId);
const acquired = await pool.query('SELECT pg_try_advisory_lock($1)', [lockId]);
if (!acquired.rows[0].pg_try_advisory_lock) {
  return; // another worker has this tenant
}
```

For row-level locking patterns, see [[db-073]].
