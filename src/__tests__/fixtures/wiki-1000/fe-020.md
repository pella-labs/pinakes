# Data Fetching and Caching in Next.js

## Server-Side Fetching

In the App Router, data fetching happens in server components using standard `fetch` with extended caching options:

```typescript
// Cached indefinitely (like getStaticProps)
const data = await fetch('https://api.example.com/products', {
  cache: 'force-cache',
});

// Revalidated every 60 seconds
const data = await fetch('https://api.example.com/products', {
  next: { revalidate: 60 },
});

// Never cached (like getServerSideProps)
const data = await fetch('https://api.example.com/products', {
  cache: 'no-store',
});
```

## Request Deduplication

Next.js automatically deduplicates `fetch` calls with the same URL and options within a single render pass. Multiple components can fetch the same data without redundant network requests.

## Parallel Data Fetching

Avoid sequential waterfalls by fetching data in parallel:

```typescript
async function Page() {
  const [products, categories] = await Promise.all([
    getProducts(),
    getCategories(),
  ]);
  return <ProductGrid products={products} categories={categories} />;
}
```

## Client-Side Fetching

For data that changes frequently or is user-specific, fetch on the client using **SWR** or **TanStack Query**. These libraries provide caching, revalidation, and optimistic updates.

See [[fe-021]] for ISR and [[fe-011]] for server components context.
