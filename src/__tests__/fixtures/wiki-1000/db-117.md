# Connection Draining

## The Problem

During deployments, you need to shut down the old application version without dropping active database transactions. Connection draining ensures in-flight work completes before the process exits.

## Graceful Shutdown Pattern

```typescript
import { Pool } from 'pg';

const pool = new Pool({ max: 20 });

process.on('SIGTERM', async () => {
  console.log('Draining connections...');

  // Stop accepting new work
  server.close();

  // Wait for active queries to complete
  await pool.end();

  console.log('All connections closed');
  process.exit(0);
});
```

## Load Balancer Integration

1. Health check endpoint returns 503
2. Load balancer stops sending new requests
3. In-flight requests complete
4. Database connections drain
5. Process exits

## Timeout

Set a maximum drain time. If connections aren't released within the timeout, force-close:

```typescript
const DRAIN_TIMEOUT = 30000; // 30 seconds

setTimeout(() => {
  console.error('Drain timeout reached, force exiting');
  process.exit(1);
}, DRAIN_TIMEOUT);
```
