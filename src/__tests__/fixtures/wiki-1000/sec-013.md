# Secrets Management

Managing secrets — API keys, database credentials, encryption keys, certificates — is one of the most operationally challenging aspects of security. A single leaked secret can compromise an entire system.

## Principles

- **Never hardcode secrets** in source code
- **Never commit secrets** to version control
- **Rotate secrets** on a regular schedule
- **Audit access** to all secrets
- **Encrypt secrets** at rest and in transit
- **Scope secrets** to the minimum required access

## Storage Tiers

### Environment Variables

Simple but limited. Suitable for development and simple deployments. Use `.env` files (gitignored) with a `.env.example` template.

### Cloud Secret Managers

AWS Secrets Manager, GCP Secret Manager, Azure Key Vault. Provide versioning, rotation, and audit trails out of the box.

### HashiCorp Vault

The gold standard for self-hosted secret management. See [[sec-015]] for detailed Vault patterns.

## Secret Detection

Run **secret scanning** in CI to catch accidental commits:

- GitHub secret scanning (built-in)
- GitLeaks
- TruffleHog
- detect-secrets (Yelp)

These tools scan for patterns matching API keys, passwords, and tokens. Configure pre-commit hooks so secrets are caught before they enter history.
