# API Performance Profiling

Measure before optimizing.

## Server-Timing Header

```
Server-Timing: db;dur=12.5, auth;dur=3.2, serialize;dur=1.8, total;dur=22.1
```

Chrome DevTools displays these in the Timing tab. Add them in middleware:

```typescript
app.use((req, res, next) => {
  const timings: string[] = [];
  req.timing = (name: string, fn: () => any) => {
    const start = performance.now();
    const result = fn();
    timings.push(`${name};dur=${(performance.now() - start).toFixed(1)}`);
    return result;
  };

  res.on('finish', () => {
    res.setHeader('Server-Timing', timings.join(', '));
  });
  next();
});
```

## Key Metrics

- **P50/P95/P99 latency** per endpoint
- **Throughput** (requests/second)
- **Error rate** (5xx / total)
- **Database query count** per request (N+1 detection)
- **Payload size** (bytes out)

## Slow Query Detection

Log any request taking >500ms at `warn` level with a breakdown of where time was spent. This is cheaper than APM and catches most issues.
