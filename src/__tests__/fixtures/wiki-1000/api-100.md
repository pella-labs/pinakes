# DR: Kong as API Gateway

## Context

Need an API gateway for routing, auth, rate limiting, and observability. Evaluated: Kong, Envoy + custom control plane, AWS API Gateway, Tyk.

## Decision

Kong (open source) with declarative config managed via `deck`.

## Rationale

- **Declarative config**: YAML files in Git, applied via CI/CD
- **Plugin ecosystem**: auth, rate-limiting, CORS, logging all built-in
- **Performance**: <1ms added latency for simple routing
- **Language**: Lua plugins for custom logic, or Go for performance-critical plugins
- **Self-hosted**: no vendor lock-in, runs on our existing K8s

## Rejected Alternatives

- **Envoy**: more powerful but requires building a control plane (too much work for our team size)
- **AWS API Gateway**: vendor lock-in, can't run locally, harder to test
- **Tyk**: smaller community, fewer plugins

## Consequences

- Team needs to learn Kong plugin model
- Need to maintain Kong upgrade path
- All API routing centralized — single point of config review
