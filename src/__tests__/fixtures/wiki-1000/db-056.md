# Elasticsearch Index Management

## Index Lifecycle

Elasticsearch indices go through stages: **hot** (actively written to), **warm** (read-only, queried frequently), **cold** (rarely queried), and **frozen** (archived).

Index Lifecycle Management (ILM) automates transitions:

```json
{
  "policy": {
    "phases": {
      "hot": {
        "actions": {
          "rollover": { "max_size": "50gb", "max_age": "7d" }
        }
      },
      "warm": {
        "min_age": "30d",
        "actions": {
          "shrink": { "number_of_shards": 1 },
          "forcemerge": { "max_num_segments": 1 }
        }
      },
      "delete": {
        "min_age": "90d",
        "actions": { "delete": {} }
      }
    }
  }
}
```

## Index Templates

Templates define settings and mappings that are automatically applied to new indices matching a pattern. Essential for time-series data where new indices are created daily or weekly.

## Reindexing

When mappings change, you must create a new index and reindex data into it. Use aliases to swap indices with zero downtime.
