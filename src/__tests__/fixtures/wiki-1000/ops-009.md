# VPC and Network Design

## Network Architecture

A well-designed **VPC** (Virtual Private Cloud) is the foundation of cloud security. Every production workload should run in a properly segmented network.

## Subnet Layout

We use a three-tier subnet architecture per availability zone:

- **Public subnets**: load balancers, NAT gateways, bastion hosts
- **Private subnets**: application workloads (ECS, EKS, EC2)
- **Data subnets**: databases, caches, message brokers

```hcl
resource "aws_subnet" "private" {
  count             = length(var.azs)
  vpc_id            = aws_vpc.main.id
  cidr_block        = cidrsubnet(var.vpc_cidr, 4, count.index + 4)
  availability_zone = var.azs[count.index]

  tags = {
    Name = "private-${var.azs[count.index]}"
    Tier = "private"
  }
}
```

## CIDR Planning

Plan your CIDR blocks carefully. A `/16` VPC gives 65,536 addresses. Divide it into `/20` subnets for ~4,096 addresses each. Leave room for growth and VPC peering.

## Security Groups vs NACLs

**Security groups** are stateful and attached to ENIs. Use them as the primary firewall. **NACLs** are stateless and attached to subnets. Use them as a coarse backup layer, not the primary control.
