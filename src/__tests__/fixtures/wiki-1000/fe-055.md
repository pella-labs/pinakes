# URL State Management

## State in the URL

Certain UI state belongs in the URL: search queries, filters, pagination, selected tabs, modal open/close state. This ensures shareability, bookmarkability, and proper browser back/forward behavior.

## Using nuqs (Next.js)

The **nuqs** library provides type-safe URL search parameter hooks for Next.js:

```typescript
import { useQueryState, parseAsInteger } from 'nuqs';

function ProductList() {
  const [search, setSearch] = useQueryState('q', { defaultValue: '' });
  const [page, setPage] = useQueryState('page', parseAsInteger.withDefault(1));
  const [sort, setSort] = useQueryState('sort', { defaultValue: 'newest' });

  // URL: /products?q=shoes&page=2&sort=price
}
```

## Benefits

- Browser back/forward works correctly
- Users can share filtered views via URL
- Server components can read search params for SSR
- Deep linking into specific application states

## When State Does NOT Belong in the URL

- Ephemeral UI state (hover, focus, dropdown open)
- Form input state before submission
- Client-only preferences that shouldn't be shared
