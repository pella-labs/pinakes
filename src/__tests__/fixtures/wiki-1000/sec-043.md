# Security Event Monitoring

Effective **security monitoring** requires knowing what events to watch and how to correlate them into meaningful alerts. Raw logs are noise — structured alerting is signal.

## High-Priority Events

Monitor these events with real-time alerting:

- Multiple failed login attempts from a single IP or for a single account
- Successful login from a new country or unusual time
- Privilege escalation events
- Configuration changes to security controls
- API key creation or modification
- Access to sensitive data outside normal patterns
- Certificate or TLS errors
- WAF blocks exceeding baseline rate

## Correlation Rules

Individual events are rarely meaningful in isolation. Correlate:

- Failed login + successful login from different IP = possible credential stuffing
- New API key + bulk data access = possible data exfiltration
- Configuration change + privilege escalation = possible insider threat
- Multiple WAF blocks + probing patterns = reconnaissance phase

## SIEM Integration

Feed security events to a **SIEM** (Security Information and Event Management) system:

- Splunk, Elastic Security, or Datadog Security
- Normalize event formats across services
- Define detection rules with tunable thresholds
- Automate initial response actions

## Alert Fatigue

Too many alerts desensitize the team. Reduce noise by:

- Tuning thresholds based on baseline behavior
- Grouping related alerts into incidents
- Assigning severity levels with clear escalation paths
- Regularly reviewing and retiring noisy rules
