# GraphQL N+1 Problem

The most common performance issue in GraphQL.

## What Happens

```graphql
query {
  orders(first: 50) {    # 1 query
    customer {             # 50 queries! (one per order)
      name
    }
  }
}
```

The `customer` resolver runs once per order, each hitting the database.

## Fix: DataLoader

Batch the 50 individual customer queries into one `WHERE id IN (...)`. See [[api-039]] for full implementation.

## Fix: Join Monster

For SQL databases, generate a single JOIN query from the GraphQL AST:

```sql
SELECT orders.*, customers.name
FROM orders
LEFT JOIN customers ON orders.customer_id = customers.id
LIMIT 50;
```

## Detection

Monitor query count per request. If a single GraphQL query triggers >10 DB queries, it's likely an N+1. Log a warning:

```typescript
if (queryCount > 10) {
  logger.warn({ queryCount, operationName }, 'Possible N+1 detected');
}
```

## Prevention

Make DataLoader mandatory in all resolvers that load related entities. Code review should flag any `db.findOne()` call in a child resolver.
