# Zero Trust Architecture

The **zero trust** model eliminates implicit trust from network architecture. Every request is authenticated, authorized, and encrypted regardless of where it originates.

## Core Principles

- **Never trust, always verify**: no implicit trust based on network location
- **Least privilege access**: grant minimum necessary permissions
- **Assume breach**: design systems as if the perimeter is already compromised
- **Verify explicitly**: authenticate and authorize every request using all available signals

## Implementation Pillars

### Identity Verification
Strong authentication for all users and services. Use [[sec-006]] (mTLS) for service-to-service and [[sec-020]] (OIDC) for user authentication.

### Device Trust
Evaluate device posture before granting access:
- Is the device managed?
- Is the OS patched?
- Is disk encryption enabled?
- Is there an active threat agent?

### Network Micro-segmentation
Replace flat networks with fine-grained segments. Each workload communicates only with explicitly allowed peers.

### Continuous Verification
Don't just authenticate at session start — continuously re-evaluate trust based on behavior, context, and risk signals.

## Practical Steps

Zero trust is a journey, not a switch. Start with:

1. Inventory all assets and data flows
2. Implement strong identity for users and services
3. Enforce mTLS between services
4. Add device trust signals
5. Implement real-time access decisions
6. Monitor and log everything (see [[sec-014]])
