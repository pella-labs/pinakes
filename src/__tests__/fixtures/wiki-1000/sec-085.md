# OAuth2 Device Authorization Grant

The **device authorization grant** is designed for devices with limited input capabilities (smart TVs, CLI tools, IoT devices) that cannot easily display a web browser for traditional OAuth2 flows.

## Flow

1. Device requests a **device code** and **user code** from the authorization server
2. Device displays the user code and a verification URL to the user
3. User opens the URL on a separate device (phone, laptop) and enters the code
4. User authenticates and authorizes
5. Device polls the authorization server until the user completes authorization
6. Device receives tokens

## Implementation Concerns

```typescript
// Device code request
const response = await fetch('https://auth.example.com/device/code', {
  method: 'POST',
  body: new URLSearchParams({
    client_id: 'cli-tool',
    scope: 'read write',
  }),
});

const { device_code, user_code, verification_uri, interval } = await response.json();

console.log(`Visit ${verification_uri} and enter code: ${user_code}`);

// Poll until authorized
let tokenResponse;
while (!tokenResponse) {
  await sleep(interval * 1000);
  tokenResponse = await pollForToken(device_code);
}
```

## Security Considerations

- User codes should be short (8 chars) and unambiguous (no 0/O, 1/l)
- Device codes should be high-entropy and expire quickly (5-15 minutes)
- Polling interval must be enforced server-side to prevent brute-force
- Display the full verification URI so users can verify the domain

See [[sec-001]] for the standard authorization code flow.
