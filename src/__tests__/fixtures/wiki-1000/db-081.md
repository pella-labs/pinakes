# PostgreSQL Tablespaces

## What Are Tablespaces?

A **tablespace** maps a logical name to a physical directory on disk. You can place tables, indexes, and other objects on different storage devices.

```sql
CREATE TABLESPACE fast_storage LOCATION '/mnt/nvme/pg_data';
CREATE TABLESPACE archive_storage LOCATION '/mnt/hdd/pg_archive';

-- Place a table on fast storage
CREATE TABLE hot_data (...) TABLESPACE fast_storage;

-- Move an index to fast storage
ALTER INDEX idx_hot_data SET TABLESPACE fast_storage;
```

## Use Cases

- Put frequently accessed data on NVMe, archive data on HDD
- Separate temp tables onto dedicated disks
- Distribute I/O across multiple storage devices

## Monitoring

```sql
SELECT spcname, pg_tablespace_size(oid) AS size
FROM pg_tablespace;
```

Tablespaces are a PostgreSQL-specific feature. For general storage and capacity planning, see [[db-065]].
