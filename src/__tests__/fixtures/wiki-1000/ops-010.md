# Deployment Strategies

## Blue-Green Deployments

**Blue-green deployment** maintains two identical environments. Traffic switches from blue (current) to green (new) atomically. Rollback is instant — just switch back.

### Pros
- Zero-downtime deploys
- Instant rollback
- Full environment validation before switch

### Cons
- Doubles infrastructure cost during deploy
- Database migrations need careful handling
- Session state must be externalized

## Canary Releases

**Canary releases** route a small percentage of traffic to the new version. If metrics look good, gradually increase. If something breaks, route all traffic back to the old version.

```yaml
apiVersion: flagger.app/v1beta1
kind: Canary
metadata:
  name: my-app
spec:
  targetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  progressDeadlineSeconds: 300
  analysis:
    interval: 1m
    threshold: 5
    maxWeight: 50
    stepWeight: 10
    metrics:
      - name: request-success-rate
        thresholdRange:
          min: 99
```

## Rolling Updates

Kubernetes **rolling updates** replace pods incrementally. Simple but slower to roll back than blue-green. Set `maxSurge` and `maxUnavailable` appropriately.
