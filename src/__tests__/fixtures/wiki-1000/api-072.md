# Designing Search APIs

## Simple Search

```
GET /products?q=wireless+headphones&category=electronics&price_min=50&price_max=200
```

For simple use cases, query parameters are fine.

## Advanced Search with POST

When search criteria are complex, use POST with a body:

```json
{
  "query": "wireless headphones",
  "filters": {
    "category": ["electronics", "audio"],
    "price": { "min": 50, "max": 200 },
    "in_stock": true
  },
  "sort": [
    { "field": "relevance", "order": "desc" },
    { "field": "price", "order": "asc" }
  ],
  "pagination": { "cursor": null, "limit": 25 }
}
```

## Response

Include **facets** for filtering UIs:

```json
{
  "data": [...],
  "facets": {
    "category": [
      { "value": "electronics", "count": 342 },
      { "value": "audio", "count": 128 }
    ],
    "brand": [
      { "value": "Sony", "count": 89 },
      { "value": "Bose", "count": 67 }
    ]
  },
  "total": 1204,
  "pagination": { "next_cursor": "...", "has_more": true }
}
```

## Highlighting

Return matched snippets with highlights:

```json
{
  "name": "Sony <em>Wireless Headphones</em> WH-1000XM5",
  "description": "...noise-cancelling <em>wireless headphones</em> with..."
}
```
