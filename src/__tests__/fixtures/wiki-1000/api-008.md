# Request Validation Middleware

## Layered Validation

Validate at three layers:

1. **Transport** — content-type, content-length, encoding
2. **Schema** — JSON Schema or Zod validation against the request body
3. **Business** — domain rules (e.g., "order total can't exceed credit limit")

Never skip layer 1. A `Content-Type: text/xml` hitting a JSON endpoint should fail fast.

## Zod for Runtime Validation

```typescript
import { z } from 'zod';

const CreateOrderSchema = z.object({
  customerId: z.string().uuid(),
  items: z.array(z.object({
    productId: z.string().uuid(),
    quantity: z.number().int().positive().max(999),
  })).min(1).max(100),
  shippingAddress: z.object({
    line1: z.string().max(200),
    city: z.string().max(100),
    country: z.string().length(2),
  }),
});

type CreateOrderInput = z.infer<typeof CreateOrderSchema>;
```

## Error Response Format

Return structured validation errors, not strings:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Request validation failed",
    "details": [
      { "field": "items[0].quantity", "issue": "Must be positive" },
      { "field": "shippingAddress.country", "issue": "Must be 2 characters" }
    ]
  }
}
```
