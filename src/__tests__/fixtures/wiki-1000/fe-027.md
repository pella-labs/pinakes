# Theming Strategies

## CSS Custom Properties

The simplest theming approach uses **CSS custom properties** (variables) toggled by a class or attribute on the root element:

```css
:root {
  --color-bg: #ffffff;
  --color-text: #111827;
  --color-primary: #3b82f6;
}

[data-theme="dark"] {
  --color-bg: #111827;
  --color-text: #f9fafb;
  --color-primary: #60a5fa;
}
```

All components reference these variables. Switching themes changes only the root-level values.

## System Preference Detection

Respect the user's OS preference with `prefers-color-scheme`:

```css
@media (prefers-color-scheme: dark) {
  :root:not([data-theme]) {
    --color-bg: #111827;
    --color-text: #f9fafb;
  }
}
```

## Theme Provider Pattern

In React applications, a theme provider manages the current theme and persists the user's choice:

```tsx
function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState(() => {
    return localStorage.getItem('theme') ?? 'system';
  });

  useEffect(() => {
    document.documentElement.setAttribute('data-theme', resolvedTheme);
    localStorage.setItem('theme', theme);
  }, [theme]);

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}
```

## Token Integration

Theming works best when integrated with design tokens. Each theme provides a different set of alias token values while keeping the same token names. See [[fe-023]] for token architecture.
