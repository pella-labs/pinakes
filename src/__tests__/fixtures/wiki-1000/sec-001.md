# OAuth2 Authorization Code Flow

The **authorization code flow** is the most secure OAuth2 grant type for server-side applications. It involves a two-step process where the client first obtains an authorization code, then exchanges it for tokens.

## Flow Overview

1. The client redirects the user to the **authorization server**
2. User authenticates and consents
3. Authorization server redirects back with a `code` parameter
4. Client exchanges the code for an **access token** and **refresh token**

## Implementation Notes

The authorization code should be short-lived (typically 10 minutes) and single-use. Always use **PKCE** (Proof Key for Code Exchange) even for confidential clients, as recommended by OAuth 2.1.

```typescript
const authUrl = new URL('/authorize', AUTH_SERVER);
authUrl.searchParams.set('response_type', 'code');
authUrl.searchParams.set('client_id', config.clientId);
authUrl.searchParams.set('redirect_uri', config.redirectUri);
authUrl.searchParams.set('code_challenge', codeChallenge);
authUrl.searchParams.set('code_challenge_method', 'S256');
```

## Token Storage

Store **refresh tokens** encrypted at rest. Never expose them to the browser. Access tokens should be kept in memory when possible, not in localStorage or cookies without proper protections. See [[sec-015]] for secrets management patterns.
