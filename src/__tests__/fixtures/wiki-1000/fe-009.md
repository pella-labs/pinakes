# Web Vitals and Core Performance Metrics

## What Are Core Web Vitals

**Core Web Vitals** are Google's standardized metrics for measuring real-world user experience. They focus on loading, interactivity, and visual stability.

## The Three Metrics

### Largest Contentful Paint (LCP)

Measures loading performance. LCP should occur within **2.5 seconds** of page load. The largest content element (image, video, or text block) determines this metric.

Optimization strategies:
- Optimize server response times
- Preload critical resources with `<link rel="preload">`
- Use responsive images with `srcset`
- Inline critical CSS

### Interaction to Next Paint (INP)

Replaces First Input Delay (FID) as of March 2024. INP measures responsiveness across all interactions, not just the first one. Target: **under 200ms**.

- Break long tasks with `requestIdleCallback` or `scheduler.yield()`
- Avoid synchronous layout thrashing
- Use web workers for heavy computation
- Debounce rapid user interactions

### Cumulative Layout Shift (CLS)

Measures visual stability. Target: **under 0.1**. Layout shifts occur when visible elements move unexpectedly.

- Always set explicit `width` and `height` on images and videos
- Reserve space for dynamic content (ads, embeds)
- Use CSS `contain` to limit layout recalculations
- Avoid inserting content above existing content

## Measuring in Production

Use the `web-vitals` library to capture real user metrics:

```typescript
import { onLCP, onINP, onCLS } from 'web-vitals';

onLCP(sendToAnalytics);
onINP(sendToAnalytics);
onCLS(sendToAnalytics);
```
