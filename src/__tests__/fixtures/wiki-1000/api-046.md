# GraphQL Query Complexity Analysis

## The Problem

A single GraphQL query can request deeply nested data, triggering thousands of database queries. Without limits, a malicious query can DoS the server.

## Calculating Complexity

Assign a cost to each field:

```typescript
const complexityRules = {
  Query: {
    orders: { complexity: 5, multiplier: 'first' }, // 5 * page size
  },
  Order: {
    items: { complexity: 2, multiplier: 'first' },
    customer: { complexity: 1 },
  },
};
```

A query for 50 orders with 10 items each costs `5*50 + 2*10*50 = 1250`.

## Enforcing Limits

Reject queries exceeding the complexity budget before execution:

```typescript
const MAX_COMPLEXITY = 5000;

const plugin = {
  requestDidStart: () => ({
    didResolveOperation({ document }) {
      const cost = calculateComplexity(document, complexityRules);
      if (cost > MAX_COMPLEXITY) {
        throw new GraphQLError(
          `Query complexity ${cost} exceeds maximum ${MAX_COMPLEXITY}`
        );
      }
    },
  }),
};
```

## Depth Limiting

Simpler alternative: limit query depth to 7 levels. Catches most abuse with zero configuration.
