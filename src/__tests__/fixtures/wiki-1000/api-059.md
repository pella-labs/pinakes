# API Documentation with Examples

Documentation without examples is useless.

## Every Endpoint Needs

1. **Description** — what it does, when to use it
2. **Request example** — full curl command or HTTP snippet
3. **Success response** — actual JSON with realistic data
4. **Error responses** — at least the top 3 error cases
5. **Rate limits** — applicable limits for this endpoint

## Example Template

```markdown
## Create Order

Creates a new order for the specified customer.

### Request

POST /v1/orders
Authorization: Bearer sk_live_xxx
Content-Type: application/json

{
  "customer_id": "cust_abc",
  "items": [{"product_id": "prod_123", "quantity": 2}],
  "idempotency_key": "idk_unique_abc"
}

### Response (201)

{
  "id": "ord_xyz",
  "status": "pending",
  "total": 4999,
  "created_at": "2025-11-03T14:00:00Z"
}

### Errors

- 400: Missing required fields
- 409: Duplicate idempotency key
- 422: Product out of stock
```

## Interactive Documentation

Use Swagger UI or Redoc fed from the OpenAPI spec ([[api-006]]). But always supplement with hand-written guides for complex workflows.
