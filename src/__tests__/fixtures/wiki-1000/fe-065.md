# Content Security Policy for SPAs

## What Is CSP

A **Content Security Policy** is an HTTP header that tells the browser which sources of content are allowed to load. It mitigates XSS, data injection, and clickjacking attacks.

## Basic Policy

```
Content-Security-Policy:
  default-src 'self';
  script-src 'self' 'nonce-abc123';
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: https://cdn.example.com;
  connect-src 'self' https://api.example.com;
  frame-ancestors 'none';
```

## Nonce-Based Script Loading

Instead of `'unsafe-inline'` for scripts, generate a unique nonce per request:

```html
<script nonce="abc123" src="/app.js"></script>
```

The CSP header includes `'nonce-abc123'`, and only scripts with matching nonces execute.

## SPA Challenges

- CSS-in-JS libraries often require `'unsafe-inline'` for styles
- Inline event handlers (`onclick`) are blocked by default
- Third-party scripts (analytics, chat widgets) need explicit source allowlisting

## Report-Only Mode

Deploy new policies in report-only mode first:

```
Content-Security-Policy-Report-Only: ...
```

This reports violations without blocking content, letting you tune the policy before enforcement.
