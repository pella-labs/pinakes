# REST Partial Updates

## PATCH with JSON Merge Patch

Simple, intuitive. Send only the fields you want to change:

```http
PATCH /users/u_123
Content-Type: application/merge-patch+json

{
  "name": "Alice Updated",
  "address": { "city": "Portland" }
}
```

**Limitation:** you can't set a field to `null` (it means "remove"), and you can't manipulate arrays.

## PATCH with JSON Patch (RFC 6902)

More powerful. An ordered list of operations:

```json
[
  { "op": "replace", "path": "/name", "value": "Alice Updated" },
  { "op": "add", "path": "/tags/-", "value": "premium" },
  { "op": "remove", "path": "/metadata/deprecated_field" },
  { "op": "test", "path": "/version", "value": 3 }
]
```

The `test` operation enables optimistic locking — if the version doesn't match, the whole patch fails.

## Which to Choose

- JSON Merge Patch for simple updates (most CRUD apps)
- JSON Patch when you need array manipulation or atomic test-and-set
- Custom PATCH body when neither standard fits (but document it well)
