# Upsert Patterns

## What Is an Upsert?

An **upsert** (INSERT or UPDATE) inserts a row if it doesn't exist, or updates it if it does. This is a fundamental operation for idempotent data ingestion.

## PostgreSQL: ON CONFLICT

```sql
INSERT INTO nodes (id, title, content, updated_at)
VALUES ($1, $2, $3, NOW())
ON CONFLICT (id) DO UPDATE SET
  title = EXCLUDED.title,
  content = EXCLUDED.content,
  updated_at = NOW();
```

The `EXCLUDED` pseudo-table refers to the row that was proposed for insertion.

## MySQL: ON DUPLICATE KEY UPDATE

```sql
INSERT INTO nodes (id, title, content)
VALUES (?, ?, ?)
ON DUPLICATE KEY UPDATE
  title = VALUES(title),
  content = VALUES(content);
```

## SQLite: ON CONFLICT

```sql
INSERT INTO nodes (id, title, content)
VALUES (?, ?, ?)
ON CONFLICT(id) DO UPDATE SET
  title = excluded.title,
  content = excluded.content;
```

## Bulk Upserts

For ingesting many rows, combine upserts with batch inserts:

```typescript
const upsert = db.prepare(`
  INSERT INTO chunks (id, node_id, content, chunk_sha)
  VALUES (?, ?, ?, ?)
  ON CONFLICT(id) DO UPDATE SET
    content = excluded.content,
    chunk_sha = excluded.chunk_sha
`);

const bulkUpsert = db.transaction((chunks: Chunk[]) => {
  for (const chunk of chunks) {
    upsert.run(chunk.id, chunk.nodeId, chunk.content, chunk.sha);
  }
});
```

This is essential for idempotent ingestion pipelines. See [[db-042]] for better-sqlite3 transaction patterns.
