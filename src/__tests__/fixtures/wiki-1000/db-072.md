# SQLite Limitations and Workarounds

## Single Writer

SQLite allows only one writer at a time. With WAL mode (see [[db-003]]), readers don't block writers, but concurrent write attempts get SQLITE_BUSY errors.

**Workaround**: Use a single writer connection with `busy_timeout` set to wait for the lock.

## No Network Access

SQLite databases are local files. You can't connect to a remote SQLite database.

**Workaround**: Use LiteFS or Turso for distributed SQLite. Or accept that SQLite is for local-first applications.

## No ALTER TABLE DROP COLUMN (before 3.35)

Older SQLite versions couldn't drop columns.

**Workaround**: Create a new table without the column, copy data, drop the old table, rename. SQLite 3.35+ supports `ALTER TABLE DROP COLUMN` natively.

## 16-Bit Row Limit for Compound SELECT

A UNION ALL of more than 500 compound SELECT terms hits a compile-time limit.

**Workaround**: Break into multiple queries or use INSERT ... SELECT in a loop.

## No RIGHT JOIN or FULL OUTER JOIN (before 3.39)

Added in SQLite 3.39.0. If you're on an older version, rewrite as LEFT JOIN with table order swapped.

## No GRANT/REVOKE

SQLite has no user permissions. Access control happens at the filesystem level.

**Workaround**: This is by design. SQLite is an application database, not a multi-user server.
