# Kubernetes Upgrades

## Upgrade Strategy

Kubernetes releases a new minor version every ~4 months. Each version is supported for 14 months. Stay within **N-2** of the latest version.

## Pre-Upgrade Checklist

- Review the release notes and changelog for breaking changes
- Check that all add-ons and operators support the target version
- Test the upgrade in a non-production cluster first
- Verify deprecated APIs are not in use: `kubectl api-resources --api-group=...`

## EKS Upgrade Process

```bash
# 1. Update the control plane
aws eks update-cluster-version \
  --name production \
  --kubernetes-version 1.30

# 2. Wait for control plane update to complete
aws eks describe-update --name production --update-id <id>

# 3. Update node groups
aws eks update-nodegroup-version \
  --cluster-name production \
  --nodegroup-name general

# 4. Update add-ons
aws eks update-addon --cluster-name production \
  --addon-name vpc-cni --addon-version v1.16.0
```

## Rollback

EKS does not support downgrading the control plane. If the upgrade fails, restore from a cluster backup or rebuild. This is why testing in staging is critical.

## API Deprecation

Use tools like **pluto** or **kubent** to detect deprecated APIs in your manifests before upgrading.
