# Kubernetes Jobs and CronJobs

## Jobs

A **Job** runs a pod to completion. Use it for batch processing, data migrations, and one-off tasks.

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: data-migration
spec:
  backoffLimit: 3
  activeDeadlineSeconds: 3600
  template:
    spec:
      restartPolicy: Never
      containers:
        - name: migrate
          image: registry.example.com/migrator:v1.2.3
          env:
            - name: DB_URL
              valueFrom:
                secretKeyRef:
                  name: db-credentials
                  key: url
```

## CronJobs

**CronJobs** schedule recurring jobs:

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: nightly-report
spec:
  schedule: "0 2 * * *"
  concurrencyPolicy: Forbid
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 5
  jobTemplate:
    spec:
      template:
        spec:
          restartPolicy: OnFailure
          containers:
            - name: reporter
              image: registry.example.com/reporter:latest
```

## Best Practices

- Set `activeDeadlineSeconds` to prevent runaway jobs
- Use `concurrencyPolicy: Forbid` for CronJobs that shouldn't overlap
- Monitor job completion and failure rates
- Clean up completed jobs with history limits
