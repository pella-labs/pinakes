# Security Champions Program

A **security champions** program embeds security advocates within development teams. It scales security beyond the security team by creating distributed expertise.

## Role Definition

A security champion is a developer who:

- Has interest in security (not necessarily expertise)
- Receives additional security training
- Acts as the security point of contact for their team
- Reviews PRs with a security lens
- Triages security findings for their team's code
- Escalates complex issues to the security team

## Program Structure

### Selection
Look for volunteers, not conscripts. Interest matters more than current knowledge.

### Training
Provide structured training:
- Secure coding practices (see [[sec-050]])
- Threat modeling workshops (see [[sec-030]])
- Tool-specific training (SAST, DAST)
- Incident response participation (see [[sec-054]])

### Time Commitment
Typically 10-20% of the champion's time. This must be supported by management and reflected in goals.

### Recognition
Recognize contributions through:
- Dedicated Slack channel for security champions
- Monthly meetups to share learnings
- Security champion badge/title
- Career development support

## Measuring Success

Track:
- Number of security issues found in code review
- Time to remediate vulnerabilities in champion-owned code
- Security training completion rates
- Champion retention rate
