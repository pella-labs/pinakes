# Rendering Lists Efficiently

## The Key Prop

React uses the `key` prop to track which items changed, were added, or were removed. A good key is a **stable, unique identifier** from your data.

```tsx
// Good: stable id from data
{items.map((item) => <ListItem key={item.id} {...item} />)}

// Bad: index as key (breaks on reorder/insert/delete)
{items.map((item, index) => <ListItem key={index} {...item} />)}
```

## When Index Keys Are Acceptable

Using index as a key is fine when:
- The list is static (never reordered, filtered, or modified)
- Items have no stable unique ids
- The list is not used with animations

## Avoiding Re-Renders

When the parent re-renders, every list item re-renders by default. Strategies to prevent this:

- **React.memo** on the list item component
- **Stable callbacks** (useCallback) passed as props
- **Immutable updates** so shallow comparison catches unchanged items

## Chunked Rendering

For very long lists that don't need virtualization, render in chunks using `requestIdleCallback`:

```typescript
function useChunkedRender<T>(items: T[], chunkSize = 50) {
  const [rendered, setRendered] = useState(chunkSize);

  useEffect(() => {
    if (rendered >= items.length) return;
    const id = requestIdleCallback(() => {
      setRendered((r) => Math.min(r + chunkSize, items.length));
    });
    return () => cancelIdleCallback(id);
  }, [rendered, items.length, chunkSize]);

  return items.slice(0, rendered);
}
```
