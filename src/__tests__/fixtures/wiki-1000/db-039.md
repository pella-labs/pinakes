# Database Security Basics

## Authentication

- Use **strong passwords** or certificate-based auth
- PostgreSQL: configure `pg_hba.conf` to restrict access by IP, user, and database
- MySQL: use `caching_sha2_password` (default in 8.0+)
- Never expose databases directly to the internet

## Authorization

Principle of least privilege. Application users should NOT be database superusers.

```sql
-- Create a restricted application role
CREATE ROLE app_user WITH LOGIN PASSWORD 'strong_password';
GRANT CONNECT ON DATABASE myapp TO app_user;
GRANT USAGE ON SCHEMA public TO app_user;
GRANT SELECT, INSERT, UPDATE ON ALL TABLES IN SCHEMA public TO app_user;
-- No DELETE, no DDL, no TRUNCATE
```

## SQL Injection

The number one database vulnerability. Always use **parameterized queries**:

```typescript
// WRONG — vulnerable to SQL injection
const query = `SELECT * FROM users WHERE email = '${email}'`;

// RIGHT — parameterized
const query = 'SELECT * FROM users WHERE email = $1';
const result = await pool.query(query, [email]);
```

## Encryption

- **In transit**: use TLS for all database connections
- **At rest**: enable transparent data encryption (TDE) or full-disk encryption
- **Column-level**: encrypt sensitive fields (PII, credentials) at the application layer

## Auditing

Track who did what and when. PostgreSQL's `pgaudit` extension logs all SQL statements with full context. See [[db-039]] for setting up comprehensive audit logging.
