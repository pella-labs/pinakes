# PostgreSQL Foreign Data Wrappers

## What Are FDWs?

**Foreign Data Wrappers** let PostgreSQL query external data sources as if they were local tables. You can join local tables with remote PostgreSQL, MySQL, Redis, S3 files, and more.

## postgres_fdw

Query a remote PostgreSQL database:

```sql
CREATE EXTENSION postgres_fdw;

CREATE SERVER remote_server
FOREIGN DATA WRAPPER postgres_fdw
OPTIONS (host 'remote.example.com', dbname 'analytics', port '5432');

CREATE USER MAPPING FOR current_user
SERVER remote_server
OPTIONS (user 'readonly', password 'secret');

CREATE FOREIGN TABLE remote_orders (
  id INT,
  customer_id INT,
  total DECIMAL,
  created_at TIMESTAMPTZ
) SERVER remote_server OPTIONS (table_name 'orders');

-- Query as if it were local
SELECT * FROM remote_orders WHERE created_at > '2024-01-01';
```

## Use Cases

- Cross-database queries without ETL
- Gradual database migrations
- Accessing legacy systems
- Data federation for analytics

## Performance Caveats

FDW queries push predicates to the remote server when possible, but complex operations may pull entire tables across the network. Use EXPLAIN to verify what's pushed down.

For moving data between systems more efficiently, see CDC ([[db-026]]) or ETL ([[db-025]]).
