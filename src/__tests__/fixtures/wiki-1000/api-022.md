# Request Lifecycle in Express

A detailed trace of what happens from incoming socket to response.

## Phase 1: TCP + Parsing

Node's `http` module accepts the TCP connection, parses the HTTP request line and headers. The body is NOT parsed yet — it's a readable stream.

## Phase 2: Middleware Stack

Express runs middleware in registration order:

1. `express.json()` — reads the body stream, parses JSON, attaches to `req.body`
2. `cors()` — checks origin, adds headers
3. Custom auth middleware — verifies JWT, attaches `req.user`
4. Route handler — matched by method + path

## Phase 3: Route Handler

The handler receives `(req, res, next)`. It can:
- Send a response: `res.json(data)`
- Pass to next middleware: `next()`
- Pass an error: `next(new AppError(...))`

## Phase 4: Error Handling

If `next(error)` is called, Express skips to the error-handling middleware (4 args). This is where you format the error response.

## Phase 5: Response

`res.json()` serializes to JSON, sets `Content-Type`, and calls `res.end()`. Node's HTTP module writes to the socket.

The entire lifecycle should complete in <100ms for simple CRUD. If it takes longer, add a `Server-Timing` header. See [[api-011]] for middleware ordering.
