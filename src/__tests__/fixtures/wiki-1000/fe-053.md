# Migrating from Webpack to Vite

## Why Migrate

**Vite** offers dramatically faster development server startup (100-1000x for large projects) by leveraging native ES modules and esbuild for dependency pre-bundling.

## Key Differences

| Feature | Webpack | Vite |
|---------|---------|------|
| Dev server | Bundle then serve | Serve unbundled ESM |
| HMR | Full module graph | Precise module boundary |
| Config | webpack.config.js | vite.config.ts |
| Production | webpack + terser | Rollup + esbuild |

## Migration Steps

1. Replace `webpack.config.js` with `vite.config.ts`
2. Convert `require()` calls to ES imports
3. Replace webpack-specific env vars (`process.env.NODE_ENV`) with `import.meta.env`
4. Move `public/` static assets (Vite serves from `public/` root)
5. Update path aliases from webpack `resolve.alias` to Vite `resolve.alias`
6. Replace webpack loaders with Vite plugins

## Common Gotchas

- CommonJS dependencies may need explicit `optimizeDeps.include`
- CSS modules syntax differences
- `global` is not defined in ESM (use `globalThis`)
- Some webpack plugins have no Vite equivalent

## Incremental Approach

For large codebases, consider running Vite for development while keeping webpack for production builds initially. Migrate production last, after validating the development experience.
