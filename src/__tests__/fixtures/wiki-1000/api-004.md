# Rate Limiting Patterns

## Token Bucket Algorithm

The **token bucket** is the standard for API rate limiting. Each client gets a bucket that refills at a fixed rate. Each request consumes one token. When the bucket is empty, requests are rejected with `429 Too Many Requests`.

```typescript
interface RateLimitConfig {
  maxTokens: number;      // bucket capacity
  refillRate: number;     // tokens per second
  refillInterval: number; // ms between refills
}

class TokenBucket {
  private tokens: number;
  private lastRefill: number;

  constructor(private config: RateLimitConfig) {
    this.tokens = config.maxTokens;
    this.lastRefill = Date.now();
  }

  tryConsume(): boolean {
    this.refill();
    if (this.tokens > 0) {
      this.tokens--;
      return true;
    }
    return false;
  }

  private refill(): void {
    const now = Date.now();
    const elapsed = now - this.lastRefill;
    const newTokens = Math.floor(elapsed / this.config.refillInterval) * this.config.refillRate;
    this.tokens = Math.min(this.tokens + newTokens, this.config.maxTokens);
    this.lastRefill = now;
  }
}
```

## Response Headers

Always include rate limit info in response headers:

- `X-RateLimit-Limit` — max requests in window
- `X-RateLimit-Remaining` — requests left
- `X-RateLimit-Reset` — UTC epoch seconds when bucket refills
- `Retry-After` — seconds to wait (on 429 only)

## Tiered Limits

Different tiers for different auth levels. Anonymous gets 60/hour, authenticated gets 1000/hour, premium gets 10000/hour. See [[api-023]] for authentication middleware setup.
