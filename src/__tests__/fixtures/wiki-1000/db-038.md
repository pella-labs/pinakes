# Redis Pub/Sub and Streams

## Pub/Sub

Redis Pub/Sub is a fire-and-forget messaging system. Publishers send messages to channels; subscribers receive them in real-time. Messages are not persisted — if a subscriber is offline, it misses the message.

```
SUBSCRIBE notifications
PUBLISH notifications "New order #1234"
```

Use cases: real-time notifications, chat, live dashboards.

## Streams

**Redis Streams** (5.0+) are a durable, append-only log with consumer groups. Unlike Pub/Sub, messages persist and consumers can replay from any point.

```
-- Producer
XADD orders * customer_id 42 total 99.99

-- Consumer group
XGROUP CREATE orders processing $ MKSTREAM
XREADGROUP GROUP processing worker-1 COUNT 10 BLOCK 5000 STREAMS orders >

-- Acknowledge processed messages
XACK orders processing 1706745600000-0
```

## Streams vs Kafka

For most applications, Redis Streams are sufficient. Choose Kafka when you need:
- Multi-datacenter replication
- Months of retention
- Millions of messages per second
- Complex stream processing (Kafka Streams, ksqlDB)
