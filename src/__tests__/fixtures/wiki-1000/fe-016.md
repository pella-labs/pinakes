# Code Splitting and Lazy Loading

## Why Split Code

A single monolithic JavaScript bundle delays initial page load. **Code splitting** divides your application into smaller chunks that load on demand, reducing the initial payload.

## React.lazy and Suspense

```tsx
import { lazy, Suspense } from 'react';

const AdminPanel = lazy(() => import('./AdminPanel'));

function App() {
  return (
    <Suspense fallback={<Spinner />}>
      {isAdmin && <AdminPanel />}
    </Suspense>
  );
}
```

The `AdminPanel` module is fetched only when `isAdmin` is true. The `Suspense` boundary displays a fallback while loading.

## Route-Based Splitting

The most impactful splitting strategy is at the route level. Each page becomes its own chunk, loaded when the user navigates to it.

## Component-Level Splitting

For heavy components that are not always visible (modals, charts, rich text editors), lazy-load them individually. This is especially useful for below-the-fold content.

## Prefetching

Combine lazy loading with **prefetching** to eliminate perceived latency. When the user hovers over a navigation link, start loading the target route's chunk.

```tsx
function NavLink({ to, children }) {
  const prefetch = () => {
    const route = routeMap[to];
    if (route?.preload) route.preload();
  };

  return (
    <a href={to} onMouseEnter={prefetch}>
      {children}
    </a>
  );
}
```
