# GraphQL Subscriptions

## Transport

Subscriptions use WebSockets (graphql-ws protocol) for real-time push.

```graphql
type Subscription {
  orderStatusChanged(orderId: ID!): OrderStatusEvent!
  newNotification: Notification!
}

type OrderStatusEvent {
  orderId: ID!
  previousStatus: OrderStatus!
  newStatus: OrderStatus!
  changedAt: DateTime!
}
```

## Server Setup (Apollo)

```typescript
import { createServer } from 'http';
import { WebSocketServer } from 'ws';
import { useServer } from 'graphql-ws/lib/use/ws';

const httpServer = createServer(app);
const wsServer = new WebSocketServer({ server: httpServer, path: '/graphql' });

useServer({
  schema,
  context: async (ctx) => {
    const token = ctx.connectionParams?.authorization;
    return { user: await verifyToken(token) };
  },
}, wsServer);
```

## Scaling

Each server instance only sees its local subscribers. Use Redis Pub/Sub to broadcast events across all instances. When a mutation fires on server A, it publishes to Redis, and server B (which has the subscriber's WebSocket) receives and pushes the event.

See [[api-021]] for general WebSocket design and [[api-020]] for the SSE alternative.
