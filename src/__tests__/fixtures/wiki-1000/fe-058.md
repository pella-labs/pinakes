# Tailwind CSS v4 Changes

## CSS-First Configuration

Tailwind v4 moves configuration from `tailwind.config.js` to CSS using `@theme`:

```css
@import "tailwindcss";

@theme {
  --color-primary: #3b82f6;
  --color-secondary: #6b7280;
  --font-display: "Inter", sans-serif;
  --breakpoint-lg: 1024px;
}
```

## New Engine

The v4 engine is written in Rust (via **Oxide**), providing 10x faster builds. It automatically detects content files without explicit `content` configuration.

## Breaking Changes

- `@apply` with arbitrary values has new syntax
- Some modifier syntax changed
- Prefix support removed
- `safelist` moved to CSS

## Upgrade Path

Use the official codemod:

```bash
npx @tailwindcss/upgrade
```

This handles most automatic transformations. Manual review is needed for:
- Custom plugin compatibility
- Complex `@apply` usage
- JavaScript-based config that must move to CSS

See [[fe-007]] for Tailwind best practices.
