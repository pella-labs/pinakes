# AWS ECS Fargate

## Serverless Containers

**Fargate** runs containers without managing servers. You define the task (container spec) and Fargate handles provisioning, scaling, and patching the underlying infrastructure.

## Task Definition

```json
{
  "family": "api-server",
  "networkMode": "awsvpc",
  "requiresCompatibilities": ["FARGATE"],
  "cpu": "512",
  "memory": "1024",
  "containerDefinitions": [{
    "name": "api",
    "image": "123456789.dkr.ecr.us-east-1.amazonaws.com/api:latest",
    "portMappings": [{ "containerPort": 8080 }],
    "logConfiguration": {
      "logDriver": "awslogs",
      "options": {
        "awslogs-group": "/ecs/api-server",
        "awslogs-region": "us-east-1",
        "awslogs-stream-prefix": "ecs"
      }
    }
  }]
}
```

## When to Use Fargate vs EKS

Choose **Fargate** when:
- Small team, simple services
- Want zero cluster management
- Running fewer than 10 services

Choose **EKS** when:
- Need the Kubernetes ecosystem
- Running many services with complex networking
- Need GPU or specialized instance types
- Team has Kubernetes expertise

## Cost Comparison

Fargate costs more per compute unit than EC2 but eliminates node management overhead. For bursty workloads that scale to zero, Fargate often wins on total cost.
