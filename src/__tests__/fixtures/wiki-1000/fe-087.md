# Astro and Islands Architecture

## What Is Astro

**Astro** is a web framework designed for content-rich websites. It renders pages to static HTML by default and hydrates interactive components on demand—the **islands architecture**.

## How Islands Work

Each interactive widget is an isolated island of JavaScript. The rest of the page is pure HTML with zero client-side JavaScript.

```astro
---
// Server-side: runs at build time or request time
import Header from '../components/Header.astro';
import ProductCarousel from '../components/ProductCarousel.tsx';
---

<Header />
<!-- Static HTML, no JS -->
<article>
  <h1>Product Details</h1>
  <p>This content is pure HTML.</p>
</article>

<!-- Island: hydrates on the client -->
<ProductCarousel client:visible products={products} />
```

## Client Directives

- `client:load` — hydrate immediately on page load
- `client:idle` — hydrate when the browser is idle
- `client:visible` — hydrate when the element enters the viewport
- `client:media` — hydrate when a CSS media query matches
- `client:only` — skip SSR, render only on the client

## Framework Agnostic

Islands can be built with React, Vue, Svelte, Solid, or Preact. Multiple frameworks can coexist on the same page—each island is independent.

## When to Use Astro

Content-first websites: blogs, documentation, marketing sites, e-commerce product pages. For highly interactive applications (dashboards, editors), a full SPA framework is more appropriate.
