# Database Constraints

## Types of Constraints

### NOT NULL
Column must have a value. The simplest and most important constraint.

### UNIQUE
No two rows can have the same value. Creates an index automatically.

### PRIMARY KEY
NOT NULL + UNIQUE. One per table.

### FOREIGN KEY
References a row in another table. Enforces referential integrity.

### CHECK
Custom validation expression:

```sql
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL,
  price DECIMAL NOT NULL CHECK (price > 0),
  discount DECIMAL DEFAULT 0 CHECK (discount >= 0 AND discount <= price),
  status TEXT CHECK (status IN ('draft', 'active', 'archived'))
);
```

### EXCLUSION
Prevents overlapping values. PostgreSQL-specific:

```sql
CREATE TABLE reservations (
  room_id INT,
  during TSTZRANGE,
  EXCLUDE USING gist (room_id WITH =, during WITH &&)
);
```

## Deferrable Constraints

Some constraints can be deferred to commit time:

```sql
ALTER TABLE orders ADD CONSTRAINT fk_customer
FOREIGN KEY (customer_id) REFERENCES customers (id)
DEFERRABLE INITIALLY DEFERRED;
```

This allows inserting the order before the customer within the same transaction. Useful for circular dependencies.

## Constraint Naming

Always name your constraints. Unnamed constraints get auto-generated names that are hard to debug:

```sql
ALTER TABLE users ADD CONSTRAINT users_email_unique UNIQUE (email);
```
