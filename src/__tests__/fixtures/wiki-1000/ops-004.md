# Kubernetes Fundamentals

## What is Kubernetes

**Kubernetes** (K8s) is a container orchestration platform that automates deployment, scaling, and management of containerized applications. It abstracts the underlying infrastructure and provides a declarative API for managing workloads.

## Core Objects

### Pods

A **Pod** is the smallest deployable unit. It contains one or more containers that share networking and storage. In practice, most pods run a single application container.

### Deployments

A **Deployment** manages a set of identical pods. It handles rolling updates, rollbacks, and scaling. This is what you use for stateless workloads.

### Services

A **Service** provides stable networking for a set of pods. Types include **ClusterIP** (internal), **NodePort** (external via node), and **LoadBalancer** (cloud LB integration).

### ConfigMaps and Secrets

Configuration is externalized into **ConfigMaps** (plain text) and **Secrets** (base64-encoded, optionally encrypted at rest). Pods reference them as environment variables or mounted volumes.

## Namespaces

Use **namespaces** to isolate workloads by team or environment. We typically run `dev`, `staging`, and `production` namespaces in each cluster.
