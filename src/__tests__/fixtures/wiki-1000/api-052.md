# API Monitoring and Alerting

## The Four Golden Signals

1. **Latency** — time to serve a request (separate success/error latency)
2. **Traffic** — requests per second
3. **Errors** — 5xx rate, 4xx rate (separate)
4. **Saturation** — CPU, memory, connection pool usage

## Alert Thresholds

| Signal | Warning | Critical |
|---|---|---|
| P99 latency | >1s | >5s |
| Error rate (5xx) | >1% | >5% |
| Traffic drop | >30% from baseline | >70% from baseline |
| Connection pool | >70% used | >90% used |

## Dashboard Layout

- Top row: request rate, error rate, latency P50/P95/P99
- Second row: per-endpoint breakdown
- Third row: downstream dependency health
- Bottom: recent errors log

## Anomaly Detection

Static thresholds miss gradual degradation. Use rolling averages:

If current P95 latency is >2x the 7-day rolling average for this time of day, alert. This catches "it's getting slower" before it hits the hard threshold.
