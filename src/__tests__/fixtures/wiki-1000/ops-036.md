# Kubernetes Ingress

## Ingress Controllers

An **Ingress** resource defines routing rules, but you need an **Ingress controller** to implement them. We use the **AWS Load Balancer Controller** which provisions ALBs automatically.

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: api-ingress
  annotations:
    alb.ingress.kubernetes.io/scheme: internet-facing
    alb.ingress.kubernetes.io/target-type: ip
    alb.ingress.kubernetes.io/certificate-arn: arn:aws:acm:...
    alb.ingress.kubernetes.io/listen-ports: '[{"HTTPS":443}]'
    alb.ingress.kubernetes.io/ssl-redirect: "443"
spec:
  ingressClassName: alb
  rules:
    - host: api.example.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: api-server
                port:
                  number: 8080
```

## Path-Based Routing

Route different paths to different services. This is useful for microservices behind a single domain.

## Rate Limiting

Add rate limiting at the ingress level to protect backend services from traffic spikes. Most ingress controllers support annotations for rate limiting.
