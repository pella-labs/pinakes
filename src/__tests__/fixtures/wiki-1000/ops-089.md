# Kubernetes Vertical Pod Autoscaler

## What is VPA

The **Vertical Pod Autoscaler** (VPA) automatically adjusts CPU and memory requests based on actual usage. It right-sizes pods without manual intervention.

## Modes

- **Off**: only provides recommendations, doesn't change pods
- **Initial**: sets requests only at pod creation time
- **Auto**: evicts and recreates pods with updated requests

## Configuration

```yaml
apiVersion: autoscaling.k8s.io/v1
kind: VerticalPodAutoscaler
metadata:
  name: api-server-vpa
spec:
  targetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: api-server
  updatePolicy:
    updateMode: "Off"
  resourcePolicy:
    containerPolicies:
      - containerName: api-server
        minAllowed:
          cpu: "100m"
          memory: "128Mi"
        maxAllowed:
          cpu: "4"
          memory: "8Gi"
```

## Recommendations

View VPA recommendations:

```bash
kubectl describe vpa api-server-vpa
```

The output shows lower bound, target, upper bound, and uncapped target for both CPU and memory.

## Caveats

Never use VPA Auto mode with HPA on the same CPU/memory metric — they'll fight. Use VPA in recommendation mode alongside HPA, or use VPA for CPU/memory and HPA for custom metrics only.
