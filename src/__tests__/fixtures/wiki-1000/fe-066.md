# Skeleton Screens and Loading States

## Why Skeletons

**Skeleton screens** show a low-fidelity preview of the page layout while content loads. They feel faster than spinners because they communicate progress and set expectations about what will appear.

## Implementation

```tsx
function CardSkeleton() {
  return (
    <div className="animate-pulse space-y-4 rounded-lg border p-4">
      <div className="h-4 w-3/4 rounded bg-gray-200" />
      <div className="h-3 w-full rounded bg-gray-200" />
      <div className="h-3 w-5/6 rounded bg-gray-200" />
      <div className="flex gap-2">
        <div className="h-8 w-20 rounded bg-gray-200" />
        <div className="h-8 w-20 rounded bg-gray-200" />
      </div>
    </div>
  );
}
```

## Integration with Suspense

Pair skeletons with React Suspense boundaries:

```tsx
<Suspense fallback={<CardSkeleton />}>
  <CardContent />
</Suspense>
```

## Loading State Hierarchy

1. **Instant** (<100ms): no indicator needed
2. **Brief** (100-300ms): subtle opacity change or pulse
3. **Medium** (300ms-1s): skeleton screen
4. **Long** (>1s): skeleton + progress indicator or status message

Match the loading indicator to the expected duration. Never show a full-page spinner for a partial update.
