# Suspense Cache and React Cache

## React cache()

The `cache` function deduplicates data fetching in server components. Multiple components calling the same cached function receive the same result:

```typescript
import { cache } from 'react';

export const getUser = cache(async (id: string) => {
  const res = await fetch(`/api/users/${id}`);
  return res.json();
});

// Both components share the same request
async function UserName({ id }: { id: string }) {
  const user = await getUser(id);
  return <span>{user.name}</span>;
}

async function UserAvatar({ id }: { id: string }) {
  const user = await getUser(id);
  return <img src={user.avatar} alt={user.name} />;
}
```

## Cache Scope

`cache()` deduplicates per server request. Each incoming HTTP request gets a fresh cache, preventing data leaks between users.

## Unstable_cache (Next.js)

For caching across requests, Next.js provides `unstable_cache`:

```typescript
import { unstable_cache } from 'next/cache';

const getCachedProducts = unstable_cache(
  async () => db.product.findMany(),
  ['products'],
  { revalidate: 3600, tags: ['products'] }
);
```

## Cache Invalidation

Use `revalidateTag('products')` to invalidate cached data on demand, typically from a Server Action or webhook handler.

See [[fe-020]] for data fetching patterns and [[fe-021]] for ISR.
