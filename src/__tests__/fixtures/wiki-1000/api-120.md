# API Internationalization

## Accept-Language Header

```
Accept-Language: en-US, en;q=0.9, fr;q=0.8
```

Parse and use the best match:

```typescript
function resolveLocale(header: string, supported: string[]): string {
  const requested = header.split(',').map(part => {
    const [locale, qPart] = part.trim().split(';');
    const q = qPart ? parseFloat(qPart.split('=')[1]) : 1.0;
    return { locale: locale.trim(), q };
  }).sort((a, b) => b.q - a.q);

  for (const { locale } of requested) {
    const match = supported.find(s => s.startsWith(locale.split('-')[0]));
    if (match) return match;
  }
  return supported[0]; // fallback
}
```

## Response Localization

- Error messages in the requested language
- Date/time formatted per locale
- Currency formatting per locale
- Content in the requested language (if available)

## Field-Level Translations

For content APIs, return translations as a map:

```json
{
  "id": "prod_123",
  "name": {
    "en": "Wireless Headphones",
    "fr": "Casque sans fil",
    "de": "Kabellose Kopfhörer"
  }
}
```

Or use the `Accept-Language` header to return a single language and include `Content-Language` in the response.
