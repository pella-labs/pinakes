# GitOps with ArgoCD

## What is GitOps

**GitOps** is an operational framework where Git is the single source of truth for declarative infrastructure and application configuration. Changes are applied by reconciling the desired state in Git with the actual state in the cluster.

## ArgoCD Architecture

ArgoCD watches Git repositories and compares the desired manifests against what's running in the cluster. When it detects drift, it either auto-syncs or alerts the team.

Key components:
- **Application Controller** — monitors applications and compares state
- **Repo Server** — clones repos and renders manifests
- **API Server** — exposes the UI and CLI interface
- **Redis** — caches repo state

## Application Definition

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: my-service
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/myorg/k8s-manifests.git
    path: services/my-service
    targetRevision: main
  destination:
    server: https://kubernetes.default.svc
    namespace: production
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
```

## Sync Strategies

We use **automated sync with self-heal** for staging and **manual sync** for production. This gives fast iteration in staging while maintaining a human gate for prod.
