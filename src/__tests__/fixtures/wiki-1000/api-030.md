# API Response Compression

## When to Compress

Compress responses over 1KB. Below that, the overhead of compression headers and CPU isn't worth it.

## Supported Encodings

- **gzip** — universal support, good compression ratio
- **br** (Brotli) — better ratio than gzip, slower to compress, great for static assets
- **deflate** — legacy, avoid

Check `Accept-Encoding` header and pick the best match.

## Express Setup

```typescript
import compression from 'compression';

app.use(compression({
  threshold: 1024,       // min bytes to compress
  level: 6,              // zlib compression level (1-9)
  filter: (req, res) => {
    // Don't compress SSE streams
    if (req.headers.accept === 'text/event-stream') return false;
    return compression.filter(req, res);
  },
}));
```

## JSON-specific Optimization

Before compression, minimize JSON payload size:
- Strip null fields (don't send `"field": null`)
- Use short key names in high-volume endpoints
- Consider JSON:API sparse fieldsets for large objects

Compression typically reduces JSON payloads by 70-85%.
