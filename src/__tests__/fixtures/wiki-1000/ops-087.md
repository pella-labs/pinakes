# AWS CloudWatch

## Metrics

**CloudWatch** collects metrics from AWS services automatically. Custom metrics can be published from your applications.

## Custom Metrics

```bash
aws cloudwatch put-metric-data \
  --namespace "MyApp" \
  --metric-name "OrdersProcessed" \
  --value 42 \
  --unit Count
```

## Alarms

```hcl
resource "aws_cloudwatch_metric_alarm" "api_errors" {
  alarm_name          = "api-5xx-errors"
  comparison_operator = "GreaterThanThreshold"
  evaluation_periods  = 2
  metric_name         = "5XXError"
  namespace           = "AWS/ApplicationELB"
  period              = 60
  statistic           = "Sum"
  threshold           = 10
  alarm_actions       = [aws_sns_topic.alerts.arn]

  dimensions = {
    LoadBalancer = aws_lb.api.arn_suffix
  }
}
```

## Log Insights

Query CloudWatch Logs with **Log Insights**:

```
fields @timestamp, @message
| filter @message like /ERROR/
| stats count() by bin(5m)
| sort @timestamp desc
```

## Dashboards

Create operational dashboards that show the health of your system at a glance. Include metrics from EC2, RDS, ALB, and custom application metrics. Share dashboards with the team via CloudWatch's built-in sharing.
