# Progressive Web Apps (PWAs)

## Core Technologies

A **Progressive Web App** uses three key technologies to deliver an app-like experience from a web URL:

- **Service Worker**: intercepts network requests for offline support and caching
- **Web App Manifest**: defines how the app appears when installed (icon, name, theme)
- **HTTPS**: required for service workers and other modern APIs

## Service Worker Caching Strategies

- **Cache First**: serve from cache, fall back to network (good for static assets)
- **Network First**: try network, fall back to cache (good for API data)
- **Stale While Revalidate**: serve from cache immediately, update cache from network in background

## Workbox

**Workbox** (by Google) simplifies service worker development:

```typescript
import { registerRoute } from 'workbox-routing';
import { CacheFirst, StaleWhileRevalidate } from 'workbox-strategies';

registerRoute(
  ({ request }) => request.destination === 'image',
  new CacheFirst({ cacheName: 'images' })
);

registerRoute(
  ({ url }) => url.pathname.startsWith('/api/'),
  new StaleWhileRevalidate({ cacheName: 'api-cache' })
);
```

## Installation Prompt

Browsers provide an install prompt when PWA criteria are met. Capture the `beforeinstallprompt` event to show a custom install button.

## Limitations

PWAs have limited access to device APIs compared to native apps. Push notifications, background sync, and file system access are available but with restrictions. iOS Safari has historically lagged in PWA support.
