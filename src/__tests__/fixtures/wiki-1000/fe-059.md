# Error Tracking and Monitoring

## Frontend Error Types

- **JavaScript exceptions**: uncaught errors, promise rejections
- **Network errors**: failed API calls, timeouts
- **Resource loading failures**: missing scripts, stylesheets, images
- **Performance degradation**: slow interactions, memory leaks

## Sentry Integration

```typescript
import * as Sentry from '@sentry/react';

Sentry.init({
  dsn: 'https://key@sentry.io/project',
  integrations: [
    Sentry.browserTracingIntegration(),
    Sentry.replayIntegration(),
  ],
  tracesSampleRate: 0.1,
  replaysSessionSampleRate: 0.01,
  replaysOnErrorSampleRate: 1.0,
});

// Error boundary integration
function App() {
  return (
    <Sentry.ErrorBoundary fallback={<ErrorFallback />}>
      <Router />
    </Sentry.ErrorBoundary>
  );
}
```

## Source Maps

Upload source maps to your error tracking service during the build. This maps minified stack traces back to original source code. Never serve source maps publicly in production.

## Custom Context

Attach user context, feature flags, and application state to error reports. This helps engineers reproduce the exact conditions that caused the error.

## Alert Fatigue

Configure alerts for new error types and significant increases in error rates, not for every individual occurrence. Group errors by their root cause.

See [[fe-028]] for React error boundaries and [[fe-009]] for performance monitoring.
