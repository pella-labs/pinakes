# Terraform Workspaces

## What Are Workspaces

Terraform **workspaces** allow you to manage multiple environments (dev, staging, prod) from the same configuration. Each workspace has its own state file.

## Basic Usage

```bash
terraform workspace new dev
terraform workspace new staging
terraform workspace new prod

terraform workspace select prod
terraform apply -var-file="envs/prod.tfvars"
```

## Workspace-Aware Configuration

```hcl
locals {
  env = terraform.workspace

  instance_types = {
    dev     = "t3.small"
    staging = "t3.medium"
    prod    = "m5.large"
  }

  replica_counts = {
    dev     = 1
    staging = 2
    prod    = 3
  }
}

resource "aws_instance" "app" {
  instance_type = local.instance_types[local.env]
  count         = local.replica_counts[local.env]
}
```

## Workspace vs Directory Structure

Some teams prefer **separate directories** per environment instead of workspaces. The tradeoff: directories give more isolation but require duplicating configuration. Workspaces are simpler but can lead to accidental `terraform apply` in the wrong workspace.

We use workspaces for simple setups and directories for complex, divergent environments.
