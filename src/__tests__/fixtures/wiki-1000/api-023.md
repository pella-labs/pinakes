# Authentication Middleware

## JWT Verification

```typescript
import { verify } from 'jsonwebtoken';

interface AuthContext {
  userId: string;
  roles: string[];
  tenantId: string;
}

function authMiddleware(req: Request, res: Response, next: NextFunction) {
  const header = req.headers.authorization;
  if (!header?.startsWith('Bearer ')) {
    return res.status(401).json({
      error: { code: 'UNAUTHENTICATED', message: 'Missing bearer token' }
    });
  }

  try {
    const token = header.slice(7);
    const payload = verify(token, process.env.JWT_PUBLIC_KEY!, {
      algorithms: ['RS256'],
      issuer: 'auth.example.com',
    }) as AuthContext;

    req.auth = payload;
    next();
  } catch (err) {
    return res.status(401).json({
      error: { code: 'UNAUTHENTICATED', message: 'Invalid or expired token' }
    });
  }
}
```

## API Key Authentication

For service-to-service calls, use API keys passed in `X-API-Key` header. Hash the key with SHA-256 before storing. Compare using **constant-time** equality.

## Multi-Tenant Isolation

After auth, attach the `tenantId` to a request-scoped context. Every database query must include a tenant filter. Never trust client-supplied tenant IDs — derive from the token.

See [[api-004]] for rate limiting per-tenant.
