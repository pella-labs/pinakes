# REST API Collection Filtering

## Filter Syntax Options

### Simple Key-Value
```
GET /orders?status=pending&customer_id=cust_123
```

### Operator-Based
```
GET /orders?total[gte]=100&total[lte]=500&created_at[after]=2025-01-01
```

### LHS Brackets (JSON:API style)
```
GET /orders?filter[status]=pending&filter[total][gte]=100
```

## Implementation

```typescript
interface FilterConfig {
  allowedFields: string[];
  operators: Record<string, (col: string, value: any) => SQL>;
}

const operators = {
  eq: (col, val) => sql`${col} = ${val}`,
  gte: (col, val) => sql`${col} >= ${val}`,
  lte: (col, val) => sql`${col} <= ${val}`,
  like: (col, val) => sql`${col} LIKE ${val}`,
  in: (col, val) => sql`${col} IN (${val.split(',')})`,
};
```

## Security

- **Allowlist fields** — never let users filter on arbitrary columns
- **Validate operators** — prevent SQL injection via operator manipulation
- **Limit filter complexity** — max 5 filter conditions per request
- **Index filtered columns** — unindexed filters cause full table scans

See [[api-109]] for sorting and [[api-005]] for pagination.
