# Webhook Security

## Payload Signing

Every webhook payload is signed with HMAC-SHA256:

```
X-Webhook-Signature: v1=5d41402abc4b2a76b9719d911017c592
X-Webhook-Timestamp: 1699023000
```

## Verification Steps

```typescript
function verifyWebhook(
  signature: string,
  timestamp: string,
  body: string,
  secret: string
): boolean {
  // 1. Check timestamp freshness (prevent replay)
  const age = Math.abs(Date.now() / 1000 - parseInt(timestamp));
  if (age > 300) return false; // reject if >5 min old

  // 2. Compute expected signature
  const payload = `${timestamp}.${body}`;
  const expected = crypto.createHmac('sha256', secret).update(payload).digest('hex');

  // 3. Constant-time comparison
  return crypto.timingSafeEqual(
    Buffer.from(`v1=${expected}`),
    Buffer.from(signature)
  );
}
```

## IP Allowlisting

Publish the IP ranges your webhook servers send from. Subscribers can firewall to only accept traffic from these IPs.

## Secret Rotation

Support two active signing secrets simultaneously:
1. Add new secret
2. Start signing with new secret
3. Subscribers add new secret to their verification
4. Remove old secret

This allows zero-downtime rotation. See [[api-010]] for the full webhook design.
