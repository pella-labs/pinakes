# Responsive Design Strategies

## Mobile-First

The **mobile-first** approach starts with styles for the smallest screens and layers on complexity for larger viewports. This ensures the core experience works everywhere.

```css
/* Base styles — mobile */
.card {
  padding: 1rem;
  display: flex;
  flex-direction: column;
}

/* Tablet and up */
@media (min-width: 768px) {
  .card {
    flex-direction: row;
    padding: 1.5rem;
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .card {
    max-width: 1200px;
    margin: 0 auto;
  }
}
```

## Container Queries

**Container queries** let components respond to their container's size, not the viewport. This makes components truly reusable across different layout contexts.

```css
.card-container {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    flex-direction: row;
  }
}
```

## Fluid Typography

Use `clamp()` for text that scales smoothly between viewport sizes:

```css
h1 {
  font-size: clamp(1.5rem, 4vw, 3rem);
}
```

## Responsive Images

Always provide multiple sources using `srcset` and `sizes`. Use the `<picture>` element for art direction changes across breakpoints. See [[fe-009]] for how image handling affects Core Web Vitals.
