# Terraform Locals and Expressions

## Locals Block

**Locals** define computed values used throughout the configuration:

```hcl
locals {
  env         = terraform.workspace
  name_prefix = "${var.project}-${local.env}"

  common_tags = {
    Environment = local.env
    Project     = var.project
    ManagedBy   = "terraform"
  }

  subnet_cidrs = {
    public  = [for i in range(3) : cidrsubnet(var.vpc_cidr, 4, i)]
    private = [for i in range(3) : cidrsubnet(var.vpc_cidr, 4, i + 4)]
    data    = [for i in range(3) : cidrsubnet(var.vpc_cidr, 4, i + 8)]
  }
}
```

## Useful Expressions

### String Templates

```hcl
name = "${local.name_prefix}-${each.key}"
```

### For Expressions

```hcl
# Transform a list
upper_names = [for name in var.names : upper(name)]

# Filter a map
prod_instances = {
  for k, v in var.instances : k => v if v.environment == "production"
}
```

### Try and Can

```hcl
# Graceful fallback
region = try(var.override_region, data.aws_region.current.name)

# Validation
valid = can(regex("^[a-z]+$", var.name))
```

Use locals to avoid repeating complex expressions and to make configurations self-documenting.
