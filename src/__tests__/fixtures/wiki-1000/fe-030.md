# TanStack Query for Client-Side Data

## Why Not useEffect

Fetching data with `useEffect` + `useState` leads to boilerplate for loading states, error handling, caching, deduplication, and stale data. **TanStack Query** handles all of this declaratively.

## Basic Usage

```typescript
import { useQuery } from '@tanstack/react-query';

function UserProfile({ userId }: { userId: string }) {
  const { data, isLoading, error } = useQuery({
    queryKey: ['user', userId],
    queryFn: () => fetchUser(userId),
    staleTime: 5 * 60 * 1000, // 5 minutes
  });

  if (isLoading) return <Skeleton />;
  if (error) return <ErrorMessage error={error} />;
  return <ProfileCard user={data} />;
}
```

## Key Concepts

- **Query keys** identify cached data uniquely
- **staleTime** controls when data is considered stale
- **gcTime** (formerly cacheTime) controls when unused cache is garbage collected
- **Automatic refetching** on window focus, network reconnect, and interval

## Mutations

Mutations handle create, update, and delete operations with built-in optimistic update support. See [[fe-024]] for optimistic update patterns.

## Prefetching

Prefetch data before the user navigates to improve perceived performance:

```typescript
queryClient.prefetchQuery({
  queryKey: ['user', nextUserId],
  queryFn: () => fetchUser(nextUserId),
});
```
