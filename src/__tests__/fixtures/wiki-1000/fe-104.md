# Native Dialog and Popover APIs

## The dialog Element

The HTML `<dialog>` element provides built-in modal behavior:

```tsx
function ConfirmDialog({ onConfirm, onCancel }: DialogProps) {
  const dialogRef = useRef<HTMLDialogElement>(null);

  useEffect(() => {
    dialogRef.current?.showModal();
  }, []);

  return (
    <dialog ref={dialogRef} onClose={onCancel}>
      <h2>Confirm Action</h2>
      <p>Are you sure you want to proceed?</p>
      <div>
        <button onClick={onCancel}>Cancel</button>
        <button onClick={onConfirm}>Confirm</button>
      </div>
    </dialog>
  );
}
```

The `<dialog>` provides:
- Built-in backdrop
- Focus trapping
- Escape key to close
- Top-layer rendering (no z-index issues)

## Popover API

The `popover` attribute creates non-modal overlays:

```html
<button popovertarget="menu">Open Menu</button>
<div id="menu" popover>
  <nav>
    <a href="/settings">Settings</a>
    <a href="/profile">Profile</a>
  </nav>
</div>
```

Popovers auto-dismiss when clicking outside and stack in the top layer.

## When to Use

- **dialog (modal)**: confirmations, forms that require attention
- **dialog (non-modal)**: notifications, in-page panels
- **popover**: tooltips, dropdowns, menus
