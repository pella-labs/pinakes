# PostgreSQL Extensions

## The Extension Ecosystem

PostgreSQL's **extension system** is one of its killer features. Extensions can add new types, functions, indexes, and even entire query languages.

### Essential Extensions

- **pg_stat_statements** — tracks query statistics (execution time, call count). Install this on every production database.
- **pg_trgm** — trigram similarity for fuzzy text matching
- **btree_gist** — enables exclusion constraints with scalar types
- **uuid-ossp** or **pgcrypto** — UUID generation

### Data Extensions

- **PostGIS** — geospatial data with geographic queries
- **pgvector** — vector similarity search for embeddings
- **hstore** — legacy key-value store (mostly replaced by JSONB)
- **ltree** — hierarchical label trees

### Performance Extensions

- **pg_repack** — online table repacking without heavy locks
- **pg_partman** — automated partition management
- **hypopg** — hypothetical indexes for testing without creating them

## Installing Extensions

```sql
-- Check available extensions
SELECT * FROM pg_available_extensions WHERE name LIKE 'pg_%';

-- Install
CREATE EXTENSION IF NOT EXISTS pg_stat_statements;
CREATE EXTENSION IF NOT EXISTS pgvector;
```

For vector search with pgvector, see [[db-088]].
