# CI Security Scanning

## Shift Left Security

**Shift left** means moving security checks earlier in the development lifecycle. Catching vulnerabilities in CI is cheaper than finding them in production.

## SAST (Static Application Security Testing)

Analyze source code for vulnerabilities:

```yaml
security-scan:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - name: Run Semgrep
      uses: semgrep/semgrep-action@v1
      with:
        config: >-
          p/security-audit
          p/secrets
          p/typescript
```

## Dependency Scanning

Check for known vulnerabilities in dependencies:

```bash
# Node.js
npm audit --audit-level=high

# Container images
trivy image --severity HIGH,CRITICAL myapp:latest

# Infrastructure
tfsec .
checkov -d .
```

## Secret Detection

Prevent secrets from being committed:

- **pre-commit hooks**: `detect-secrets` or `gitleaks`
- **CI checks**: scan all commits in the PR branch
- **GitHub Advanced Security**: enables secret scanning on the repository

## License Compliance

Audit dependency licenses. Block copyleft licenses (GPL) in proprietary code. Use tools like **FOSSA** or **license-checker** in CI.
