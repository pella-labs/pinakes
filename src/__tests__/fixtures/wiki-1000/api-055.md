# API Changelog

## Format

Keep a machine-readable changelog per API version:

```yaml
# changelog.yaml
- version: "2.3.0"
  date: "2025-11-01"
  changes:
    - type: added
      description: "New `shipping_estimate` field on Order response"
      endpoint: "GET /orders/{id}"
    - type: deprecated
      description: "`delivery_date` field deprecated, use `shipping_estimate`"
      endpoint: "GET /orders/{id}"
      sunset: "2026-03-01"
    - type: fixed
      description: "Pagination cursor now stable across concurrent writes"
      endpoint: "GET /orders"
```

## Types

- **added** — new endpoint or field (non-breaking)
- **changed** — modified behavior (may be breaking)
- **deprecated** — still works, will be removed
- **removed** — breaking change (major version bump)
- **fixed** — bug fix
- **security** — security-related change

## Delivery

Publish the changelog at `/api/changelog` as JSON. Include a link in the `Link` header of API responses when there are recent breaking or deprecation changes.
