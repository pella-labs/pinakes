# Canary Analysis Automation

## Automated Canary Decisions

Manual canary analysis doesn't scale. Automated analysis compares canary metrics against the baseline and makes promote/rollback decisions.

## Kayenta (Spinnaker)

**Kayenta** performs statistical comparison between canary and baseline metrics. It uses Mann-Whitney U tests to determine if the canary is significantly different.

## Flagger (Kubernetes-native)

**Flagger** integrates with Istio, Linkerd, or NGINX to automate canary analysis:

```yaml
apiVersion: flagger.app/v1beta1
kind: Canary
metadata:
  name: api-server
spec:
  targetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: api-server
  service:
    port: 8080
  analysis:
    interval: 1m
    threshold: 5
    maxWeight: 50
    stepWeight: 10
    metrics:
      - name: request-success-rate
        thresholdRange:
          min: 99
        interval: 1m
      - name: request-duration
        thresholdRange:
          max: 500
        interval: 1m
    webhooks:
      - name: load-test
        url: http://flagger-loadtester/
        metadata:
          cmd: "hey -z 1m -q 10 http://api-server-canary:8080/"
```

## Key Metrics for Canary Analysis

- **Success rate**: percentage of non-error responses
- **Latency**: p99 and p50 response time
- **Error rate**: 5xx responses per second
- **Business metrics**: conversion rate, order completion rate
