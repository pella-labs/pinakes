# Data Classification

**Data classification** categorizes information by sensitivity level. It determines what security controls are required for storage, transmission, and access.

## Classification Levels

### Public
Information that can be freely disclosed. No confidentiality controls needed.
- Marketing materials
- Public documentation
- Open-source code

### Internal
Information for internal use only. Unauthorized disclosure is undesirable but not harmful.
- Internal wiki pages
- Architecture diagrams
- Non-sensitive business metrics

### Confidential
Information whose disclosure would harm the organization. Access restricted to authorized personnel.
- Customer data
- Financial records
- Business strategy documents

### Restricted
The most sensitive information. Unauthorized access could cause severe harm.
- Credentials and encryption keys (see [[sec-013]])
- PHI (see [[sec-024]])
- PII subject to GDPR (see [[sec-023]])
- Payment card data (PCI DSS scope)

## Implementation

### Schema-Level Tagging

Tag data at the schema level so controls can be automated:

```typescript
const schema = {
  users: {
    id: { classification: 'internal' },
    email: { classification: 'confidential' },
    ssn: { classification: 'restricted' },
    name: { classification: 'confidential' },
  },
};
```

### Control Mapping

Each classification level maps to required controls:
- Restricted: encryption at rest + in transit, audit logging, access review
- Confidential: encryption in transit, access controls, retention policy
- Internal: standard access controls
- Public: no restrictions
