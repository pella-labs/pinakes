# Vue 3 Composition API

## From Options to Composition

The **Composition API** introduced in Vue 3 provides a function-based way to organize component logic. Instead of splitting concerns across `data`, `methods`, `computed`, and `watch` options, related logic is grouped together.

```typescript
import { ref, computed, onMounted } from 'vue';

export function useCounter(initial = 0) {
  const count = ref(initial);
  const doubled = computed(() => count.value * 2);

  function increment() {
    count.value++;
  }

  onMounted(() => {
    console.log('Counter mounted with value:', count.value);
  });

  return { count, doubled, increment };
}
```

## Script Setup

The `<script setup>` syntax is syntactic sugar for the Composition API that reduces boilerplate:

```vue
<script setup lang="ts">
import { useCounter } from './useCounter';

const { count, doubled, increment } = useCounter(10);
</script>

<template>
  <button @click="increment">{{ count }} ({{ doubled }})</button>
</template>
```

## Reactivity System

Vue 3 uses **Proxy-based reactivity**. Unlike React's immutable updates, Vue tracks dependencies automatically. When you read `ref.value` inside a `computed` or `watchEffect`, Vue knows to re-run when that ref changes.

## Comparison with React Hooks

Both solve the same problem—reusable stateful logic—but with different tradeoffs. Vue composables have explicit dependency tracking, while React hooks rely on the dependency array convention. See [[fe-001]] for React's composition patterns.
