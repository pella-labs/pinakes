# Implementing Search Autocomplete

## Approaches

### Prefix Search with B-tree

```sql
-- Use a text_pattern_ops index for prefix matching
CREATE INDEX idx_users_name_prefix ON users (name text_pattern_ops);

SELECT name FROM users WHERE name LIKE 'ali%' LIMIT 10;
```

### Trigram Similarity

```sql
CREATE EXTENSION pg_trgm;
CREATE INDEX idx_users_name_trgm ON users USING GIN (name gin_trgm_ops);

SELECT name, similarity(name, 'alice') AS sim
FROM users
WHERE name % 'alice'  -- similarity threshold
ORDER BY sim DESC
LIMIT 10;
```

### Full-Text Search with Prefix

```sql
SELECT title FROM articles
WHERE search_vector @@ to_tsquery('english', 'datab:*');
```

### Elasticsearch Completion Suggester

```json
{
  "suggest": {
    "title-suggest": {
      "prefix": "dat",
      "completion": { "field": "title_suggest" }
    }
  }
}
```

## Choosing the Right Approach

- Simple prefix match: B-tree with `text_pattern_ops`
- Fuzzy/typo-tolerant: trigram with `pg_trgm`
- Full-text with prefix: FTS `prefix:*` syntax
- High-volume, low-latency: Elasticsearch or Redis sorted sets

For FTS details, see [[db-033]]. For Elasticsearch queries, see [[db-103]].
