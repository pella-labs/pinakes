# Micro-Animation Libraries Comparison

## CSS Transitions and Keyframes

For simple animations, pure CSS is the lightest option. No library needed for hover effects, fades, and simple transforms.

## Framer Motion

The most popular React animation library. Declarative API, layout animations, exit animations via AnimatePresence. ~30KB bundle size.

Best for: component-level animations, page transitions, gesture-driven interactions.

## GSAP

**GreenSock Animation Platform** is a professional-grade animation engine. Framework-agnostic, extremely performant, handles complex timelines.

Best for: complex sequences, scroll-triggered animations, SVG morphing, interactive infographics.

## Motion One

A lightweight (~3KB) Web Animations API wrapper. Uses hardware-accelerated animations by default.

```typescript
import { animate } from 'motion';

animate('.element', { opacity: [0, 1], y: [20, 0] }, { duration: 0.3 });
```

Best for: simple animations where bundle size matters.

## Lottie

Renders **After Effects** animations as JSON. Perfect for complex illustrated animations (onboarding flows, loading states) designed by motion designers.

## Selection Guide

| Need | Library |
|------|---------|
| Simple CSS state changes | CSS transitions |
| React component animations | Framer Motion |
| Complex timelines | GSAP |
| Minimal bundle | Motion One |
| Designer-created animations | Lottie |
