# Webhook Reliability

Making webhooks actually reliable.

## Delivery Receipt

The subscriber must return `2xx` within 30 seconds. Anything else (timeout, 4xx, 5xx) counts as a failure.

## Retry Queue

```typescript
interface WebhookDelivery {
  id: string;
  webhookId: string;
  eventId: string;
  url: string;
  payload: string;
  attempts: number;
  nextRetryAt: Date | null;
  status: 'pending' | 'delivered' | 'failed' | 'disabled';
}
```

## Dead Letter Queue

After max retries (typically 7), move to a **dead letter queue**. Provide a dashboard where subscribers can:
- View failed deliveries
- Manually retry
- Download the payload

## Event Ordering

Webhooks are NOT guaranteed to arrive in order. The subscriber must handle out-of-order events. Include a `sequence_number` or `timestamp` so the subscriber can sort.

## Duplicate Detection

At-least-once delivery means duplicates happen. The subscriber should deduplicate using the `event_id`. See [[api-007]] for idempotency patterns.
