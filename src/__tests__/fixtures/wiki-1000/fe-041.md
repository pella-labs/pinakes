# React Portals and Modal Patterns

## What Are Portals

**React Portals** render children into a DOM node outside the parent component's hierarchy. This is essential for modals, tooltips, and dropdowns that need to escape overflow hidden or z-index stacking contexts.

```tsx
import { createPortal } from 'react-dom';

function Modal({ children, isOpen }: ModalProps) {
  if (!isOpen) return null;

  return createPortal(
    <div className="modal-overlay">
      <div className="modal-content" role="dialog" aria-modal="true">
        {children}
      </div>
    </div>,
    document.getElementById('modal-root')!
  );
}
```

## Event Bubbling

Portal children still participate in the React event bubbling tree of their parent component, even though they live elsewhere in the DOM. This means a click inside a portal will bubble up through the React tree, not the DOM tree.

## Accessible Modals

A proper modal implementation requires:

- Focus trap (see [[fe-042]])
- Scroll lock on the body
- Escape key to close
- `aria-modal="true"` and `role="dialog"`
- Return focus to the trigger element on close

## Headless UI Libraries

Libraries like **Radix UI** and **Headless UI** provide unstyled, accessible modal primitives. They handle all the accessibility concerns while letting you control the styling.
