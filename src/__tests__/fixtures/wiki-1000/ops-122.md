# Terraform Testing with Terratest

## Why Terratest

**Terratest** is a Go library for testing infrastructure code. It provisions real infrastructure, validates it, and tears it down.

## Example Test

```go
package test

import (
    "testing"
    "github.com/gruntwork-io/terratest/modules/terraform"
    "github.com/gruntwork-io/terratest/modules/aws"
    "github.com/stretchr/testify/assert"
)

func TestVPCModule(t *testing.T) {
    t.Parallel()

    terraformOptions := &terraform.Options{
        TerraformDir: "../modules/vpc",
        Vars: map[string]interface{}{
            "cidr_block":  "10.99.0.0/16",
            "environment": "test",
            "azs":         []string{"us-east-1a", "us-east-1b"},
        },
    }

    defer terraform.Destroy(t, terraformOptions)
    terraform.InitAndApply(t, terraformOptions)

    vpcId := terraform.Output(t, terraformOptions, "vpc_id")
    assert.NotEmpty(t, vpcId)

    subnets := aws.GetSubnetsForVpc(t, vpcId, "us-east-1")
    assert.Equal(t, 6, len(subnets))  // 2 AZs * 3 tiers
}
```

## Best Practices

- Run tests in a dedicated AWS account
- Use unique naming to avoid conflicts with parallel tests
- Set timeouts appropriate for resource creation time
- Clean up with `defer terraform.Destroy` to handle test failures
- Tag test resources for easy identification and cleanup

## Cost Control

Terratest creates real infrastructure. Run comprehensive tests nightly, not on every PR. Use shorter tests (validate plan, check syntax) for PR checks.
