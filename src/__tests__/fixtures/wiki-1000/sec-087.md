# Digital Signatures

**Digital signatures** provide authentication, integrity, and non-repudiation. They prove that a message was created by the claimed sender and hasn't been altered.

## How They Work

1. Signer hashes the message
2. Signer encrypts the hash with their private key
3. Recipient decrypts the hash with the signer's public key
4. Recipient hashes the message independently and compares

## Algorithm Selection

- **Ed25519**: fast, small signatures, strong security. Preferred for new implementations.
- **ECDSA (P-256)**: widely supported, good performance. Required by some standards.
- **RSA-PSS (2048+ bit)**: legacy compatibility. Large keys and signatures.

## Implementation

```typescript
import { generateKeyPairSync, sign, verify } from 'crypto';

const { privateKey, publicKey } = generateKeyPairSync('ed25519');

const data = Buffer.from('important message');
const signature = sign(null, data, privateKey);
const isValid = verify(null, data, publicKey, signature);
```

## Use Cases

- API request signing (ensures payload integrity)
- Software distribution (code signing)
- Document signing
- JWT signatures (see [[sec-003]])
- Git commit signing

## Key Management

- Keep private keys in HSMs or secure key stores
- Distribute public keys via well-known endpoints or key servers
- Implement key rotation with overlap periods (see [[sec-016]])
- Revoke compromised keys immediately and maintain a revocation list
