# Security Headers for APIs

APIs require a different set of **security headers** than web pages. Some web-oriented headers are irrelevant; others are essential.

## Essential API Headers

```
Strict-Transport-Security: max-age=31536000
X-Content-Type-Options: nosniff
Cache-Control: no-store
Content-Type: application/json
```

## Headers to Omit

APIs typically don't need:
- CSP (no HTML rendering)
- X-Frame-Options (no framing)
- Referrer-Policy (no navigation)

## Cache-Control

API responses containing sensitive data should never be cached:

```
Cache-Control: no-store, no-cache, must-revalidate, private
Pragma: no-cache
```

This prevents proxies and browsers from storing sensitive API responses.

## Content-Type Enforcement

Always set `Content-Type` and validate the request's `Content-Type`:

```typescript
app.use((req, res, next) => {
  if (req.method !== 'GET' && !req.is('application/json')) {
    return res.status(415).json({ error: 'Unsupported Media Type' });
  }
  next();
});
```

This helps prevent CSRF attacks via form submissions (see [[sec-012]]).

## Rate Limit Headers

Include rate limit information so clients can self-regulate (see [[sec-026]]).

See [[sec-033]] for the web-focused security headers checklist.
