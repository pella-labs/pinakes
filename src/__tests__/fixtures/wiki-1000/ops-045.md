# Environment Promotion Rules

## Promotion Flow

Code moves through environments with automated gates:

```
feature branch → dev → staging → production
```

Each transition requires specific criteria to be met.

## Dev Gate

- All CI checks pass (lint, typecheck, test)
- PR approved by at least one reviewer
- No critical security findings in SAST scan

## Staging Gate

- All dev gate criteria plus:
- Integration tests pass against staging dependencies
- Database migration runs successfully
- Performance regression check (latency within 10% of baseline)
- Smoke tests pass

## Production Gate

- All staging gate criteria plus:
- Manual approval from on-call engineer or team lead
- Change window respected (no deploys Friday 4pm-Monday 8am without VP approval)
- Rollback plan documented for high-risk changes
- Feature flag configured for gradual rollout if applicable

## Automated Promotion

Dev-to-staging is fully automated. Staging-to-production is automated for low-risk changes (config updates, minor patches) and manual for high-risk changes (schema migrations, new services, major features).
