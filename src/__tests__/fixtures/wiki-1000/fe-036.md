# Image Optimization for the Web

## Why It Matters

Images account for the largest portion of page weight on most websites. Unoptimized images are the single biggest cause of poor **LCP** scores.

## Modern Formats

- **WebP**: 25-35% smaller than JPEG at comparable quality
- **AVIF**: 50% smaller than JPEG, but slower to encode
- **SVG**: vector format for icons and illustrations

Use the `<picture>` element for format fallbacks:

```html
<picture>
  <source srcset="hero.avif" type="image/avif" />
  <source srcset="hero.webp" type="image/webp" />
  <img src="hero.jpg" alt="Hero image" width="1200" height="600" />
</picture>
```

## Next.js Image Component

The `next/image` component handles optimization automatically:

- Lazy loading by default
- Automatic format selection (WebP/AVIF)
- Responsive `srcset` generation
- Size optimization per device

## Best Practices

- Always set explicit `width` and `height` to prevent CLS
- Use `loading="lazy"` for below-the-fold images
- Use `loading="eager"` and `fetchpriority="high"` for the LCP image
- Serve images from a CDN with proper cache headers

See [[fe-009]] for Core Web Vitals and [[fe-010]] for performance budgets.
