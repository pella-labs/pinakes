# Terraform Output Values

## Basics

**Outputs** expose values from your Terraform configuration for use by other configurations or scripts.

```hcl
output "vpc_id" {
  description = "ID of the VPC"
  value       = aws_vpc.main.id
}

output "private_subnet_ids" {
  description = "IDs of private subnets"
  value       = aws_subnet.private[*].id
}

output "database_endpoint" {
  description = "RDS endpoint"
  value       = aws_db_instance.main.endpoint
  sensitive   = true
}
```

## Cross-Configuration References

Use **remote state** to reference outputs from another configuration:

```hcl
data "terraform_remote_state" "network" {
  backend = "s3"
  config = {
    bucket = "company-terraform-state"
    key    = "network/terraform.tfstate"
    region = "us-east-1"
  }
}

resource "aws_instance" "app" {
  subnet_id = data.terraform_remote_state.network.outputs.private_subnet_ids[0]
}
```

## Sensitive Outputs

Mark outputs as `sensitive = true` to prevent them from being displayed in terminal output. They're still stored in state — state encryption is the real protection.

## JSON Output

```bash
terraform output -json > outputs.json
```

Useful for feeding values into scripts or other automation tools.
