# SQLite WAL Mode

## What is WAL?

**Write-Ahead Logging** in SQLite replaces the default rollback journal with an append-only log file. Instead of writing changes to the database file and keeping a rollback journal, changes go to a separate WAL file. Readers see a consistent snapshot without blocking writers.

## Enabling WAL

```sql
PRAGMA journal_mode = WAL;
```

Once set, WAL mode persists across connections. It is the recommended mode for any application with concurrent readers and a single writer.

## How It Works

Writers append to the WAL file. Readers check the WAL for the latest committed version of each page, falling back to the main database file. A **checkpoint** operation transfers WAL contents back to the main database file.

Key pragmas for tuning:

```sql
PRAGMA synchronous = NORMAL;  -- safe with WAL
PRAGMA busy_timeout = 5000;   -- wait up to 5s for locks
PRAGMA wal_autocheckpoint = 1000;  -- checkpoint every 1000 pages
```

## Limitations

- WAL does not work over network filesystems (NFS, SMB)
- Maximum WAL size can grow if checkpoints are delayed
- Only one writer at a time (but readers never block)
