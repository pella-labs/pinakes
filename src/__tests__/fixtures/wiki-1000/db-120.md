# Database Observability with OpenTelemetry

## Tracing Database Calls

Instrument database queries with OpenTelemetry spans to trace latency through your application:

```typescript
import { trace } from '@opentelemetry/api';

const tracer = trace.getTracer('db');

async function queryWithTracing(sql: string, params: any[]) {
  return tracer.startActiveSpan('db.query', async (span) => {
    span.setAttribute('db.system', 'postgresql');
    span.setAttribute('db.statement', sql);
    try {
      const result = await pool.query(sql, params);
      span.setAttribute('db.rows_affected', result.rowCount);
      return result;
    } catch (error) {
      span.recordException(error);
      throw error;
    } finally {
      span.end();
    }
  });
}
```

## Metrics to Export

- Query duration histogram
- Connection pool utilization gauge
- Error rate counter
- Rows returned/affected histogram

## Correlating with Application Traces

When a request is slow, the trace shows exactly which database query is the bottleneck. This is far more useful than aggregate statistics alone.
