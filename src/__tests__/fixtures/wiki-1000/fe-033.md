# Redux Middleware and Side Effects

## Middleware Basics

Redux **middleware** intercepts dispatched actions before they reach the reducer. This is where side effects like API calls, logging, and analytics live.

## Redux Thunk

The simplest middleware for async operations. A thunk is a function that returns a function:

```typescript
import { createAsyncThunk } from '@reduxjs/toolkit';

export const fetchUsers = createAsyncThunk(
  'users/fetchAll',
  async (_, { rejectWithValue }) => {
    try {
      const response = await api.getUsers();
      return response.data;
    } catch (err) {
      return rejectWithValue(err.message);
    }
  }
);
```

RTK's `createAsyncThunk` generates pending/fulfilled/rejected action types automatically.

## RTK Query

For data fetching specifically, **RTK Query** provides a complete solution:

- Auto-generated hooks from API definitions
- Caching and invalidation
- Optimistic updates
- Polling and streaming

```typescript
const api = createApi({
  baseQuery: fetchBaseQuery({ baseUrl: '/api' }),
  endpoints: (builder) => ({
    getUsers: builder.query<User[], void>({
      query: () => 'users',
      providesTags: ['User'],
    }),
  }),
});
```

## Redux Saga vs Thunk

**Redux Saga** uses generator functions for complex async flows. It is more powerful but significantly more complex. For most applications, thunks or RTK Query are sufficient. See [[fe-002]] for state management comparison.
