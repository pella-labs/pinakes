# Certificate Management

## cert-manager

**cert-manager** automates TLS certificate provisioning and renewal in Kubernetes. It supports Let's Encrypt, AWS ACM Private CA, and other issuers.

## Let's Encrypt Setup

```yaml
apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: letsencrypt-prod
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: platform@example.com
    privateKeySecretRef:
      name: letsencrypt-prod
    solvers:
      - dns01:
          route53:
            region: us-east-1
```

## Certificate Resources

Request certificates declaratively:

```yaml
apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: api-tls
  namespace: production
spec:
  secretName: api-tls-secret
  issuerRef:
    name: letsencrypt-prod
    kind: ClusterIssuer
  dnsNames:
    - api.example.com
    - "*.api.example.com"
```

## Renewal

cert-manager automatically renews certificates 30 days before expiry. Monitor the `cert-manager_certificate_expiration_timestamp_seconds` Prometheus metric to alert on renewal failures.

## Internal mTLS

For internal service-to-service TLS, use **Istio** (see [[ops-029]]) or cert-manager with a private CA. Don't use self-signed certificates in production.
