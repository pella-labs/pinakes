# Container Security

## Image Security

- Use **minimal base images** (Alpine, distroless) to reduce attack surface
- Scan images for vulnerabilities in CI with **Trivy**
- Never run containers as root
- Use read-only root filesystems where possible

## Runtime Security

### Security Contexts

```yaml
securityContext:
  runAsNonRoot: true
  runAsUser: 1000
  readOnlyRootFilesystem: true
  allowPrivilegeEscalation: false
  capabilities:
    drop:
      - ALL
```

### Pod Security Standards

Kubernetes **Pod Security Standards** define three levels:

- **Privileged** — unrestricted (only for system-level pods)
- **Baseline** — prevents known privilege escalations
- **Restricted** — hardened, best practices enforced

Apply the **restricted** profile to all application namespaces.

## Supply Chain Security

- Sign images with **cosign** and verify signatures before deployment
- Use **SBOM** (Software Bill of Materials) generation in CI
- Pin all base image digests, not tags
- Use **admission controllers** (Kyverno, OPA Gatekeeper) to enforce image policies

See [[ops-002]] for Docker best practices and [[ops-025]] for registry management.
