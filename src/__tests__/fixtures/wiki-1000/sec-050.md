# Security Code Review Checklist

A focused **security code review** catches vulnerabilities that automated tools miss. This checklist targets the most common and impactful issues.

## Authentication

- [ ] Is authentication required on all non-public endpoints?
- [ ] Are passwords hashed with Argon2id or bcrypt? (see [[sec-037]])
- [ ] Are session IDs regenerated after login? (see [[sec-040]])
- [ ] Is multi-factor authentication supported for sensitive operations?

## Authorization

- [ ] Is authorization checked on every request, not just the UI?
- [ ] Can users access other users' resources by changing IDs? (IDOR)
- [ ] Are admin functions protected by role checks? (see [[sec-004]])
- [ ] Is the default access denied?

## Input Handling

- [ ] Are all inputs validated against a schema? (see [[sec-036]])
- [ ] Are SQL queries parameterized? (see [[sec-011]])
- [ ] Is user output HTML-encoded? (see [[sec-009]])
- [ ] Are file uploads validated and stored safely? (see [[sec-039]])

## Data Protection

- [ ] Are secrets stored in environment variables, not code? (see [[sec-013]])
- [ ] Is sensitive data encrypted at rest? (see [[sec-025]])
- [ ] Are API responses free of sensitive data leaks?
- [ ] Is PII logged only when necessary, and never in debug mode?

## Infrastructure

- [ ] Are security headers set? (see [[sec-033]])
- [ ] Are cookies configured securely? (see [[sec-034]])
- [ ] Is CORS configured restrictively? (see [[sec-007]])
- [ ] Are dependencies free of known vulnerabilities?

Review against this checklist for every PR that touches authentication, authorization, or data handling.
