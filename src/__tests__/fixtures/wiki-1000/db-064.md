# PostgreSQL JSONB Indexing Strategies

## GIN Index Types

There are two GIN operator classes for JSONB:

### jsonb_ops (default)
Supports `@>`, `?`, `?|`, `?&` operators:

```sql
CREATE INDEX idx_data ON products USING GIN (data);

-- Uses the index
SELECT * FROM products WHERE data @> '{"category": "electronics"}';
SELECT * FROM products WHERE data ? 'discontinued';
```

### jsonb_path_ops
Only supports `@>` but is smaller and faster:

```sql
CREATE INDEX idx_data_path ON products USING GIN (data jsonb_path_ops);
```

## Expression Indexes

For specific keys you query frequently, a B-tree expression index is more efficient than GIN:

```sql
CREATE INDEX idx_category ON products ((data->>'category'));

-- Uses the index
SELECT * FROM products WHERE data->>'category' = 'electronics';
```

## Composite Approaches

Combine JSONB columns with regular columns. Index the regular columns for filtering, use JSONB for flexible schema:

```sql
CREATE TABLE events (
  id BIGSERIAL PRIMARY KEY,
  event_type TEXT NOT NULL,      -- indexed, filterable
  created_at TIMESTAMPTZ NOT NULL, -- indexed, sortable
  payload JSONB                   -- flexible, GIN-indexed
);
```

For general JSONB usage, see [[db-019]]. For indexing fundamentals, see [[db-020]].
