# GraphQL Security

**GraphQL** APIs have unique security considerations compared to REST. The query language's flexibility is both its strength and its primary attack vector.

## Query Depth Limiting

Deeply nested queries can exhaust server resources:

```graphql
# Malicious query
{
  user { posts { comments { author { posts { comments { author { ... } } } } } } }
}
```

Limit query depth to a reasonable maximum (typically 7-10 levels).

## Query Complexity Analysis

Assign cost to each field and reject queries that exceed a budget:

```typescript
const costMap = {
  'Query.users': { complexity: 10 },
  'User.posts': { complexity: 5 },
  'Post.comments': { complexity: 3 },
};
// Total complexity must be < 1000
```

## Introspection

Disable **introspection** in production. Introspection reveals your entire schema, including internal types and fields:

```typescript
const server = new ApolloServer({
  introspection: process.env.NODE_ENV !== 'production',
});
```

## Batch Attack Prevention

GraphQL allows sending multiple operations in a single request. Limit batch size and apply rate limiting per operation, not per HTTP request (see [[sec-026]]).

## Authorization

Implement authorization at the **resolver level**, not the endpoint level. Every field that returns sensitive data must check permissions independently.

See [[sec-036]] for input validation patterns applicable to GraphQL variables.
