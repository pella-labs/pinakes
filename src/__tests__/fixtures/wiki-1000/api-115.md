# API Schema Validation at the Edge

## Why Validate Early

Rejecting invalid requests at the gateway saves backend resources. A malformed body that gets parsed, routed, and then rejected at the business logic layer wastes 10x the CPU.

## JSON Schema at the Gateway

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["customerId", "items"],
  "properties": {
    "customerId": { "type": "string", "pattern": "^cust_[a-zA-Z0-9]+$" },
    "items": {
      "type": "array",
      "minItems": 1,
      "maxItems": 100,
      "items": {
        "type": "object",
        "required": ["productId", "quantity"],
        "properties": {
          "productId": { "type": "string" },
          "quantity": { "type": "integer", "minimum": 1, "maximum": 999 }
        }
      }
    }
  },
  "additionalProperties": false
}
```

## Edge vs Backend Validation

| Concern | Edge (Gateway) | Backend (Service) |
|---|---|---|
| Schema shape | Yes | No (trust gateway) |
| Business rules | No | Yes |
| Type coercion | No (strict) | If needed |
| Error format | Generic | Domain-specific |

## Performance

JSON Schema validation adds <1ms for typical payloads. It's negligible compared to the cost of processing an invalid request.
