# API Key Management

## Key Structure

`prefix_environmenttype_randompart`

```
sk_live_EXAMPLE_KEY_REPLACE_ME_0000
sk_test_EXAMPLE_KEY_REPLACE_ME_0000
pk_live_EXAMPLE_KEY_REPLACE_ME_0000
```

- `sk` = secret key (server-side only)
- `pk` = publishable key (client-safe)
- `live` / `test` = environment
- Random part: 24+ characters, cryptographically random

## Storage

Hash with SHA-256 before storing. Never store plaintext keys. Show the full key only once at creation — after that, show only the last 4 characters.

## Rotation

- Support multiple active keys per account
- New key issued, old key deprecated with a TTL
- After TTL, old key returns `401` with a `key_expired` error code

## Scoping

Keys should have **scoped permissions**:

```json
{
  "key_id": "key_abc",
  "permissions": ["orders:read", "orders:write"],
  "rate_limit_tier": "standard",
  "ip_allowlist": ["10.0.0.0/8"]
}
```
