# MySQL Replication

## Binary Log Replication

MySQL replication works by streaming the **binary log** (binlog) from the primary to replicas. There are three binlog formats:

- **Statement-based**: logs SQL statements. Compact but non-deterministic functions (NOW(), RAND()) can cause divergence.
- **Row-based**: logs actual row changes. Safe but verbose.
- **Mixed**: uses statement-based by default, switches to row-based when needed.

Row-based replication is recommended for reliability.

## Setting Up Replication

```sql
-- On primary
CREATE USER 'repl'@'%' IDENTIFIED BY 'repl_password';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';

-- On replica
CHANGE REPLICATION SOURCE TO
  SOURCE_HOST='primary.example.com',
  SOURCE_USER='repl',
  SOURCE_PASSWORD='repl_password',
  SOURCE_AUTO_POSITION=1;
START REPLICA;
```

## GTID Replication

**Global Transaction Identifiers** (GTIDs) simplify failover. Each transaction gets a unique ID, so replicas can auto-position without specifying binlog coordinates.

## Group Replication

MySQL Group Replication provides virtually synchronous replication with automatic failover. It's MySQL's answer to PostgreSQL's Patroni/Citus for high availability.

See [[db-015]] for replication topology patterns.
