# Database Connection Strings

A connection string encodes everything needed to connect to a database. Formats vary by database and driver.

## PostgreSQL

```
postgresql://user:password@host:5432/dbname?sslmode=require
```

Components:
- Protocol: `postgresql://` or `postgres://`
- Credentials: `user:password@`
- Host and port: `host:5432`
- Database name: `/dbname`
- Parameters: `?sslmode=require&connect_timeout=10`

## MySQL

```
mysql://user:password@host:3306/dbname
```

## SQLite

Just a file path:
```
/path/to/database.sqlite
```

Or for in-memory: `:memory:`

## Environment Variables

Never hardcode connection strings. Use environment variables:

```typescript
const connectionString = process.env.DATABASE_URL;
if (!connectionString) {
  throw new Error('DATABASE_URL is required');
}
```

## Connection String Security

- Never log connection strings (they contain passwords)
- Use SSL in production (`sslmode=require` or `sslmode=verify-full`)
- Rotate credentials regularly
- Consider IAM-based authentication for cloud databases
