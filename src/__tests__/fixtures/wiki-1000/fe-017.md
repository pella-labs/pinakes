# Testing React Components with Playwright

## Why Playwright

**Playwright** offers cross-browser end-to-end testing with a modern API. Unlike Cypress, Playwright runs outside the browser process, enabling true multi-tab and multi-origin testing.

## Component Testing

Playwright now supports **component testing** directly, rendering React components in a real browser without a full application server.

```typescript
import { test, expect } from '@playwright/experimental-ct-react';
import { Counter } from './Counter';

test('increments on click', async ({ mount }) => {
  const component = await mount(<Counter initial={5} />);
  await component.getByRole('button', { name: 'Increment' }).click();
  await expect(component.getByText('6')).toBeVisible();
});
```

## E2E Testing Patterns

- Use **page object models** to encapsulate page interactions
- Prefer `getByRole` and `getByText` over CSS selectors
- Use `test.describe` for grouping related tests
- Take screenshots on failure for debugging

## Visual Regression

Playwright supports **screenshot comparison** for visual regression testing. This catches subtle CSS changes that unit tests miss.

```typescript
test('matches snapshot', async ({ page }) => {
  await page.goto('/dashboard');
  await expect(page).toHaveScreenshot('dashboard.png', {
    maxDiffPixelRatio: 0.01,
  });
});
```
