# Performance Budgets

## Defining a Budget

A **performance budget** is a set of limits on metrics that affect user experience. These limits are enforced in CI to prevent regressions.

Common budget categories:

- **Bundle size**: max JS payload per route (e.g., 150KB gzipped for initial load)
- **LCP**: target under 2.5s on 3G throttled
- **Total blocking time**: under 300ms
- **Image weight**: max 500KB per page
- **Third-party scripts**: max 50KB total

## Enforcement

Integrate budget checks into your build pipeline:

```json
{
  "budgets": [
    {
      "path": "/",
      "resourceSizes": [
        { "resourceType": "script", "budget": 150 },
        { "resourceType": "stylesheet", "budget": 30 },
        { "resourceType": "image", "budget": 500 }
      ]
    }
  ]
}
```

Tools like **bundlesize**, **Lighthouse CI**, and **webpack-bundle-analyzer** can automate checks. A failing budget should block the merge just like a failing test.

## Incremental Strategy

Start by measuring your current baseline, then set budgets 10% below. Tighten them quarterly. The goal is preventing regressions, not achieving perfection overnight. See [[fe-009]] for which metrics to prioritize.
