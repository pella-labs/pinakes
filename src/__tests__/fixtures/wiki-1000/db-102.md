# Database Logging and Slow Query Analysis

## PostgreSQL Logging Configuration

```
# postgresql.conf
log_min_duration_statement = 100    # log queries taking >100ms
log_statement = 'none'              # don't log all statements
log_lock_waits = on                 # log waits longer than deadlock_timeout
log_temp_files = 0                  # log all temp file usage
log_autovacuum_min_duration = 0     # log all autovacuum activity
```

## pg_stat_statements

The essential extension for query performance analysis:

```sql
SELECT
  query,
  calls,
  total_exec_time / calls AS avg_time_ms,
  rows / calls AS avg_rows,
  shared_blks_hit / NULLIF(shared_blks_hit + shared_blks_read, 0) AS cache_hit_ratio
FROM pg_stat_statements
ORDER BY total_exec_time DESC
LIMIT 20;
```

## MySQL Slow Query Log

```ini
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 0.1
log_queries_not_using_indexes = 1
```

Use `pt-query-digest` (Percona Toolkit) to analyze slow query logs.

## Actionable Analysis

For each slow query:
1. Run EXPLAIN ANALYZE (see [[db-009]])
2. Check for missing indexes ([[db-020]])
3. Look for lock contention ([[db-032]])
4. Consider query rewriting ([[db-018]])
