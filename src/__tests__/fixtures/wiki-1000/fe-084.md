# CSS Logical Properties

## Physical vs Logical

Traditional CSS uses physical directions (left, right, top, bottom). **Logical properties** use flow-relative directions (inline-start, inline-end, block-start, block-end), adapting automatically to writing direction.

## Mapping

| Physical | Logical |
|----------|---------|
| margin-left | margin-inline-start |
| margin-right | margin-inline-end |
| padding-top | padding-block-start |
| width | inline-size |
| height | block-size |
| top | inset-block-start |
| border-left | border-inline-start |

## Why They Matter

When supporting RTL languages (Arabic, Hebrew), logical properties eliminate the need for separate RTL stylesheets:

```css
/* Works in both LTR and RTL */
.sidebar {
  margin-inline-start: 1rem;
  padding-inline-end: 2rem;
  border-inline-end: 1px solid #e5e7eb;
}
```

## Shorthand

```css
.element {
  /* block-start block-end inline-start inline-end */
  margin: 1rem 2rem;      /* traditional */
  margin-block: 1rem;      /* top + bottom */
  margin-inline: 2rem;     /* left + right (or start + end in RTL) */
}
```

## Browser Support

Logical properties are supported in all modern browsers. They should be the default for any project that might need internationalization.

See [[fe-026]] for i18n patterns and [[fe-015]] for responsive design.
