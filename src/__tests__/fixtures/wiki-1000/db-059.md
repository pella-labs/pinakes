# Redis Clustering

## Redis Cluster Architecture

**Redis Cluster** provides automatic sharding across multiple Redis nodes. Data is split into 16,384 hash slots, distributed across the cluster nodes.

## How Slots Work

Each key is hashed to a slot: `HASH_SLOT = CRC16(key) mod 16384`. Each node is responsible for a subset of slots. When you add or remove nodes, slots are rebalanced.

## Setup

A minimal cluster requires 6 nodes — 3 masters and 3 replicas:

```bash
redis-cli --cluster create \
  127.0.0.1:7000 127.0.0.1:7001 127.0.0.1:7002 \
  127.0.0.1:7003 127.0.0.1:7004 127.0.0.1:7005 \
  --cluster-replicas 1
```

## Multi-Key Operations

Operations spanning multiple keys work only if all keys hash to the same slot. Use **hash tags** to force co-location:

```
MGET {user:1}:name {user:1}:email  -- same slot (hash tag is "user:1")
```

## Cluster Limitations

- No multi-database support (only database 0)
- Some commands are restricted or unavailable
- Pub/Sub messages are broadcast to all nodes (not slot-aware)
- Lua scripts must use keys in the same slot
