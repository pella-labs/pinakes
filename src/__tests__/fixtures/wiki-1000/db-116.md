# MongoDB Change Streams

## Real-Time Data Changes

**Change streams** let you watch for real-time changes to your MongoDB data. Built on the oplog, they provide a cursor-based API for subscribing to inserts, updates, deletes, and replacements.

```javascript
const pipeline = [
  { $match: { operationType: { $in: ['insert', 'update'] } } },
  { $match: { 'fullDocument.status': 'urgent' } }
];

const changeStream = db.collection('tickets').watch(pipeline);

changeStream.on('change', (change) => {
  console.log('Change detected:', change.operationType);
  console.log('Document:', change.fullDocument);
});
```

## Resume Tokens

Each change event includes a **resume token**. Store it and use it to resume the stream after a disconnect:

```javascript
let resumeToken = loadResumeToken();
const options = resumeToken ? { resumeAfter: resumeToken } : {};
const stream = collection.watch(pipeline, options);

stream.on('change', (change) => {
  processChange(change);
  saveResumeToken(change._id);  // persist for crash recovery
});
```

## Use Cases

- Keeping search indexes in sync
- Real-time notifications
- Triggering workflows on data changes
- CDC without external tools (compare with Debezium, [[db-026]])

Change streams require a replica set or sharded cluster. They don't work on standalone MongoDB instances.
