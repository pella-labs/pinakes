# Vector Search with pgvector

## Setup

```sql
CREATE EXTENSION vector;

CREATE TABLE documents (
  id SERIAL PRIMARY KEY,
  content TEXT NOT NULL,
  embedding vector(384)  -- dimension matches your model
);
```

## Inserting Vectors

```typescript
const embedding = await embedModel.encode(text);
await pool.query(
  'INSERT INTO documents (content, embedding) VALUES ($1, $2)',
  [text, `[${embedding.join(',')}]`]
);
```

## Similarity Search

```sql
-- Cosine similarity (most common for normalized embeddings)
SELECT id, content, 1 - (embedding <=> $1) AS similarity
FROM documents
ORDER BY embedding <=> $1
LIMIT 10;

-- L2 distance
SELECT id, content, embedding <-> $1 AS distance
FROM documents
ORDER BY embedding <-> $1
LIMIT 10;

-- Inner product
SELECT id, content, (embedding <#> $1) * -1 AS inner_product
FROM documents
ORDER BY embedding <#> $1
LIMIT 10;
```

## Indexing

Without an index, pgvector does an exact nearest-neighbor scan. For large datasets, use an approximate index:

```sql
-- HNSW index (recommended, better recall)
CREATE INDEX ON documents USING hnsw (embedding vector_cosine_ops);

-- IVFFlat index (faster build, more tuning needed)
CREATE INDEX ON documents USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
```

## Hybrid Search

Combine vector search with FTS (see [[db-033]]) using **Reciprocal Rank Fusion** for best results. Query both indexes, merge rankings with `1/(k+rank)` scoring.
