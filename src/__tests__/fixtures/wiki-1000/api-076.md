# gRPC vs REST Performance Comparison

## Serialization

| Format | Encode (μs) | Decode (μs) | Size (bytes) |
|---|---|---|---|
| JSON | 45 | 38 | 284 |
| Protobuf | 8 | 6 | 112 |

Protobuf is ~5x faster and ~2.5x smaller for typical API payloads.

## HTTP/2 Multiplexing

gRPC uses HTTP/2 natively. Multiple requests share a single TCP connection with no head-of-line blocking. REST over HTTP/1.1 is limited to 6 parallel connections per domain.

## Streaming

gRPC supports four streaming patterns out of the box:
- Unary (request-response)
- Server streaming
- Client streaming
- Bidirectional streaming

REST requires separate solutions for each (SSE, chunked upload, WebSocket).

## When REST Still Wins

- Browser-native (no gRPC-web proxy needed)
- Better caching (HTTP cache semantics)
- Simpler debugging (curl, browser)
- Wider ecosystem (every language, every tool)
