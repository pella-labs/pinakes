# Secrets Management

## Principles

**Secrets** (API keys, database credentials, TLS certificates) must never appear in source code, environment variable definitions committed to Git, or container images.

## AWS Secrets Manager

We use **AWS Secrets Manager** as the primary secrets store:

- Automatic rotation for RDS credentials
- Fine-grained IAM policies per secret
- Versioning and audit trail via CloudTrail
- Cross-region replication for DR

## Kubernetes Integration

Secrets flow from AWS Secrets Manager to pods via the **External Secrets Operator**:

```yaml
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: db-credentials
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: aws-secrets-manager
    kind: ClusterSecretStore
  target:
    name: db-credentials
  data:
    - secretKey: username
      remoteRef:
        key: prod/database
        property: username
    - secretKey: password
      remoteRef:
        key: prod/database
        property: password
```

## Rotation

All secrets should have an automated rotation schedule. Database passwords rotate every 30 days. API keys rotate every 90 days. TLS certificates auto-renew via **cert-manager** 30 days before expiry.

## Emergency Rotation

When a secret is compromised, rotate immediately, then audit access logs to determine blast radius. See [[ops-055]] for incident response procedures.
