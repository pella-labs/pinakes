# API Request Body Size Limits

## Default Limits

| Endpoint Type | Max Body | Rationale |
|---|---|---|
| Standard CRUD | 100KB | Typical JSON payload |
| File upload | 50MB | User content |
| Bulk import | 10MB | Large batch |
| Webhook receiver | 1MB | External event |

## Express Configuration

```typescript
// Global default
app.use(express.json({ limit: '100kb' }));

// Per-route override
app.post('/files', express.raw({ limit: '50mb', type: '*/*' }), uploadHandler);
app.post('/bulk/import', express.json({ limit: '10mb' }), importHandler);
```

## Why It Matters

Without limits, an attacker can send a 1GB JSON body and:
- Exhaust server memory (JSON parsing buffers the whole body)
- Trigger garbage collection pauses
- Block the event loop during parsing

## Client Feedback

Return `413 Payload Too Large` with a helpful message:

```json
{
  "error": {
    "code": "PAYLOAD_TOO_LARGE",
    "message": "Request body exceeds 100KB limit. For bulk operations, use POST /bulk/import."
  }
}
```
