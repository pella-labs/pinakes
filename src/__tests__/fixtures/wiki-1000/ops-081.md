# Kubernetes Service Types

## ClusterIP

The default type. Creates an internal IP only reachable within the cluster. Use for service-to-service communication.

## NodePort

Exposes the service on a static port on every node's IP. Range: 30000-32767. Rarely used directly in production — use an Ingress or LoadBalancer instead.

## LoadBalancer

Provisions a cloud load balancer (ALB/NLB on AWS). Each LoadBalancer service gets its own LB, which can be expensive. Prefer Ingress for HTTP services.

## ExternalName

Maps a service to a DNS name. Useful for pointing to external databases or services outside the cluster:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: external-db
spec:
  type: ExternalName
  externalName: mydb.rds.amazonaws.com
```

## Headless Services

Set `clusterIP: None` to create a **headless service**. Instead of a virtual IP, DNS returns the pod IPs directly. Used with StatefulSets for stable DNS names per pod.
