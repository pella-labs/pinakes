# API Logging Strategy

## What to Log

Every request:
- Request ID
- Method, path, query params
- Authenticated user ID (not the token itself)
- Response status code
- Duration in ms
- Response size in bytes

Selectively:
- Request body for mutating endpoints (scrub PII)
- Response body for debugging (never in production by default)

## Structured Logging

Use JSON, never free-form text.

```json
{
  "level": "info",
  "ts": "2025-11-03T14:30:00.123Z",
  "requestId": "req_abc",
  "method": "POST",
  "path": "/orders",
  "userId": "usr_123",
  "status": 201,
  "duration_ms": 45,
  "bytes": 892
}
```

## Log Levels

- **error** — unhandled exceptions, failed health checks
- **warn** — deprecated endpoint hit, rate limit approached
- **info** — every request/response (the default)
- **debug** — request/response bodies, SQL queries (never in prod)

## PII Scrubbing

Never log: passwords, tokens, credit card numbers, SSNs, full email addresses. Use a scrubber middleware that redacts known sensitive fields before the log line is emitted.
