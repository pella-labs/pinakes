# Streaming APIs

## HTTP Streaming

Keep the connection open, flush chunks as they become available. Uses `Transfer-Encoding: chunked`.

```typescript
app.get('/stream/logs', (req, res) => {
  res.setHeader('Content-Type', 'application/x-ndjson');
  res.setHeader('Transfer-Encoding', 'chunked');

  const stream = logService.tail();
  stream.on('data', (entry) => {
    res.write(JSON.stringify(entry) + '\n');
  });
  stream.on('end', () => res.end());
  req.on('close', () => stream.destroy());
});
```

## NDJSON (Newline-Delimited JSON)

One JSON object per line. Easy to parse incrementally:

```
{"id":1,"event":"start","ts":"2025-11-03T10:00:00Z"}
{"id":2,"event":"progress","ts":"2025-11-03T10:00:01Z","pct":45}
{"id":3,"event":"complete","ts":"2025-11-03T10:00:03Z"}
```

## gRPC Server Streaming

```protobuf
rpc StreamLogs(StreamLogsRequest) returns (stream LogEntry);
```

gRPC handles backpressure natively via HTTP/2 flow control. See [[api-009]] for protobuf design.

## Backpressure

If the server produces faster than the client consumes, you need backpressure. Node streams handle this with `highWaterMark`. For HTTP, use `res.write()` return value — if it returns `false`, pause until the `drain` event.
