# API Design Review Checklist

Quick checklist before shipping any new endpoint.

- [ ] Resource names are plural nouns, kebab-case
- [ ] HTTP methods match semantics (GET=read, POST=create, PUT=replace, PATCH=update, DELETE=remove)
- [ ] Response includes `request_id` for tracing
- [ ] Error responses follow the standard envelope
- [ ] Pagination is cursor-based with `has_more` flag
- [ ] Rate limit headers present on all responses
- [ ] Auth required (unless explicitly public)
- [ ] Input validated with schema (Zod or JSON Schema)
- [ ] OpenAPI spec updated
- [ ] Changelog entry added
- [ ] No breaking changes to existing clients
- [ ] Idempotency key supported for mutating endpoints
- [ ] Response size bounded (no unbounded arrays)
- [ ] Sensitive data excluded from logs
- [ ] Integration test covers happy path + top 3 error cases
