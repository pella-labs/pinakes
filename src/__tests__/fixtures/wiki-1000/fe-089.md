# React Profiler and Performance Debugging

## React DevTools Profiler

The **Profiler** tab in React DevTools records renders and shows:

- Which components rendered
- How long each render took
- Why each component rendered (props changed, state changed, parent re-rendered)

## Programmatic Profiler

```tsx
<Profiler id="sidebar" onRender={onRenderCallback}>
  <Sidebar />
</Profiler>

function onRenderCallback(
  id: string,
  phase: 'mount' | 'update',
  actualDuration: number,
  baseDuration: number,
  startTime: number,
  commitTime: number
) {
  if (actualDuration > 16) {
    console.warn(`Slow render: ${id} took ${actualDuration.toFixed(1)}ms`);
  }
}
```

## Chrome DevTools Performance Tab

For deeper analysis, use Chrome's Performance tab to:
- Identify long tasks blocking the main thread
- Analyze layout thrashing
- Profile memory usage
- View a flame chart of JavaScript execution

## Common Performance Issues

- Large component trees re-rendering unnecessarily
- Expensive calculations in render without memoization
- Frequent state updates causing layout thrashing
- Unoptimized images and fonts blocking rendering

## Performance Testing in CI

Use Lighthouse CI to catch regressions automatically. Set thresholds for LCP, CLS, and Total Blocking Time. See [[fe-009]] for metrics and [[fe-054]] for optimization strategies.
