# SQLite FTS5

## Full-Text Search in SQLite

**FTS5** is SQLite's latest full-text search extension. It creates a virtual table backed by an inverted index, enabling fast text search across large document collections.

## Creating an FTS5 Table

```sql
CREATE VIRTUAL TABLE documents_fts USING fts5(
  title,
  body,
  tokenize='unicode61 remove_diacritics 2'
);

-- Insert data
INSERT INTO documents_fts (title, body) VALUES
  ('Database Indexing', 'B-tree indexes are the default in most databases...'),
  ('Query Optimization', 'Start with EXPLAIN ANALYZE to understand...');
```

## Querying

```sql
-- Basic search
SELECT * FROM documents_fts WHERE documents_fts MATCH 'indexing';

-- Phrase search
SELECT * FROM documents_fts WHERE documents_fts MATCH '"query optimization"';

-- Boolean operators
SELECT * FROM documents_fts WHERE documents_fts MATCH 'database AND NOT mongodb';

-- BM25 ranking
SELECT *, bm25(documents_fts) AS rank
FROM documents_fts
WHERE documents_fts MATCH 'indexing strategies'
ORDER BY rank;
```

## Tokenizer Choice

The tokenizer choice matters enormously:
- `unicode61` — default, good for most languages
- `porter` — English stemming (search → search, searching, searches)
- `trigram` — character trigrams, supports substring matching but 3x storage

Never use the trigram tokenizer for large datasets — it triples the DB size. See [[db-003]] for SQLite configuration pragmas.
