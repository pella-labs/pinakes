# Vector Databases and Embeddings

## What Are Vector Databases?

A **vector database** stores high-dimensional vectors and enables similarity search. Instead of exact matches, you find the K nearest neighbors to a query vector. This powers semantic search, recommendation engines, and RAG pipelines.

## Embedding Models

Text is converted to vectors using embedding models:
- **all-MiniLM-L6-v2** — 384 dimensions, fast, good for general text
- **nomic-embed-text** — 768 dimensions, strong code understanding
- **voyage-code-3** — optimized for code search

## Database Options

### Dedicated Vector Databases
- **Pinecone** — fully managed, serverless
- **Weaviate** — open-source, hybrid search
- **Qdrant** — Rust-based, high performance
- **Milvus** — distributed, GPU-accelerated

### Vector Extensions for Existing Databases
- **pgvector** for PostgreSQL — see [[db-029]]
- **sqlite-vec** for SQLite — lightweight, embedded
- **MongoDB Atlas Vector Search** — built into Atlas

## Hybrid Search

Combining vector similarity with keyword search (BM25) via **Reciprocal Rank Fusion** produces better results than either alone. Query the FTS index and vector index separately, then merge rankings.

See [[db-088]] for implementing vector search with pgvector.
