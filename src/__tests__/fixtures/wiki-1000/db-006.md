# Elasticsearch Fundamentals

## Inverted Index

At its core, Elasticsearch is built on **Apache Lucene** and uses an inverted index to map terms to the documents that contain them. This is what makes full-text search fast — instead of scanning every document, you look up the term and get a list of matching document IDs.

## Mappings and Analyzers

Mappings define how fields are indexed. An **analyzer** is a pipeline of character filters, a tokenizer, and token filters. Choosing the right analyzer is critical:

```json
{
  "mappings": {
    "properties": {
      "title": {
        "type": "text",
        "analyzer": "english"
      },
      "slug": {
        "type": "keyword"
      },
      "created_at": {
        "type": "date"
      }
    }
  }
}
```

## Shards and Replicas

An Elasticsearch index is split into **shards** distributed across nodes. Each shard is a self-contained Lucene index. Replicas provide redundancy and read throughput. Over-sharding is a common mistake — see [[db-024]] for partitioning anti-patterns.

## Relevance Scoring

Elasticsearch uses **BM25** by default for scoring. Understanding TF-IDF and BM25 helps you tune relevance. Function score queries, boosting, and custom scoring scripts let you blend text relevance with business logic.
