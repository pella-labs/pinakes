# AWS CloudTrail

## Purpose

**CloudTrail** records every API call made in your AWS account. It's the audit log for your entire cloud infrastructure.

## Configuration

```hcl
resource "aws_cloudtrail" "main" {
  name                       = "organization-trail"
  s3_bucket_name             = aws_s3_bucket.cloudtrail.id
  is_organization_trail      = true
  is_multi_region_trail      = true
  enable_log_file_validation = true
  kms_key_id                 = aws_kms_key.cloudtrail.arn

  event_selector {
    read_write_type           = "All"
    include_management_events = true

    data_resource {
      type   = "AWS::S3::Object"
      values = ["arn:aws:s3"]
    }
  }
}
```

## Investigating Events

Query CloudTrail events with Athena:

```sql
SELECT eventTime, eventName, userIdentity.arn, sourceIPAddress
FROM cloudtrail_logs
WHERE eventName = 'DeleteBucket'
AND eventTime > '2025-12-01'
ORDER BY eventTime DESC;
```

## Use Cases

- Security incident investigation
- Compliance auditing
- Detecting unauthorized API calls
- Understanding who changed what and when
- Triggering automated responses via EventBridge

## Retention

CloudTrail events are stored in S3 with lifecycle policies. Keep 90 days hot, 1 year in Standard-IA, 7 years in Glacier for compliance.
