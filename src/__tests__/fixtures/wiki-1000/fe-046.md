# Server Actions in Next.js

## What Are Server Actions

**Server Actions** are async functions that run on the server, invoked directly from client components. They replace API routes for mutations.

```tsx
'use server';

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string;
  const body = formData.get('body') as string;

  await db.post.create({ data: { title, body } });
  revalidatePath('/posts');
}
```

## Usage in Forms

```tsx
function NewPostForm() {
  return (
    <form action={createPost}>
      <input name="title" required />
      <textarea name="body" required />
      <button type="submit">Create</button>
    </form>
  );
}
```

## Progressive Enhancement

Forms using Server Actions work without JavaScript. The form submits as a standard POST request if JS hasn't loaded yet. With JS, the submission is intercepted for a smoother experience.

## Validation

Always validate on the server. Client-side validation is a UX enhancement, not a security measure. Use Zod or a similar library:

```typescript
'use server';
import { z } from 'zod';

const schema = z.object({
  title: z.string().min(1).max(200),
  body: z.string().min(1),
});

export async function createPost(formData: FormData) {
  const result = schema.safeParse(Object.fromEntries(formData));
  if (!result.success) return { errors: result.error.flatten() };
  // ...
}
```
