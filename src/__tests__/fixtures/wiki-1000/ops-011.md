# Load Balancers

## Types of Load Balancers

**Load balancers** distribute incoming traffic across multiple targets to improve availability and fault tolerance.

### Application Load Balancer (ALB)

Operates at **Layer 7** (HTTP/HTTPS). Supports path-based routing, host-based routing, and WebSocket connections. Best for web applications and microservices.

### Network Load Balancer (NLB)

Operates at **Layer 4** (TCP/UDP). Ultra-low latency and handles millions of requests per second. Best for TCP-heavy workloads, gRPC, and anything that needs static IPs.

### Gateway Load Balancer (GWLB)

Routes traffic through third-party virtual appliances like firewalls and IDS/IPS. Used in network security architectures.

## Health Checks

Configure health checks carefully:

- Use a dedicated `/healthz` endpoint
- Check downstream dependencies (DB, cache) in a `/readyz` endpoint
- Set reasonable thresholds: 3 consecutive failures before marking unhealthy
- Use shorter intervals for critical services (10s) and longer for batch (30s)

## TLS Termination

Terminate TLS at the load balancer using **ACM certificates**. This offloads crypto from your application servers and centralizes certificate management. Internal traffic between the LB and targets can use HTTP if the VPC is trusted, or mutual TLS for zero-trust.
