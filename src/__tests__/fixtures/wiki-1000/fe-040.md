# Virtual Scrolling and Large Lists

## The Problem

Rendering thousands of DOM nodes causes jank and high memory usage. A list of 10,000 items creates 10,000 DOM elements even though only ~20 are visible at any time.

## Virtual Scrolling

**Virtual scrolling** (windowing) renders only the visible items plus a small overscan buffer. As the user scrolls, items are recycled—old ones are destroyed, new ones are created.

## TanStack Virtual

```tsx
import { useVirtualizer } from '@tanstack/react-virtual';

function VirtualList({ items }: { items: Item[] }) {
  const parentRef = useRef<HTMLDivElement>(null);

  const virtualizer = useVirtualizer({
    count: items.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 48,
    overscan: 5,
  });

  return (
    <div ref={parentRef} style={{ height: '600px', overflow: 'auto' }}>
      <div style={{ height: virtualizer.getTotalSize() }}>
        {virtualizer.getVirtualItems().map((virtual) => (
          <div
            key={virtual.key}
            style={{
              position: 'absolute',
              top: virtual.start,
              height: virtual.size,
              width: '100%',
            }}
          >
            <ItemRow item={items[virtual.index]} />
          </div>
        ))}
      </div>
    </div>
  );
}
```

## Variable Height Items

When items have different heights, provide a `measureElement` callback. TanStack Virtual measures items dynamically and adjusts positions accordingly.

## When Not to Virtualize

Virtualization adds complexity. If your list has fewer than 100 items, just render them all. If the items are simple (text only), you can handle even 500 without virtualization.
