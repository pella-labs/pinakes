# SQL Joins Explained

## Join Types

### INNER JOIN
Returns only rows that match in both tables.

### LEFT (OUTER) JOIN
Returns all rows from the left table, with NULLs for unmatched right-side rows.

### RIGHT (OUTER) JOIN
Opposite of LEFT JOIN. Rarely used — just swap the table order and use LEFT JOIN.

### FULL (OUTER) JOIN
Returns all rows from both tables. NULLs where there's no match.

### CROSS JOIN
Cartesian product of both tables. Every row paired with every other row. Useful for generating combinations, dangerous if accidental.

## Visual Example

```sql
-- Find customers with no orders
SELECT c.name
FROM customers c
LEFT JOIN orders o ON o.customer_id = c.id
WHERE o.id IS NULL;

-- Equivalent using NOT EXISTS (often faster)
SELECT c.name
FROM customers c
WHERE NOT EXISTS (
  SELECT 1 FROM orders o WHERE o.customer_id = c.id
);
```

## Join Performance

- Index the join columns (especially foreign keys)
- The planner chooses between nested loop, hash join, and merge join
- For large tables, hash joins are common and need `work_mem`
- Use EXPLAIN to verify the join strategy (see [[db-009]])
