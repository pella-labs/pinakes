# Infrastructure Testing

## Why Test Infrastructure

**Infrastructure code** should be tested just like application code. A misconfigured security group or an overly permissive IAM policy can have catastrophic consequences.

## Testing Levels

### Static Analysis

Lint and validate before applying:

```bash
# Terraform
terraform validate
tflint
tfsec

# Kubernetes
kubeval manifests/
kube-score score manifests/
```

### Policy as Code

Use **OPA/Rego** or **Kyverno** to enforce policies:

- All S3 buckets must have encryption enabled
- No public ingress rules in security groups
- All pods must have resource requests and limits
- Container images must come from approved registries

### Integration Tests

**Terratest** (Go) lets you deploy infrastructure, validate it, and tear it down:

```go
func TestVPC(t *testing.T) {
    terraformOptions := &terraform.Options{
        TerraformDir: "../modules/vpc",
        Vars: map[string]interface{}{
            "cidr_block": "10.0.0.0/16",
        },
    }
    defer terraform.Destroy(t, terraformOptions)
    terraform.InitAndApply(t, terraformOptions)

    vpcId := terraform.Output(t, terraformOptions, "vpc_id")
    assert.NotEmpty(t, vpcId)
}
```

## Cost of Not Testing

An untested Terraform change once opened all security groups to 0.0.0.0/0 in production. That incident drove the adoption of `tfsec` and policy-as-code. Test your infrastructure.
