# API Mocking for Development

## Mock Server from OpenAPI

Generate a mock server directly from the spec:

```bash
prism mock openapi.yaml --port 4010
```

Returns example data from the spec. Validates requests against the schema. Frontend team can build against it before the backend is ready.

## WireMock for Integration Tests

```json
{
  "request": {
    "method": "GET",
    "urlPattern": "/orders/ord_.*"
  },
  "response": {
    "status": 200,
    "jsonBody": {
      "id": "ord_mock",
      "status": "pending",
      "total": 999
    },
    "headers": {
      "Content-Type": "application/json"
    }
  }
}
```

## MSW for Frontend Tests

Mock Service Worker intercepts at the network level:

```typescript
import { rest } from 'msw';

const handlers = [
  rest.get('/api/orders', (req, res, ctx) => {
    return res(
      ctx.json({ data: [{ id: 'ord_1', status: 'pending' }] })
    );
  }),
];
```

## Rules

- Mock servers must validate requests against the schema
- Update mocks when the spec changes (automate this)
- Never mock in production (obviously)
- Contract tests ([[api-050]]) catch drift between mocks and reality
