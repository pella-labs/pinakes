# Database Anti-Patterns

## The God Table

A single table with 50+ columns that tries to represent every entity in the system. Split it into normalized tables with clear relationships.

## SELECT * in Production

Always specify the columns you need. `SELECT *` prevents index-only scans, wastes bandwidth, and breaks when columns are added.

## Missing Indexes on Foreign Keys

Every foreign key column should have an index. Without one, JOINs and CASCADE deletes trigger full table scans.

## Storing IP Addresses as Strings

Use PostgreSQL's `inet` type or store as integers. Strings waste space and prevent range queries.

## Premature Denormalization

Denormalize based on measured query patterns, not speculation. Start normalized (see [[db-007]]) and denormalize when you have evidence.

## UUID Primary Keys Without Consideration

Random UUIDs cause B-tree index fragmentation because inserts are scattered. Use UUIDv7 (time-ordered) or ULID instead. Or just use `BIGSERIAL` — it's fine for most applications.

## Not Using Transactions

Every multi-statement operation that should be atomic needs explicit transactions. "It usually works" is not a concurrency strategy. See [[db-011]] for transaction fundamentals.

## Ignoring EXPLAIN

If you haven't looked at the query plan, you don't know if your query is fast. It might be doing a sequential scan on a million-row table. See [[db-009]].
