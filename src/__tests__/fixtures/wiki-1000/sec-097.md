# Security Onboarding Checklist

A structured **security onboarding** ensures new team members understand and can apply the organization's security practices from day one.

## Account Setup

- [ ] MFA enabled on all accounts (see [[sec-056]])
- [ ] SSH keys generated and registered
- [ ] GPG key generated for commit signing
- [ ] VPN access configured
- [ ] Password manager provisioned

## Access Provisioning

- [ ] Granted minimum necessary repository access
- [ ] Added to appropriate security groups
- [ ] Database access provisioned (read-only initially)
- [ ] Cloud IAM roles assigned (see [[sec-089]])
- [ ] Secrets management access configured (see [[sec-015]])

## Training

- [ ] Security awareness training completed
- [ ] Incident response process reviewed (see [[sec-054]])
- [ ] Secure coding guidelines reviewed (see [[sec-050]])
- [ ] Data classification policy reviewed (see [[sec-076]])
- [ ] Compliance requirements briefed (SOC2, GDPR as applicable)

## Tools

- [ ] Static analysis tooling configured locally
- [ ] Pre-commit hooks installed (secret scanning)
- [ ] Security testing tools available
- [ ] Access to security dashboards and alerts

## Review

- [ ] First PR reviewed with security focus by a senior engineer
- [ ] Shadow an on-call shift within first month
- [ ] Complete a security-focused code review within first two weeks

The same checklist should be run in reverse during offboarding — revoke all access promptly.
