# Microservice Communication Patterns

## Synchronous

- **REST** — simple, cacheable, wide tooling. See [[api-001]].
- **gRPC** — fast, typed, streaming. See [[api-009]].
- **GraphQL Federation** — unified graph across services.

## Asynchronous

- **Message queue** (RabbitMQ, SQS) — work distribution, exactly-once with dedup
- **Event stream** (Kafka, Kinesis) — event log, multiple consumers, replay
- **Pub/Sub** (Redis, NATS) — fire-and-forget fanout

## When to Use Which

| Pattern | Use When |
|---|---|
| Sync REST/gRPC | Caller needs response to continue |
| Async queue | Work can be deferred, needs guaranteed delivery |
| Event stream | Multiple consumers need the same event, audit trail needed |
| Pub/Sub | Real-time fanout, OK to miss events |

## Saga Pattern

For distributed transactions across services, use a **saga** — a sequence of local transactions with compensating actions on failure.

```
Order created → Payment charged → Inventory reserved → Shipping scheduled
                 ↑ if fails        ↑ if fails           ↑ if fails
                 (no-op)           (refund payment)     (release inventory + refund)
```

Each step publishes an event. The next step subscribes. Failure triggers compensating events in reverse order.
