# Kubernetes Operators

## What is an Operator

A Kubernetes **Operator** extends the API with custom resources and controllers that encode operational knowledge. Instead of writing runbooks, you encode them in code.

## Common Operators

- **cert-manager** — TLS certificate lifecycle
- **External Secrets Operator** — syncs secrets from external stores
- **Prometheus Operator** — manages Prometheus instances
- **Strimzi** — Apache Kafka on Kubernetes
- **CloudNativePG** — PostgreSQL on Kubernetes

## Custom Operator Development

Build custom operators when you have application-specific operational logic. Use frameworks like **Kubebuilder** or **Operator SDK**:

```go
func (r *MyAppReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
    var myapp v1.MyApp
    if err := r.Get(ctx, req.NamespacedName, &myapp); err != nil {
        return ctrl.Result{}, client.IgnoreNotFound(err)
    }
    // Reconciliation logic here
    return ctrl.Result{RequeueAfter: 30 * time.Second}, nil
}
```

## When Not to Use Operators

Don't build an operator for simple configuration management. Helm or Kustomize is sufficient for most services. Build operators for stateful applications with complex lifecycle management.
