# Webhook Design

Webhooks are the reverse API. You send HTTP callbacks to subscribers when events occur.

## Event Envelope

```json
{
  "id": "evt_abc123",
  "type": "order.completed",
  "created_at": "2025-11-03T14:30:00Z",
  "data": {
    "order_id": "ord_xyz",
    "total": 4999,
    "currency": "usd"
  },
  "webhook_id": "wh_001"
}
```

## Delivery Guarantees

Aim for **at-least-once** delivery. The subscriber must be idempotent (use the event `id` for dedup). See [[api-007]] for idempotency patterns.

## Retry Policy

Exponential backoff: 1s, 5s, 30s, 5m, 30m, 2h, 12h. After 7 failures, mark the endpoint as **disabled** and notify the subscriber via email.

## Signature Verification

Sign every payload with HMAC-SHA256 using a per-subscriber secret:

```typescript
const signature = crypto
  .createHmac('sha256', webhookSecret)
  .update(rawBody)
  .digest('hex');

// Send as: X-Webhook-Signature: sha256=<signature>
```

The subscriber verifies by recomputing the HMAC. Use **constant-time comparison** to prevent timing attacks.

## Subscriber Registration

- `POST /webhooks` — register a URL + events
- `GET /webhooks` — list subscriptions
- `DELETE /webhooks/{id}` — unsubscribe
- `POST /webhooks/{id}/test` — send a test event
