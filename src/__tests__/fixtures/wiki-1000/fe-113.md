# Form Edge Cases and Solutions

## File Upload Handling

File inputs are always uncontrolled. Handle them with refs or FormData:

```tsx
function FileUpload() {
  async function handleSubmit(formData: FormData) {
    const file = formData.get('document') as File;
    if (file.size > 5 * 1024 * 1024) {
      setError('File too large (max 5MB)');
      return;
    }
    await uploadFile(file);
  }

  return (
    <form action={handleSubmit}>
      <input type="file" name="document" accept=".pdf,.doc,.docx" />
      <button type="submit">Upload</button>
    </form>
  );
}
```

## Multi-Step Form State

Persist state across steps without losing data on back navigation. Use a parent state container or URL search params:

```typescript
const [formData, setFormData] = useState({
  step1: {},
  step2: {},
  step3: {},
});

function updateStep(step: string, data: object) {
  setFormData((prev) => ({ ...prev, [step]: data }));
}
```

## Debounced Validation

For async validation (checking username availability), debounce to avoid hammering the server:

```typescript
const debouncedCheck = useMemo(
  () => debounce(async (username: string) => {
    const available = await checkUsername(username);
    setFieldError('username', available ? null : 'Username taken');
  }, 300),
  []
);
```

## Form Reset Pitfalls

After submission, `form.reset()` clears native inputs but not React state. Explicitly reset state or use `key` to remount the form component.
