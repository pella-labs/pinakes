# MongoDB Indexes

## Index Types

### Single Field
```javascript
db.users.createIndex({ email: 1 });  // ascending
```

### Compound
```javascript
db.orders.createIndex({ status: 1, createdAt: -1 });
```

### Text
```javascript
db.articles.createIndex({ title: "text", body: "text" });
db.articles.find({ $text: { $search: "database optimization" } });
```

### Geospatial
```javascript
db.locations.createIndex({ coordinates: "2dsphere" });
```

### Wildcard
```javascript
db.products.createIndex({ "metadata.$**": 1 });
```

## Index Properties

- **Unique**: `{ unique: true }`
- **Sparse**: only index documents that have the field
- **TTL**: auto-delete documents after a time period
- **Partial**: index a subset of documents matching a filter

## The explain() Method

```javascript
db.orders.find({ status: "active" }).explain("executionStats");
// Check: totalDocsExamined vs nReturned
// If they differ significantly, you need a better index
```

## Covered Queries

If the index contains all fields needed by the query, MongoDB returns results directly from the index without reading documents. This is called a **covered query** and is the fastest possible execution path.

For MongoDB data modeling that informs index design, see [[db-005]].
