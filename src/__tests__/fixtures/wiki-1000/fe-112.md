# Responsive Tables

## The Challenge

Data tables are inherently wide. On mobile screens, horizontal scrolling is a poor experience.

## Strategies

### Horizontal Scroll

The simplest approach. Wrap the table in an overflow container:

```css
.table-container {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
}
```

### Stacked Cards

On mobile, transform each row into a card:

```css
@media (max-width: 768px) {
  table, thead, tbody, th, td, tr {
    display: block;
  }

  thead { display: none; }

  td {
    display: flex;
    justify-content: space-between;
    padding: 8px 16px;
  }

  td::before {
    content: attr(data-label);
    font-weight: 600;
  }
}
```

### Column Priority

Hide less important columns on smaller screens:

```css
.col-priority-low {
  display: none;
}

@media (min-width: 768px) {
  .col-priority-low { display: table-cell; }
}
```

### Expandable Rows

Show key columns on mobile with an expand button to reveal the full row data.

## Accessibility

Always include proper `<thead>`, `<th scope="col">`, and `<caption>` elements regardless of the responsive strategy.
