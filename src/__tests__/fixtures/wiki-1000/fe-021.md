# Incremental Static Regeneration (ISR)

## The Middle Ground

**ISR** combines the speed of static generation with the freshness of server rendering. Pages are generated at build time and revalidated on a configurable schedule.

## How It Works

1. First request serves the statically generated page
2. After the `revalidate` period expires, the next request triggers background regeneration
3. The stale page is served until the new version is ready
4. Subsequent requests receive the updated page

This is the **stale-while-revalidate** pattern applied to full pages.

## On-Demand Revalidation

Instead of time-based revalidation, trigger regeneration programmatically when content changes:

```typescript
// app/api/revalidate/route.ts
import { revalidatePath, revalidateTag } from 'next/cache';

export async function POST(request: Request) {
  const { path, tag } = await request.json();
  if (tag) revalidateTag(tag);
  if (path) revalidatePath(path);
  return Response.json({ revalidated: true });
}
```

## When to Use ISR

ISR is ideal for content that changes infrequently but cannot be fully static: product catalogs, blog posts, documentation sites. For real-time data, use SSR or client-side fetching instead. See [[fe-019]] and [[fe-020]] for alternatives.
