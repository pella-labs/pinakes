# REST API Bulk Delete

## Design Options

### Option A: DELETE with Body

```http
DELETE /users
Content-Type: application/json

{ "ids": ["u_001", "u_002", "u_003"] }
```

Technically allowed by HTTP spec but poorly supported by some clients and proxies.

### Option B: POST to Action Endpoint

```http
POST /users/bulk-delete
Content-Type: application/json

{ "ids": ["u_001", "u_002", "u_003"] }
```

More compatible. Clear intent. Easier to audit.

### Option C: Query Parameter

```http
DELETE /users?ids=u_001,u_002,u_003
```

Works for small sets but URL length limits (2048 chars) cap it.

## Our Choice

Option B. POST to a clearly named action endpoint. It works everywhere and the audit log captures the full request body.

## Response

```json
{
  "deleted": 2,
  "errors": [
    { "id": "u_003", "error": "Not found" }
  ]
}
```

Return `200` with per-item results, not `204`. The caller needs to know what succeeded and what didn't.
