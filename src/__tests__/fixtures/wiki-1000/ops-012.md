# DNS Management

## Route 53 Configuration

We use **Route 53** for DNS management. All DNS records are managed through Terraform to prevent configuration drift.

```hcl
resource "aws_route53_zone" "main" {
  name = "example.com"
}

resource "aws_route53_record" "api" {
  zone_id = aws_route53_zone.main.zone_id
  name    = "api.example.com"
  type    = "A"

  alias {
    name                   = aws_lb.api.dns_name
    zone_id                = aws_lb.api.zone_id
    evaluate_target_health = true
  }
}
```

## DNS Patterns

- **CNAME** for pointing subdomains to other hostnames
- **A/AAAA alias** for pointing to AWS resources (free, unlike CNAME at zone apex)
- **Weighted routing** for gradual traffic shifts between regions
- **Failover routing** for DR scenarios with health-check-based failover
- **Latency-based routing** for multi-region deployments

## TTL Strategy

Use low TTLs (60s) for records that might change during deploys. Use higher TTLs (300-3600s) for stable records to reduce DNS query volume and improve client performance.
