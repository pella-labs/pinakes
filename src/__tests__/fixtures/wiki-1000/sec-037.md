# Password Hashing

Storing passwords securely requires **one-way hashing** with algorithms specifically designed for password storage. General-purpose hash functions like SHA-256 are not suitable.

## Algorithm Selection

Use one of these purpose-built algorithms:

- **Argon2id**: the current recommendation (winner of Password Hashing Competition)
- **bcrypt**: well-established, widely supported, but limited to 72-byte input
- **scrypt**: memory-hard, good alternative to Argon2id

Never use MD5, SHA-1, SHA-256, or any non-password-specific hash.

## Argon2id Configuration

```typescript
import argon2 from 'argon2';

const hash = await argon2.hash(password, {
  type: argon2.argon2id,
  memoryCost: 65536,  // 64 MB
  timeCost: 3,        // 3 iterations
  parallelism: 4,     // 4 threads
});

const isValid = await argon2.verify(hash, password);
```

## Salt Management

All modern password hashing libraries handle salting automatically. The salt is stored as part of the hash output. Never implement manual salting unless you're using a legacy system.

## Migration Strategy

If your system currently uses weak hashing:

1. On next login, verify against old hash
2. If valid, rehash with the new algorithm
3. Store the new hash
4. After sufficient time, require password reset for remaining accounts
