# Database Connection Timeout Strategies

## Types of Timeouts

### Connection Timeout
How long to wait when establishing a new connection:

```typescript
const pool = new Pool({
  connectionTimeoutMillis: 5000,  // 5 second connection timeout
});
```

### Query Timeout
Maximum execution time for a single query:

```sql
-- PostgreSQL: per-session
SET statement_timeout = '30s';

-- Per-transaction
SET LOCAL statement_timeout = '10s';
```

### Idle Timeout
How long an idle connection stays in the pool:

```typescript
const pool = new Pool({
  idleTimeoutMillis: 30000,  // close idle connections after 30s
});
```

## Setting Appropriate Values

| Context | Connection | Query | Idle |
|---------|-----------|-------|------|
| Web API | 3-5s | 10-30s | 30-60s |
| Background job | 10-30s | 5-60min | 5min |
| Migration | 30-60s | no limit | 5min |

## Handling Timeouts

Never ignore a timeout error. Log it, return an appropriate error to the user, and investigate if it happens frequently. A spike in timeouts often indicates a missing index, a lock contention issue ([[db-032]]), or resource exhaustion.
