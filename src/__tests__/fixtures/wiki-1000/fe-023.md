# Design Tokens Architecture

## What Are Design Tokens

**Design tokens** are the atomic values of a design system: colors, spacing, typography, shadows, and motion. They bridge design tools and code, ensuring consistency across platforms.

## Token Hierarchy

Structure tokens in three tiers:

- **Global tokens**: raw values (`blue-500: #3B82F6`, `spacing-4: 16px`)
- **Alias tokens**: semantic names (`color-primary: blue-500`, `spacing-md: spacing-4`)
- **Component tokens**: scoped to a component (`button-padding: spacing-md`)

## Implementation

```typescript
// tokens.ts
export const tokens = {
  color: {
    primary: '#3B82F6',
    primaryHover: '#2563EB',
    secondary: '#6B7280',
    error: '#EF4444',
    success: '#10B981',
    surface: '#FFFFFF',
    background: '#F9FAFB',
    text: '#111827',
    textMuted: '#6B7280',
  },
  spacing: {
    xs: '4px',
    sm: '8px',
    md: '16px',
    lg: '24px',
    xl: '32px',
  },
  radius: {
    sm: '4px',
    md: '8px',
    lg: '12px',
    full: '9999px',
  },
} as const;
```

## Multi-Platform Delivery

Use **Style Dictionary** or **Tokens Studio** to transform tokens into platform-specific formats: CSS custom properties, Tailwind config, iOS Swift constants, and Android XML resources.
