# REST API Sorting

## Query Parameter Convention

```
GET /orders?sort=created_at:desc,total:asc
```

Comma-separated field:direction pairs. Default direction is ascending.

## Alternative Syntaxes

```
# JSON-API style
GET /orders?sort=-created_at,total

# Prefix style (- for desc)
GET /orders?order_by=created_at&order_dir=desc
```

## Implementation

```typescript
function parseSortParam(sort: string, allowed: string[]): Array<{ field: string; dir: 'asc' | 'desc' }> {
  return sort.split(',').map(part => {
    const [field, dir = 'asc'] = part.trim().split(':');
    if (!allowed.includes(field)) throw new ValidationError(`Cannot sort by: ${field}`);
    if (!['asc', 'desc'].includes(dir)) throw new ValidationError(`Invalid direction: ${dir}`);
    return { field, dir: dir as 'asc' | 'desc' };
  });
}
```

## Rules

- Only allow sorting by **indexed columns** (unindexed sort = full table scan)
- Limit to 3 sort fields maximum
- Always include a **tiebreaker** (usually `id`) as the last sort field for deterministic pagination
- Document which fields are sortable in the OpenAPI spec

See [[api-005]] for how sorting interacts with cursor-based pagination.
