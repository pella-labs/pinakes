# Shadow DOM and Web Components

## Web Components Standards

**Web Components** consist of three specifications:

- **Custom Elements**: define new HTML elements
- **Shadow DOM**: encapsulated DOM tree with scoped styles
- **HTML Templates**: reusable markup templates

## Creating a Custom Element

```typescript
class MyCounter extends HTMLElement {
  private count = 0;
  private shadow: ShadowRoot;

  constructor() {
    super();
    this.shadow = this.attachShadow({ mode: 'open' });
    this.render();
  }

  private render() {
    this.shadow.innerHTML = `
      <style>
        button { padding: 8px 16px; font-size: 16px; }
        span { margin: 0 8px; }
      </style>
      <button id="dec">-</button>
      <span>${this.count}</span>
      <button id="inc">+</button>
    `;

    this.shadow.getElementById('inc')!.onclick = () => {
      this.count++;
      this.render();
    };
    this.shadow.getElementById('dec')!.onclick = () => {
      this.count--;
      this.render();
    };
  }
}

customElements.define('my-counter', MyCounter);
```

## Shadow DOM Encapsulation

Styles inside shadow DOM do not leak out, and external styles do not leak in. This provides true CSS isolation—unlike CSS Modules or BEM, which are conventions that can be broken.

## Interop with Frameworks

React requires special handling for web component events and properties. Libraries like **Lit** simplify web component creation and provide better framework interop.

## When to Use

Web components shine for framework-agnostic widgets: embeddable components, micro-frontends, design system primitives shared across React/Vue/Angular teams.
