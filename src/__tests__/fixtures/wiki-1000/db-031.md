# Drizzle ORM Patterns

## Why Drizzle

**Drizzle ORM** takes a SQL-first approach. Instead of abstracting SQL away, it gives you a type-safe query builder that maps closely to the SQL you'd write by hand. Zero overhead at runtime — queries compile to exactly the SQL you'd expect.

## Schema Definition

```typescript
import { sqliteTable, text, integer, real } from 'drizzle-orm/sqlite-core';

export const projects = sqliteTable('projects', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  description: text('description'),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
});

export const tasks = sqliteTable('tasks', {
  id: text('id').primaryKey(),
  projectId: text('project_id').notNull().references(() => projects.id),
  title: text('title').notNull(),
  priority: integer('priority').default(0),
});
```

## Querying

```typescript
// Select with joins
const result = await db
  .select({
    projectName: projects.name,
    taskTitle: tasks.title,
  })
  .from(projects)
  .innerJoin(tasks, eq(tasks.projectId, projects.id))
  .where(eq(projects.id, projectId));
```
