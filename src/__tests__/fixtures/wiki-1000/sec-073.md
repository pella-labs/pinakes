# Security Monitoring Metrics

Tracking the right **security metrics** helps teams measure their security posture and identify trends before they become incidents.

## Operational Metrics

- **Mean time to detect (MTTD)**: how quickly are security events identified?
- **Mean time to respond (MTTR)**: how quickly are incidents contained?
- **Patch latency**: time between vulnerability disclosure and deployment of fix
- **Failed authentication rate**: baseline and anomalies
- **Certificate expiration countdown**: days until next cert expires

## Vulnerability Metrics

- Open vulnerability count by severity
- Vulnerability age (time since discovery)
- Vulnerability reopen rate
- Dependency vulnerability ratio (vulnerable deps / total deps)

## Access Metrics

- Number of active service accounts
- Stale account count (no login in 90 days)
- Overprivileged account count
- MFA adoption rate (see [[sec-056]])
- API key age distribution

## Compliance Metrics

- Control effectiveness (passing controls / total controls)
- Audit finding closure rate
- Policy exception count and age
- Training completion rate

## Dashboard Design

Present security metrics in layers:

1. **Executive dashboard**: 5-7 high-level KPIs
2. **Team dashboard**: operational metrics with trends
3. **Detail views**: drill-down for investigation

See [[sec-043]] for security event monitoring and [[sec-014]] for the audit logging that feeds these metrics.
