# Terraform Fundamentals

## What is Terraform

**Terraform** by HashiCorp is an infrastructure as code tool that lets you define cloud resources in declarative configuration files. It supports AWS, GCP, Azure, and hundreds of other providers.

## Core Workflow

```bash
terraform init      # download providers + modules
terraform plan      # preview changes
terraform apply     # apply changes
terraform destroy   # tear down resources
```

Always run `terraform plan` before `apply`. Review the diff carefully. In CI, use `-auto-approve` only after a plan has been reviewed and approved by a human.

## State Management

Terraform tracks resource state in a **state file**. For team use, store state remotely:

```hcl
terraform {
  backend "s3" {
    bucket         = "mycompany-tf-state"
    key            = "prod/network/terraform.tfstate"
    region         = "us-east-1"
    dynamodb_table = "tf-locks"
    encrypt        = true
  }
}
```

**State locking** via DynamoDB prevents concurrent modifications. Never edit the state file manually.

## Modules

Organize infrastructure into reusable **modules**. Each module has its own `variables.tf`, `main.tf`, and `outputs.tf`. Pin module versions for reproducibility.
