# DDoS Mitigation

**Distributed Denial of Service** attacks overwhelm applications by flooding them with traffic from many sources. Effective mitigation requires multiple defensive layers.

## Attack Types

- **Volumetric attacks**: flood the network pipe (UDP floods, DNS amplification)
- **Protocol attacks**: exhaust server resources (SYN floods, Slowloris)
- **Application-layer attacks**: target specific endpoints with expensive operations (HTTP floods, API abuse)

## Defense Layers

### Edge / CDN

Put a **CDN** or DDoS mitigation provider (Cloudflare, AWS Shield, Akamai) in front of your infrastructure. They absorb volumetric and protocol attacks at the edge.

### Network Layer

- Configure **ingress filtering** (BCP38) at your network boundary
- Set appropriate **SYN cookie** settings
- Maintain **blackhole routing** capability for emergencies

### Application Layer

- Implement **rate limiting** (see [[sec-026]])
- Add **CAPTCHA** challenges on suspicious traffic
- Use **proof-of-work** challenges for expensive endpoints
- Deploy a **WAF** with DDoS-specific rules (see [[sec-029]])

## Incident Response

Have a DDoS playbook that includes:

- Contact information for your upstream provider
- Steps to enable enhanced protection
- Communication plan for affected users
- Post-incident analysis template

## Cost Considerations

DDoS protection has costs. Some providers charge for attack traffic, which creates a perverse incentive. Evaluate pricing models carefully — Cloudflare's unmetered mitigation is attractive for this reason.
