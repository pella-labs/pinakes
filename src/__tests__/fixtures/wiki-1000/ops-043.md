# Terraform Modules

## Module Design

Good Terraform **modules** are reusable, composable, and well-documented. Each module encapsulates a logical piece of infrastructure.

## Module Structure

```
modules/
  vpc/
    main.tf
    variables.tf
    outputs.tf
    versions.tf
    README.md
  rds/
    main.tf
    variables.tf
    outputs.tf
    versions.tf
```

## Calling a Module

```hcl
module "vpc" {
  source = "./modules/vpc"

  cidr_block         = "10.0.0.0/16"
  availability_zones = ["us-east-1a", "us-east-1b", "us-east-1c"]
  environment        = "production"
}

module "database" {
  source = "./modules/rds"

  vpc_id            = module.vpc.vpc_id
  subnet_ids        = module.vpc.data_subnet_ids
  instance_class    = "db.r6g.xlarge"
  engine_version    = "15.4"
  multi_az          = true
  environment       = "production"
}
```

## Versioning

For shared modules in a registry, use **semantic versioning**. Pin module versions in production code. Use `~>` constraints for minor version flexibility.

## Testing

Every module should have a Terratest suite that provisions, validates, and destroys. See [[ops-037]] for infrastructure testing details.
