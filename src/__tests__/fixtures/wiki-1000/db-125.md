# Database Design Checklist

## Schema Design

- [ ] Primary keys defined on every table
- [ ] Foreign keys with appropriate ON DELETE behavior
- [ ] NOT NULL constraints on required columns
- [ ] CHECK constraints for domain validation
- [ ] Appropriate data types (don't use TEXT for everything)
- [ ] Normalize to 3NF, then selectively denormalize ([[db-007]])

## Indexing

- [ ] Indexes on all foreign key columns
- [ ] Indexes for frequent WHERE, JOIN, ORDER BY columns ([[db-020]])
- [ ] No unnecessary indexes (check pg_stat_user_indexes for unused)
- [ ] Partial indexes where applicable
- [ ] Covering indexes for critical queries

## Performance

- [ ] Connection pooling configured ([[db-010]])
- [ ] Query timeout set ([[db-095]])
- [ ] Slow query logging enabled ([[db-102]])
- [ ] EXPLAIN ANALYZE run on critical queries ([[db-009]])
- [ ] Pagination uses cursors, not OFFSET ([[db-054]])

## Security

- [ ] Parameterized queries everywhere (no string concatenation)
- [ ] Least-privilege database roles ([[db-068]])
- [ ] TLS for all connections ([[db-091]])
- [ ] Connection strings in environment variables, not code
- [ ] Audit logging for sensitive operations ([[db-078]])

## Operations

- [ ] Automated backups with tested restores ([[db-034]])
- [ ] Monitoring dashboards with alerting ([[db-041]])
- [ ] Replication configured for HA ([[db-114]])
- [ ] Migration strategy documented ([[db-016]])
- [ ] Autovacuum tuned for high-churn tables ([[db-014]])

## Data Integrity

- [ ] Transactions used for multi-step operations ([[db-011]])
- [ ] Idempotent write operations ([[db-121]])
- [ ] Consistency checks in application and database layers
- [ ] Soft deletes or temporal patterns for audit requirements ([[db-076]])
