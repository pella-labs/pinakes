# Multi-Tenancy in APIs

## Tenant Isolation Strategies

### Database per Tenant
Complete isolation. Each tenant has its own database. Most secure but hardest to manage at scale.

### Schema per Tenant
Shared database, separate schemas. Good isolation with easier management.

### Row-Level Isolation
Single schema, tenant ID on every row. Simplest but requires discipline.

```sql
-- Every query MUST include tenant filter
SELECT * FROM orders WHERE tenant_id = $1 AND id = $2;
```

## API-Level Isolation

### Subdomain
`acme.api.example.com` — tenant from DNS, clean URLs.

### Header
`X-Tenant-ID: acme` — flexible, works with single domain.

### JWT Claim
Tenant derived from auth token. Most secure — client can't forge it.

## Implementation

```typescript
function tenantMiddleware(req: Request, res: Response, next: NextFunction) {
  // Prefer JWT claim over header
  const tenantId = req.auth?.tenantId || req.headers['x-tenant-id'];
  if (!tenantId) {
    return res.status(400).json({ error: { code: 'MISSING_TENANT' } });
  }
  req.tenantId = tenantId;
  next();
}
```
