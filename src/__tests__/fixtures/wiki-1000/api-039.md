# GraphQL DataLoader Pattern

Solves the N+1 query problem in GraphQL resolvers.

## The Problem

```graphql
query {
  orders(first: 10) {
    edges {
      node {
        customer { name }  # each order triggers a separate DB query
      }
    }
  }
}
```

10 orders = 10 separate `SELECT * FROM customers WHERE id = ?` queries.

## The Solution

DataLoader batches and deduplicates within a single tick:

```typescript
import DataLoader from 'dataloader';

const customerLoader = new DataLoader<string, Customer>(async (ids) => {
  const customers = await db.query(
    'SELECT * FROM customers WHERE id IN (?)',
    [ids]
  );
  // Must return results in same order as ids
  const map = new Map(customers.map(c => [c.id, c]));
  return ids.map(id => map.get(id) ?? new Error(`Customer ${id} not found`));
});

// In resolver
const resolvers = {
  Order: {
    customer: (order) => customerLoader.load(order.customerId),
  },
};
```

## Key Rules

- Create a **new DataLoader instance per request** (not global)
- The batch function must return results in the **same order** as input keys
- DataLoader also **deduplicates** — if 3 orders share the same customer, only 1 query
- Works with any data source: SQL, REST, Redis
