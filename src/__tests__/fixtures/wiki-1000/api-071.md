# HTTP Methods Semantics

Quick reference for correct HTTP method usage.

## GET
- **Safe:** yes (no side effects)
- **Idempotent:** yes
- **Cacheable:** yes
- **Body:** no (technically allowed but widely unsupported)
- Use for: reading resources

## POST
- **Safe:** no
- **Idempotent:** no
- **Cacheable:** no
- **Body:** yes
- Use for: creating resources, triggering actions

## PUT
- **Safe:** no
- **Idempotent:** yes (same PUT = same result)
- **Cacheable:** no
- **Body:** yes (full resource representation)
- Use for: replacing a resource entirely

## PATCH
- **Safe:** no
- **Idempotent:** depends on implementation
- **Cacheable:** no
- **Body:** yes (partial update)
- Use for: partial updates. See [[api-044]].

## DELETE
- **Safe:** no
- **Idempotent:** yes (deleting twice = same result)
- **Cacheable:** no
- **Body:** discouraged
- Use for: removing resources

## HEAD
Same as GET but no body. Used for checking existence or headers without downloading content.

## OPTIONS
Returns allowed methods. Used by CORS preflight.
