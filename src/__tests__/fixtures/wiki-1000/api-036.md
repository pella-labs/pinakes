# gRPC Error Handling

## Status Codes

gRPC has its own status code system, distinct from HTTP:

| gRPC Code | HTTP Equivalent | When |
|---|---|---|
| OK | 200 | Success |
| INVALID_ARGUMENT | 400 | Bad request |
| NOT_FOUND | 404 | Resource missing |
| ALREADY_EXISTS | 409 | Duplicate |
| PERMISSION_DENIED | 403 | Forbidden |
| UNAUTHENTICATED | 401 | No auth |
| RESOURCE_EXHAUSTED | 429 | Rate limited |
| INTERNAL | 500 | Bug |
| UNAVAILABLE | 503 | Transient failure |
| DEADLINE_EXCEEDED | 504 | Timeout |

## Rich Error Details

Use the `google.rpc.Status` pattern to attach structured details:

```protobuf
import "google/rpc/status.proto";
import "google/rpc/error_details.proto";

// Server-side
const status = new Status({
  code: Code.INVALID_ARGUMENT,
  message: 'Validation failed',
  details: [
    new BadRequest({
      fieldViolations: [
        { field: 'email', description: 'Invalid email format' }
      ]
    })
  ]
});
```

## Retryable Errors

Only retry `UNAVAILABLE` and `DEADLINE_EXCEEDED`. Other errors won't fix themselves.
