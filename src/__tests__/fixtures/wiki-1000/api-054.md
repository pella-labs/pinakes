# Handling Large API Responses

## The Problem

A client requests 10,000 records. Serializing, compressing, and transmitting costs seconds and hundreds of MB.

## Solutions

### Pagination (preferred)
Never return unbounded lists. Enforce `max_limit`. See [[api-005]].

### Sparse Fieldsets
Let clients request only the fields they need:

```
GET /orders?fields=id,status,total&fields[customer]=name
```

This reduces payload size by 60-80% for large objects.

### Response Streaming
For exports or reports, stream NDJSON:

```typescript
app.get('/export/orders', async (req, res) => {
  res.setHeader('Content-Type', 'application/x-ndjson');
  const cursor = db.query('SELECT * FROM orders').stream();

  for await (const row of cursor) {
    res.write(JSON.stringify(row) + '\n');
  }
  res.end();
});
```

### Async Export
For very large datasets, return a job ID and generate a downloadable file (CSV, Parquet) asynchronously. Notify via webhook when ready.

## Size Limits

Enforce a max response size at the framework level. If the serialized response exceeds 10MB, something is wrong — the client should be paginating.
