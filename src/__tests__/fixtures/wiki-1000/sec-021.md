# API Key Management

**API keys** are the simplest form of API authentication. They identify the calling application but do not authenticate a user. Despite their simplicity, mismanaging API keys is a frequent source of security incidents.

## Key Generation

- Use **cryptographically random** values (minimum 32 bytes, base64-encoded)
- Prefix keys with a service identifier for easy scanning: `kg_live_aBcDeFgH...`
- Store only a **hash** of the key in your database — treat keys like passwords

## Key Scoping

Each API key should have:

- A defined set of **permissions** (read-only, write, admin)
- An **expiration date** (no indefinite keys)
- IP address or CIDR **restrictions** when possible
- A human-readable **name** for identification in logs

## Rotation

Provide a key rotation mechanism that allows overlap:

1. Generate new key
2. Both old and new keys are valid during a transition window
3. Client updates to new key
4. Old key is revoked

## Monitoring

Track API key usage patterns. Alert on:

- Keys used from unexpected IP addresses
- Sudden spikes in usage
- Keys used after business hours for batch-only accounts
- Failed authentication attempts
