# Terraform Import and State Surgery

## Importing Existing Resources

When infrastructure was created manually and needs to be managed by Terraform:

```bash
# Import an existing S3 bucket
terraform import aws_s3_bucket.data my-existing-bucket

# Import with a specific module path
terraform import module.vpc.aws_vpc.main vpc-abc123

# Import into a specific workspace
terraform workspace select prod
terraform import aws_rds_cluster.main my-cluster
```

## Import Blocks (Terraform 1.5+)

The modern approach uses `import` blocks in configuration:

```hcl
import {
  to = aws_s3_bucket.data
  id = "my-existing-bucket"
}

resource "aws_s3_bucket" "data" {
  bucket = "my-existing-bucket"
}
```

Run `terraform plan` to generate the configuration, then adjust it to match the actual resource.

## State Surgery

Sometimes you need to manipulate state directly:

```bash
# Move a resource to a different module
terraform state mv aws_instance.web module.compute.aws_instance.web

# Rename a resource
terraform state mv aws_rds_instance.old aws_rds_instance.new

# Remove from state (doesn't destroy the resource)
terraform state rm aws_instance.manual
```

Always take a state backup before surgery: `terraform state pull > backup.tfstate`

See [[ops-049]] for state management, [[ops-006]] for Terraform fundamentals.
