# REST API Field Filtering

Allow clients to request only the fields they need.

## Query Parameter Syntax

```
GET /orders?fields=id,status,total
GET /orders?fields=id,status&expand=customer(name,email)
```

## Implementation

```typescript
function filterFields(obj: Record<string, any>, fields: string[]): Record<string, any> {
  if (!fields.length) return obj;
  return Object.fromEntries(
    fields
      .filter(f => f in obj)
      .map(f => [f, obj[f]])
  );
}

app.get('/orders', async (req, res) => {
  const fields = (req.query.fields as string)?.split(',') || [];
  const orders = await orderService.list(req.query);
  const filtered = orders.map(o => filterFields(o, fields));
  res.json({ data: filtered });
});
```

## Field Expansion

Some fields are expensive (require joins or additional queries). Don't include them by default — require explicit `expand`:

```
GET /orders/ord_123?expand=customer,items
```

Without `expand`, `customer` returns just `{ id: "cust_456" }`. With expand, it returns the full customer object. This gives clients control over response size and query cost.
