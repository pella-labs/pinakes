# React Native vs Flutter vs Web

## Architecture Comparison

**React Native**: JavaScript bridge to native UI components. Uses the same fundamental UI building blocks as native iOS and Android.

**Flutter**: Dart-based, renders everything with its own Skia engine. Doesn't use native UI components at all.

**Web (PWA)**: Standard web technologies. Runs in the browser or as an installed PWA.

## Performance

- Flutter: highest frame rates due to custom rendering
- React Native: near-native with New Architecture (JSI, Fabric)
- Web: depends heavily on optimization

## Developer Experience

| Factor | React Native | Flutter | Web |
|--------|-------------|---------|-----|
| Language | JavaScript/TS | Dart | JavaScript/TS |
| Hot reload | Yes | Yes | Yes |
| Debugging | Chrome DevTools | Dart DevTools | Chrome DevTools |
| Community | Very large | Large | Largest |
| Hiring | Easier (JS devs) | Harder (Dart) | Easiest |

## Code Sharing

- **React Native + Web**: via React Native Web (see [[fe-072]])
- **Flutter**: Flutter Web (varying quality)
- **Web only**: PWA for mobile distribution

## When to Choose What

- React Native: team knows JavaScript, need near-native UX, share code with web
- Flutter: need pixel-perfect custom UI, don't need web, can hire Dart developers
- Web/PWA: content-focused, broad reach, limited device API needs

See [[fe-039]] for PWA capabilities.
