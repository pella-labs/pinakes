# REST Resource Expansion

## The Problem

Returning nested resources by default is expensive. Returning only IDs forces multiple requests.

## Expand Pattern

```
GET /orders/ord_123
→ { "customer": "cust_456" }

GET /orders/ord_123?expand=customer
→ { "customer": { "id": "cust_456", "name": "Alice", "email": "..." } }

GET /orders/ord_123?expand=customer,items.product
→ nested expansion
```

## Implementation

```typescript
type ExpandConfig = {
  [field: string]: {
    loader: (ids: string[]) => Promise<Map<string, any>>;
    maxDepth: number;
  };
};

async function applyExpansions(
  data: any,
  expand: string[],
  config: ExpandConfig
): Promise<any> {
  for (const field of expand) {
    const parts = field.split('.');
    const topField = parts[0];
    
    if (config[topField] && parts.length <= config[topField].maxDepth) {
      const id = data[topField];
      const loaded = await config[topField].loader([id]);
      data[topField] = loaded.get(id);
    }
  }
  return data;
}
```

## Depth Limits

Cap expansion depth at 2 levels. `?expand=customer.orders.items.product` is an unbounded query disguised as a URL parameter.
