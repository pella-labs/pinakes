# better-sqlite3 in Node.js

## Why better-sqlite3

**better-sqlite3** is a synchronous SQLite driver for Node.js. Unlike async alternatives, it uses SQLite's native C API directly, making it significantly faster for local databases.

## Setup and Configuration

```typescript
import Database from 'better-sqlite3';

const db = new Database('/path/to/db.sqlite');

// Essential pragmas — set on every connection
db.pragma('journal_mode = WAL');
db.pragma('busy_timeout = 5000');
db.pragma('synchronous = NORMAL');
db.pragma('foreign_keys = ON');
db.pragma('cache_size = -20000');  // 20MB
db.pragma('temp_store = MEMORY');
```

## Prepared Statements

Prepared statements are compiled once and cached. Always prefer them over raw string queries:

```typescript
const insert = db.prepare(
  'INSERT INTO nodes (id, title, content) VALUES (?, ?, ?)'
);

const getById = db.prepare('SELECT * FROM nodes WHERE id = ?');
const node = getById.get(nodeId);
```

## Transactions

The `transaction()` helper wraps a function in BEGIN/COMMIT with automatic ROLLBACK on error:

```typescript
const insertMany = db.transaction((nodes: Node[]) => {
  for (const node of nodes) {
    insert.run(node.id, node.title, node.content);
  }
});

insertMany(nodeBatch);  // all-or-nothing
```

## Single Writer Pattern

SQLite allows only one writer at a time. In a Node.js server, use a single Database instance for writes and optionally a small read pool. Never use multiple writer connections — it reduces throughput due to SQLITE_BUSY retries.

See [[db-003]] for WAL mode details and [[db-028]] for when SQLite is the right choice.
