# Kubernetes Pod Security Standards

## The Three Levels

Kubernetes defines **Pod Security Standards** (PSS) that replace the deprecated PodSecurityPolicy:

### Privileged
No restrictions. Only for system-level workloads like CNI plugins and log collectors.

### Baseline
Prevents known privilege escalations. Blocks hostNetwork, hostPID, privileged containers, and most capabilities.

### Restricted
Maximum security. Requires non-root, read-only root filesystem, drops all capabilities, and disallows privilege escalation.

## Enforcement

Apply PSS at the namespace level using labels:

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: production
  labels:
    pod-security.kubernetes.io/enforce: restricted
    pod-security.kubernetes.io/warn: restricted
    pod-security.kubernetes.io/audit: restricted
```

## Migration Strategy

Start with `warn` mode to see what would be blocked, fix violations, then switch to `enforce`.

See [[ops-039]] for container security and [[ops-028]] for RBAC.
