# OpenID Connect

**OIDC** (OpenID Connect) is an identity layer built on top of OAuth2. It standardizes how clients verify user identity and obtain basic profile information.

## How OIDC Extends OAuth2

OAuth2 is an authorization framework — it provides access tokens but doesn't define a standard way to represent identity. OIDC adds:

- **ID Token**: a JWT containing identity claims (`sub`, `email`, `name`, etc.)
- **UserInfo Endpoint**: a standard API to fetch additional profile claims
- **Discovery**: a `.well-known/openid-configuration` endpoint for automatic configuration
- **Standard Scopes**: `openid`, `profile`, `email`, `address`, `phone`

## ID Token vs Access Token

The **ID token** is for the client — it proves who the user is. The **access token** is for the API — it proves what the client is authorized to do. Never send ID tokens to APIs; never use access tokens to determine identity at the client.

## Implementation

```typescript
// Validate ID token
const decoded = jwt.verify(idToken, publicKey, {
  algorithms: ['RS256'],
  audience: CLIENT_ID,
  issuer: ISSUER_URL,
});

// decoded.sub is the user's unique identifier
// decoded.email is their email
```

## Provider Selection

Major OIDC providers:

- Auth0
- Okta
- Azure AD / Entra ID
- Google Identity Platform
- Keycloak (self-hosted)
