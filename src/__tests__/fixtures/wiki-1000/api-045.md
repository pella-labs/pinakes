# API Timeouts

## Client Timeouts

Set timeouts at the HTTP client level, not just the server:

```typescript
const client = axios.create({
  timeout: 5000,            // 5s overall timeout
  signal: AbortSignal.timeout(5000),
});
```

## Server Timeouts

Layer timeouts from outside in:

1. **Load balancer**: 30s (outermost)
2. **Application server**: 25s
3. **Database query**: 10s
4. **External API call**: 5s

Each inner timeout must be shorter than the outer. Otherwise the outer kills the request while the inner is still running, leaking resources.

## Timeout Budgets

For microservice chains, propagate a **deadline** rather than fixed timeouts:

```typescript
function withDeadline(req: Request) {
  const remaining = parseInt(req.headers['x-deadline-ms'] || '30000');
  const elapsed = Date.now() - parseInt(req.headers['x-start-time'] || '0');
  return Math.max(remaining - elapsed, 0);
}
```

If you've used 20s of a 30s budget, the next service call only gets 10s.

## Timeout Errors

Return `504 Gateway Timeout` when a downstream call times out. Return `408 Request Timeout` when the client's request itself timed out.
