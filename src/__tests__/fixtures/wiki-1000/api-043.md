# API Request Coalescing

Deduplicate identical concurrent requests.

## Problem

Three clients request `GET /products/p_123` within 10ms. Without coalescing, the database gets hit three times for the same data.

## Solution

```typescript
const inflight = new Map<string, Promise<Response>>();

async function coalesce(key: string, fn: () => Promise<Response>): Promise<Response> {
  if (inflight.has(key)) {
    return inflight.get(key)!;
  }

  const promise = fn().finally(() => inflight.delete(key));
  inflight.set(key, promise);
  return promise;
}

// Usage
app.get('/products/:id', (req, res) => {
  const result = await coalesce(`product:${req.params.id}`, () =>
    productService.getById(req.params.id)
  );
  res.json(result);
});
```

## Caveats

- Only works for GET requests (idempotent reads)
- Must NOT coalesce requests with different auth contexts
- In-process only — for cross-instance coalescing, use Redis locking or a cache layer
- The `inflight` map should have a max size to prevent memory leaks
