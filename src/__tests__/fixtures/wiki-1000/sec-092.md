# Fuzzing for Security

**Fuzzing** (fuzz testing) feeds random or semi-random input to a program to discover crashes, hangs, and security vulnerabilities. It finds bugs that structured testing misses.

## Types of Fuzzing

- **Dumb fuzzing**: completely random input. Low signal but easy to start.
- **Mutation fuzzing**: takes valid inputs and mutates them. Better coverage.
- **Generation fuzzing**: constructs inputs based on protocol/format grammar. Best coverage.
- **Coverage-guided fuzzing**: uses code coverage feedback to explore new paths. State of the art.

## Tools

- **AFL++**: the gold standard for C/C++ fuzzing
- **libFuzzer**: Clang-integrated, fast
- **jazzer**: JVM fuzzing
- **jsfuzz**: JavaScript fuzzing
- **cargo-fuzz**: Rust fuzzing

## What to Fuzz

Prioritize code that:
- Parses untrusted input (file parsers, network protocols)
- Processes complex data formats (XML, JSON, images)
- Implements cryptographic operations
- Contains custom serialization logic (see [[sec-090]])

## Setting Up a Fuzz Target

```typescript
// Simple fuzz target for a parser
export function fuzz(data: Buffer): void {
  try {
    parser.parse(data.toString());
  } catch (e) {
    // Expected parse errors are fine
    // Crashes and hangs are bugs
  }
}
```

## Integration with CI

Run fuzz campaigns nightly. Report new crashes as high-priority bugs. Maintain a corpus of interesting inputs for regression testing.
