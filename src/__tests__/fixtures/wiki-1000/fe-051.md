# React Memory Leaks and Cleanup

## Common Sources

Memory leaks in React applications typically come from forgetting to clean up:

- Event listeners added in `useEffect`
- WebSocket connections
- Timers and intervals
- Subscriptions to external stores
- AbortController signals for fetch requests

## Proper Cleanup Pattern

```typescript
useEffect(() => {
  const controller = new AbortController();

  async function fetchData() {
    try {
      const res = await fetch('/api/data', { signal: controller.signal });
      const data = await res.json();
      setData(data);
    } catch (err) {
      if (err instanceof DOMException && err.name === 'AbortError') return;
      setError(err);
    }
  }

  fetchData();
  return () => controller.abort();
}, []);
```

## Detecting Leaks

- React Strict Mode double-invokes effects to surface missing cleanup
- Chrome DevTools Memory tab shows heap snapshots
- The `why-did-you-render` library flags unexpected re-renders

## Stale Closures

A related issue is stale closures in effects that capture outdated state values. Use refs for values that need to be current without triggering re-renders:

```typescript
const callbackRef = useRef(callback);
callbackRef.current = callback;

useEffect(() => {
  const timer = setInterval(() => callbackRef.current(), 1000);
  return () => clearInterval(timer);
}, []);
```
