# REST Naming: Actions vs Resources

## The Resource Way

Model everything as a **noun**:

```
POST /orders                    ← create
GET /orders/{id}                ← read
PUT /orders/{id}                ← replace
PATCH /orders/{id}              ← update
DELETE /orders/{id}             ← delete
```

## When Actions Are OK

Some operations don't map cleanly to CRUD:

```
POST /orders/{id}/cancel        ← state transition
POST /orders/{id}/refund        ← complex action with side effects
POST /search/products           ← complex search with body
POST /reports/generate          ← triggers async work
```

These are **controller resources** — they represent an action, and `POST` is the verb.

## Anti-Patterns

```
GET /getUser?id=123           ← verb in URL
POST /createOrder             ← verb in URL
PUT /updateOrderStatus        ← verb + specific field in URL
DELETE /deleteAllExpired       ← bulk operation hidden in URL
```

Use HTTP methods for CRUD, sub-resources for state transitions, and dedicated endpoints for complex non-CRUD operations.
