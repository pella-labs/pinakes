# DNS Security

**DNS** is a frequent target for attacks because it's foundational to internet communication. Compromised DNS allows attackers to redirect traffic, intercept credentials, and impersonate services.

## Threats

- **DNS spoofing/cache poisoning**: attacker injects false DNS records into a resolver's cache
- **DNS hijacking**: attacker modifies DNS records at the registrar or nameserver
- **DNS tunneling**: using DNS queries to exfiltrate data or establish C2 channels
- **Typosquatting**: registering domains similar to yours to catch typos

## DNSSEC

**DNSSEC** adds cryptographic signatures to DNS records, allowing resolvers to verify authenticity:

- Sign your zones with DNSSEC
- Enable DNSSEC validation on your resolvers
- Monitor for DNSSEC signature expiration

## DNS Monitoring

Monitor your DNS infrastructure for:

- Unauthorized record changes
- Unusual query volumes (may indicate tunneling)
- Resolution to unexpected IP addresses
- Certificate transparency logs for unauthorized certificates on your domains

## DNS-Based Security Controls

- **DNS sinkholes**: redirect known malicious domains to safe destinations
- **Response Policy Zones** (RPZ): enforce DNS-level blocking
- **DNS over HTTPS/TLS**: encrypt DNS queries to prevent eavesdropping

## Registrar Security

Protect your domain registrar account:

- Enable MFA (see [[sec-056]])
- Enable registrar lock (prevents unauthorized transfers)
- Monitor WHOIS records for changes
- Use a reputable registrar with strong security practices
