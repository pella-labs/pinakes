# View Transitions API

## What It Does

The **View Transitions API** enables smooth animated transitions between DOM states. It captures a screenshot of the old state, applies the DOM update, then animates between old and new.

## Basic Usage

```typescript
function handleNavigation() {
  if (!document.startViewTransition) {
    updateDOM();
    return;
  }

  document.startViewTransition(() => {
    updateDOM();
  });
}
```

## Customizing Animations

```css
::view-transition-old(root) {
  animation: fade-out 200ms ease;
}

::view-transition-new(root) {
  animation: fade-in 200ms ease;
}

@keyframes fade-out {
  to { opacity: 0; }
}

@keyframes fade-in {
  from { opacity: 0; }
}
```

## Named Transitions

Assign `view-transition-name` to specific elements for targeted animations:

```css
.hero-image {
  view-transition-name: hero;
}

::view-transition-group(hero) {
  animation-duration: 300ms;
}
```

The browser animates the hero image between its old and new positions, creating a seamless transition.

## Framework Integration

Next.js and other frameworks are adding built-in View Transitions support for route changes. This replaces heavy JavaScript animation libraries for page transitions.

See [[fe-025]] for broader animation patterns.
