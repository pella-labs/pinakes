# CORS for API Developers

## The Preflight

Before a browser sends a cross-origin request with custom headers or non-simple methods, it sends an `OPTIONS` preflight:

```
OPTIONS /api/orders
Origin: https://app.example.com
Access-Control-Request-Method: POST
Access-Control-Request-Headers: Content-Type, Authorization
```

Your server must respond:

```
Access-Control-Allow-Origin: https://app.example.com
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization
Access-Control-Max-Age: 86400
```

## Cache the Preflight

`Access-Control-Max-Age: 86400` caches the preflight for 24 hours. Without it, every cross-origin request makes two HTTP calls.

## Credentials

If the client sends cookies or Authorization headers, the server must:
1. Set `Access-Control-Allow-Credentials: true`
2. Set `Access-Control-Allow-Origin` to the specific origin (not `*`)
3. Set `Access-Control-Expose-Headers` for any custom response headers the client needs

## Common Mistakes

- Using `*` for origin with credentials → browser rejects
- Forgetting to handle OPTIONS → 405 Method Not Allowed → silent CORS failure
- Not exposing custom headers → client can't read `X-RateLimit-Remaining`
