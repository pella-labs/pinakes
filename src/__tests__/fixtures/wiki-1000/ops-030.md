# CI Pipeline Optimization

## Build Speed Matters

Slow CI pipelines kill developer productivity. A 20-minute pipeline means 20 minutes of context switching. Target **under 10 minutes** for the full pipeline.

## Optimization Techniques

### Parallelization

Run independent jobs in parallel. Tests, linting, type checking, and security scans don't depend on each other:

```yaml
jobs:
  lint:
    runs-on: ubuntu-latest
    steps: [...]
  typecheck:
    runs-on: ubuntu-latest
    steps: [...]
  test:
    runs-on: ubuntu-latest
    steps: [...]
  security:
    runs-on: ubuntu-latest
    steps: [...]
  deploy:
    needs: [lint, typecheck, test, security]
    runs-on: ubuntu-latest
    steps: [...]
```

### Caching

Cache aggressively:
- **Package manager caches** (pnpm store, pip cache)
- **Docker layer caches** (BuildKit inline cache or registry cache)
- **Build artifact caches** (TypeScript build info, Webpack cache)

### Selective Testing

Only run tests affected by changed files. Use tools like **nx affected** or **turborepo** to determine the blast radius of a change.

### Smaller Images

Use Alpine-based images and multi-stage builds. Smaller images mean faster pulls in CI.
