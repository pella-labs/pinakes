# React Suspense and Concurrent Features

## Suspense for Data Fetching

**Suspense** lets components declare loading states declaratively. Instead of managing `isLoading` flags, components throw promises and Suspense catches them.

```tsx
function Dashboard() {
  return (
    <Suspense fallback={<DashboardSkeleton />}>
      <DashboardContent />
    </Suspense>
  );
}

async function DashboardContent() {
  const metrics = await getMetrics();
  return <MetricsGrid data={metrics} />;
}
```

## Nested Suspense Boundaries

Each boundary controls its own loading state. This enables progressive loading where the page shell appears first, then sections fill in independently.

## useTransition

The `useTransition` hook marks state updates as non-urgent, keeping the current UI responsive while the new state loads:

```tsx
const [isPending, startTransition] = useTransition();

function handleSearch(query: string) {
  startTransition(() => {
    setSearchResults(search(query));
  });
}
```

## useDeferredValue

Defers re-rendering of expensive components until urgent updates complete. Useful for search results that update as the user types.
