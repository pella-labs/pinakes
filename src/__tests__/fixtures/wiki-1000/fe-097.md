# React Patterns for Data Visualization

## Library Choices

- **D3**: low-level, maximum control, steep learning curve
- **Recharts**: React-native wrapper around D3, declarative API
- **Victory**: modular chart components by Formidable
- **visx**: low-level React + D3 primitives by Airbnb
- **Nivo**: rich chart types with theming support

## D3 + React Integration

Let React own the DOM, use D3 only for math:

```tsx
function BarChart({ data, width, height }: ChartProps) {
  const xScale = d3.scaleBand()
    .domain(data.map((d) => d.label))
    .range([0, width])
    .padding(0.2);

  const yScale = d3.scaleLinear()
    .domain([0, d3.max(data, (d) => d.value)!])
    .range([height, 0]);

  return (
    <svg width={width} height={height}>
      {data.map((d) => (
        <rect
          key={d.label}
          x={xScale(d.label)}
          y={yScale(d.value)}
          width={xScale.bandwidth()}
          height={height - yScale(d.value)}
          fill="var(--color-primary)"
        />
      ))}
    </svg>
  );
}
```

## Responsive Charts

Use a resize observer hook to make charts responsive:

```typescript
function useContainerSize(ref: RefObject<HTMLDivElement>) {
  const [size, setSize] = useState({ width: 0, height: 0 });

  useEffect(() => {
    const observer = new ResizeObserver(([entry]) => {
      setSize({
        width: entry.contentRect.width,
        height: entry.contentRect.height,
      });
    });
    if (ref.current) observer.observe(ref.current);
    return () => observer.disconnect();
  }, []);

  return size;
}
```

## Accessibility

Charts must provide text alternatives. Use `aria-label` on SVG elements and provide a data table as a fallback. See [[fe-005]] for a11y fundamentals.
