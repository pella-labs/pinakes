# Multi-Factor Authentication Implementation

**MFA** adds a second verification factor beyond the password, dramatically reducing account takeover risk. Even if a password is compromised, the attacker needs the second factor.

## Factor Categories

- **Something you know**: password, PIN
- **Something you have**: phone, hardware key, authenticator app
- **Something you are**: fingerprint, face recognition

True MFA uses factors from different categories. Two passwords is not MFA.

## TOTP (Time-Based One-Time Password)

The most common software-based MFA:

```typescript
import { authenticator } from 'otplib';

// Setup
const secret = authenticator.generateSecret();
const otpauth = authenticator.keyuri(user.email, 'MyApp', secret);
// Generate QR code from otpauth URI

// Verification
const isValid = authenticator.verify({ token: userInput, secret });
```

Store the secret encrypted. Provide **backup codes** (one-time use) for recovery.

## WebAuthn / FIDO2

**WebAuthn** provides phishing-resistant MFA using hardware security keys or platform authenticators:

- Bound to the origin (phishing-proof)
- No shared secrets
- Supported by YubiKey, TouchID, Windows Hello
- Replaces both password and TOTP

## Implementation Decisions

- Offer MFA but don't force it initially (increases adoption)
- Require MFA for admin accounts and sensitive operations
- Support multiple enrolled devices (redundancy)
- Implement a secure recovery flow (backup codes, admin reset)
- Log all MFA events (see [[sec-014]])
