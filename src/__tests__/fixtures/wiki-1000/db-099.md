# EXPLAIN Output Reference

## PostgreSQL Plan Nodes

### Scan Types
- **Seq Scan**: reads every row in the table. Red flag on large tables.
- **Index Scan**: traverses the index, then fetches rows from heap.
- **Index Only Scan**: serves query entirely from the index. The best case.
- **Bitmap Index Scan** + **Bitmap Heap Scan**: scans index, builds a bitmap, then fetches matching heap pages. Efficient for moderate selectivity.

### Join Types
- **Nested Loop**: for each row in the outer table, scan the inner table. Good for small inner tables.
- **Hash Join**: build a hash table from the inner table, probe with outer rows. Good for large equi-joins.
- **Merge Join**: both inputs sorted, merge them. Good when both sides are pre-sorted.

### Aggregate/Sort
- **Sort**: sorts rows (may spill to disk if work_mem exceeded)
- **HashAggregate**: groups using a hash table
- **GroupAggregate**: groups pre-sorted input

## Key Metrics

```
Sort  (cost=100.00..105.00 rows=1000 width=44) (actual time=1.2..1.5 rows=987 loops=1)
       ^startup_cost  ^total_cost  ^est_rows              ^actual_time  ^actual_rows
```

When `rows` and `actual rows` diverge significantly, run `ANALYZE` to update statistics. See [[db-009]] for optimization techniques.
