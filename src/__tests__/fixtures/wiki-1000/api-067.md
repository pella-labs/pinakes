# API Request Signing

## Why Sign Requests

API keys authenticate the caller. **Request signing** proves the request hasn't been tampered with in transit. Essential for:

- Webhook delivery verification
- Financial APIs
- Any request passing through untrusted intermediaries

## HMAC Signature

```typescript
function signRequest(method: string, path: string, body: string, secret: string): string {
  const timestamp = Math.floor(Date.now() / 1000).toString();
  const payload = `${timestamp}.${method}.${path}.${body}`;
  const signature = crypto.createHmac('sha256', secret).update(payload).digest('hex');
  return `t=${timestamp},v1=${signature}`;
}

// Verification
function verifySignature(sig: string, method: string, path: string, body: string, secret: string): boolean {
  const parts = Object.fromEntries(sig.split(',').map(p => p.split('=')));
  const timestamp = parseInt(parts.t);
  
  // Reject if older than 5 minutes (replay protection)
  if (Date.now() / 1000 - timestamp > 300) return false;
  
  const expected = signRequest(method, path, body, secret);
  return crypto.timingSafeEqual(Buffer.from(sig), Buffer.from(expected));
}
```

## Include Timestamp

Always include a timestamp in the signed payload. Reject signatures older than 5 minutes to prevent **replay attacks**.
