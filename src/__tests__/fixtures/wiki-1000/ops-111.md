# Kubernetes Network Debugging

## DNS Resolution

```bash
# Test DNS from inside a pod
kubectl exec -it debug-pod -- nslookup my-service.my-namespace.svc.cluster.local

# Check CoreDNS logs
kubectl logs -n kube-system -l k8s-app=kube-dns --tail=100
```

## Connectivity Testing

```bash
# Test TCP connectivity
kubectl exec -it debug-pod -- nc -zv my-service 8080

# Test HTTP endpoint
kubectl exec -it debug-pod -- curl -v http://my-service:8080/healthz

# Trace route to external service
kubectl exec -it debug-pod -- traceroute api.external.com
```

## Network Policy Debugging

Network policies are additive. If traffic is being blocked, check:

```bash
# List all network policies in the namespace
kubectl get networkpolicies -n my-namespace -o yaml

# Check if the pod matches any policy's podSelector
kubectl get pods -n my-namespace --show-labels
```

If a pod matches no network policy, all traffic is allowed. If it matches any policy, only explicitly allowed traffic passes.

## tcpdump on a Pod

```bash
kubectl debug -it pod/my-pod --image=nicolaka/netshoot --target=my-container
tcpdump -i eth0 -n port 8080
```

This captures network traffic on the pod's interface for detailed packet analysis.
