# Animation Patterns for Web UIs

## Principles

Good animation serves a purpose: it guides attention, provides feedback, and communicates spatial relationships. Gratuitous animation is worse than none.

## CSS Transitions

For simple state changes, CSS transitions are sufficient and performant:

```css
.button {
  transition: background-color 150ms ease, transform 100ms ease;
}

.button:hover {
  background-color: var(--color-primary-hover);
}

.button:active {
  transform: scale(0.97);
}
```

## Framer Motion

For complex animations in React, **Framer Motion** provides a declarative API:

```tsx
import { motion, AnimatePresence } from 'framer-motion';

function Modal({ isOpen, children }) {
  return (
    <AnimatePresence>
      {isOpen && (
        <motion.div
          initial={{ opacity: 0, y: 20 }}
          animate={{ opacity: 1, y: 0 }}
          exit={{ opacity: 0, y: -20 }}
          transition={{ duration: 0.2 }}
        >
          {children}
        </motion.div>
      )}
    </AnimatePresence>
  );
}
```

## Performance Tips

- Animate only `transform` and `opacity` for 60fps (they skip layout and paint)
- Use `will-change` sparingly to hint GPU compositing
- Prefer `prefers-reduced-motion` media query for users who request less motion
- Avoid animating `height: auto`—use `max-height` or layout animation libraries
