# API Input Sanitization

## Never Trust Input

All input is untrusted until validated. This applies to:

- Request body
- URL parameters
- Query parameters
- Headers
- Cookies

## String Sanitization

```typescript
function sanitizeString(input: string, maxLength: number = 1000): string {
  return input
    .slice(0, maxLength)              // length limit
    .replace(/[\x00-\x08\x0B\x0C\x0E-\x1F]/g, ''); // strip control chars
}
```

## SQL Injection Prevention

Always use parameterized queries:

```typescript
// NEVER
db.query(`SELECT * FROM users WHERE id = '${userId}'`);

// ALWAYS
db.query('SELECT * FROM users WHERE id = $1', [userId]);
```

## NoSQL Injection

MongoDB is also vulnerable if you pass raw objects:

```typescript
// Vulnerable to { "$gt": "" }
db.collection('users').findOne({ password: req.body.password });

// Safe
db.collection('users').findOne({ password: String(req.body.password) });
```

## Header Injection

Validate `Content-Type`, `Accept`, and custom headers. Reject unexpected values. Never reflect headers into responses without sanitization (prevents CRLF injection).
