# AWS Lambda Operations

## Lambda in the DevOps Toolkit

**Lambda** functions are useful for operational automation: custom CloudWatch alarms, auto-remediation, and glue between AWS services.

## Deployment

```yaml
# SAM template
AWSTemplateFormatVersion: '2010-09-09'
Transform: AWS::Serverless-2016-10-31

Resources:
  AutoRemediator:
    Type: AWS::Serverless::Function
    Properties:
      Handler: index.handler
      Runtime: nodejs20.x
      Timeout: 60
      MemorySize: 256
      Events:
        SNSEvent:
          Type: SNS
          Properties:
            Topic: !Ref AlertTopic
      Policies:
        - Statement:
            - Effect: Allow
              Action:
                - ecs:UpdateService
                - autoscaling:SetDesiredCapacity
              Resource: "*"
```

## Operational Lambda Patterns

- **Alert handler**: receive SNS notifications from CloudWatch and perform auto-remediation
- **Scheduled cleanup**: delete old snapshots, terminate forgotten instances
- **Custom metrics publisher**: collect application metrics and push to CloudWatch
- **Webhook handler**: process GitHub webhooks for custom CI/CD logic

## Monitoring Lambda

Monitor Lambda functions with:
- Invocation count and error rate
- Duration (p50, p99)
- Throttle count
- Concurrent executions
- Dead letter queue depth

See [[ops-087]] for CloudWatch monitoring.
