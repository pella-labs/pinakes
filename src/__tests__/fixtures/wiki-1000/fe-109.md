# Suspense-Based Image Loading

## The Problem

Images load asynchronously. Without coordination, they pop in one by one, causing layout shift and a jarring experience.

## Image Resource Hook

```typescript
const imageCache = new Map<string, Promise<void>>();

function preloadImage(src: string): Promise<void> {
  if (!imageCache.has(src)) {
    imageCache.set(
      src,
      new Promise((resolve, reject) => {
        const img = new Image();
        img.onload = () => resolve();
        img.onerror = reject;
        img.src = src;
      })
    );
  }
  return imageCache.get(src)!;
}

function useImageResource(src: string) {
  const promise = preloadImage(src);
  // React Suspense: throw the promise if not resolved
  if (promise instanceof Promise) {
    throw promise;
  }
}
```

## Coordinated Loading

Wrap an image gallery in a single Suspense boundary to show all images at once:

```tsx
<Suspense fallback={<GallerySkeleton count={6} />}>
  <ImageGallery images={images} />
</Suspense>
```

All images load in parallel. The skeleton stays visible until every image is ready, then the gallery appears complete.

## Progressive Enhancement

For non-critical images, let them load individually with native `loading="lazy"`. Reserve Suspense-coordinated loading for hero sections and above-the-fold galleries.
