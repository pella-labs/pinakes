# Kubernetes Gateway API

## Evolution from Ingress

The **Gateway API** is the successor to the Ingress resource. It provides richer routing capabilities, cross-namespace references, and role-oriented design.

## Core Resources

- **GatewayClass**: defines the controller (like IngressClass)
- **Gateway**: deploys a load balancer with listeners
- **HTTPRoute**: defines routing rules

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: production
  namespace: gateway-system
spec:
  gatewayClassName: aws-alb
  listeners:
    - name: https
      protocol: HTTPS
      port: 443
      tls:
        mode: Terminate
        certificateRefs:
          - name: api-tls
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: api-routes
spec:
  parentRefs:
    - name: production
      namespace: gateway-system
  hostnames:
    - api.example.com
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /v2
      backendRefs:
        - name: api-v2
          port: 8080
    - backendRefs:
        - name: api-v1
          port: 8080
```

## Advantages Over Ingress

- **Traffic splitting** for canary deploys
- **Header-based routing** without annotations
- **Cross-namespace** references (frontend team routes to backend team services)
- **TCP/UDP** support via TCPRoute and UDPRoute

See [[ops-036]] for legacy Ingress and [[ops-011]] for load balancers.
