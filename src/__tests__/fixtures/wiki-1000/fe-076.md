# SSG vs SSR vs ISR Decision Matrix

## Static Site Generation (SSG)

Pages are generated at build time. Best for content that rarely changes.

**Pros**: fastest possible TTFB, CDN-cacheable, cheapest to host
**Cons**: build time grows with page count, stale until next build

**Use for**: marketing pages, documentation, blog posts

## Server-Side Rendering (SSR)

Pages are generated on every request. Best for personalized or real-time content.

**Pros**: always fresh, personalized per request
**Cons**: higher server costs, slower TTFB than static, requires server infrastructure

**Use for**: dashboards, user profiles, search results

## Incremental Static Regeneration (ISR)

Pages are static but regenerate on a schedule or on demand. See [[fe-021]].

**Pros**: static performance with dynamic freshness
**Cons**: brief staleness window, more complex caching logic

**Use for**: product catalogs, content sites with frequent updates

## Per-Route Strategy

Modern frameworks (Next.js, Nuxt) allow mixing strategies per route. A landing page can be static while a dashboard is server-rendered.

| Route | Strategy | Revalidation |
|-------|----------|-------------|
| / | SSG | — |
| /blog/[slug] | ISR | 60 seconds |
| /dashboard | SSR | — |
| /settings | SSR | — |
| /docs/[...path] | SSG | — |

See [[fe-019]] for SSR optimization and [[fe-004]] for Next.js routing.
