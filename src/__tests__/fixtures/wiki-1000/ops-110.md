# Terraform Cloud and Automation

## Terraform Cloud Features

**Terraform Cloud** (TFC) provides remote state, plan/apply workflows, policy enforcement, and team collaboration for Terraform.

## Remote Execution

Instead of running Terraform locally or in CI, TFC runs plans and applies in a managed environment:

```hcl
terraform {
  cloud {
    organization = "mycompany"
    workspaces {
      tags = ["networking"]
    }
  }
}
```

## Policy as Code with Sentinel

TFC supports **Sentinel** policies to enforce governance:

```python
import "tfplan/v2" as tfplan

# Require encryption on all S3 buckets
main = rule {
  all tfplan.resource_changes as _, rc {
    rc.type is "aws_s3_bucket" implies
      rc.change.after.server_side_encryption_configuration is not null
  }
}
```

## Run Triggers

Chain workspaces together with **run triggers**. When the networking workspace applies, it automatically triggers the compute workspace.

## Cost Estimation

TFC provides **cost estimation** on plans, showing the monthly cost impact of proposed changes before they're applied.

## Self-Hosted Alternative

For organizations that can't use SaaS, **Terraform Enterprise** provides the same features self-hosted. Alternatively, use Atlantis for open-source plan/apply automation.
