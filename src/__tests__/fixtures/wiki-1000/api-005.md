# Pagination Strategies

## Offset-Based

Classic `?offset=20&limit=10`. Simple but breaks when items are inserted or deleted during pagination. Fine for admin dashboards, bad for user-facing feeds.

## Cursor-Based

Use an opaque cursor (base64-encoded ID or timestamp) instead of an offset. The server decodes it and queries `WHERE id > cursor_id LIMIT n`.

```json
{
  "data": [...],
  "pagination": {
    "next_cursor": "eyJpZCI6MTAwfQ==",
    "has_more": true
  }
}
```

**Why cursors win:** stable under concurrent writes, no skipped/duplicate rows, works with any index.

## Keyset Pagination

The underlying mechanism for cursor-based. Uses `WHERE (sort_col, id) > (last_sort_val, last_id)` for deterministic ordering even with non-unique sort columns.

```sql
SELECT * FROM orders
WHERE (created_at, id) > ('2025-01-15T00:00:00Z', 'abc-123')
ORDER BY created_at ASC, id ASC
LIMIT 25;
```
