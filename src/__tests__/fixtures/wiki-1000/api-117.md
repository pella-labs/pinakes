# API Backward Compatibility Shims

## When You Need a Shim

You've changed the response shape but can't break existing clients. A shim translates between old and new formats.

## Implementation

```typescript
function shimOrderResponse(order: OrderV2, apiVersion: string): OrderV1 | OrderV2 {
  if (apiVersion === 'v1') {
    return {
      id: order.id,
      user_name: order.customerName,     // v1 field name
      delivery_date: order.shippingEstimate, // v1 field name
      amount: order.total.amount * 100,  // v1 used cents
    };
  }
  return order;
}
```

## Shim as Middleware

```typescript
function versionShim(shimMap: Record<string, (data: any) => any>): Middleware {
  return (req, res, next) => {
    const version = req.params.version || 'v2';
    const originalJson = res.json.bind(res);
    
    res.json = (body: any) => {
      const shim = shimMap[version];
      return originalJson(shim ? shim(body) : body);
    };
    
    next();
  };
}
```

## Sunset Rule

Every shim must have a removal date. Track it:

```typescript
const shims = [
  { version: 'v1', sunset: '2026-03-01', handler: shimV1 },
];
```

After sunset, the shim returns `410 Gone`. See [[api-028]] for the deprecation strategy.
