# PostgreSQL Row-Level Security

## What Is RLS?

**Row-Level Security** (RLS) restricts which rows a user can see or modify based on policies. It's the database-level enforcement of multi-tenant data isolation.

## Enabling RLS

```sql
ALTER TABLE documents ENABLE ROW LEVEL SECURITY;

-- Policy: users can only see their own documents
CREATE POLICY user_isolation ON documents
  USING (owner_id = current_setting('app.current_user_id')::int);

-- Policy: admins can see everything
CREATE POLICY admin_access ON documents
  USING (current_setting('app.current_role') = 'admin');
```

## Setting Context

Before each query, set the session variables that policies reference:

```typescript
await client.query("SET app.current_user_id = $1", [userId]);
await client.query("SET app.current_role = $1", [role]);
const docs = await client.query("SELECT * FROM documents");
// RLS filters automatically applied
```

## Use Cases

- Multi-tenant SaaS (tenant isolation without WHERE clauses everywhere)
- HIPAA compliance (restrict access to patient data)
- Supabase uses RLS as its primary authorization mechanism

## Performance

RLS policies are applied as additional WHERE clauses. Ensure the policy columns are indexed. For complex policies, test with EXPLAIN to verify the planner handles them efficiently.
