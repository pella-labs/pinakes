# XML External Entity Prevention

**XXE** (XML External Entity) attacks exploit XML parsers that process external entity references in XML input. A successful XXE attack can read local files, trigger SSRF, or cause denial of service.

## The Attack

```xml
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<data>&xxe;</data>
```

If the parser resolves external entities, it will include the contents of `/etc/passwd` in the parsed output.

## Prevention

### Disable External Entities

Configure your XML parser to disable external entity processing:

```java
// Java
DocumentBuilderFactory dbf = DocumentBuilderFactory.newInstance();
dbf.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);
```

```python
# Python (defusedxml)
from defusedxml import ElementTree
tree = ElementTree.parse('input.xml')
```

### Use JSON

If possible, use JSON instead of XML. JSON has no entity mechanism and is immune to XXE.

### Input Validation

Reject XML input that contains DOCTYPE declarations:

```typescript
if (xmlInput.includes('<!DOCTYPE') || xmlInput.includes('<!ENTITY')) {
  throw new ValidationError('DOCTYPE not allowed');
}
```

## SAML Implications

SAML assertions are XML-based. Ensure your SAML library disables external entity processing. See [[sec-019]] for SAML security considerations.

See [[sec-058]] for SSRF prevention, as XXE can be used to trigger SSRF.
