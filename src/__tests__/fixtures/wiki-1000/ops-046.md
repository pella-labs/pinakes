# Kubernetes Persistent Volumes

## Storage Classes

**StorageClasses** define the type of storage available in the cluster. On AWS EKS, common classes include:

- `gp3` — general purpose SSD (default)
- `io2` — high-performance SSD for databases
- `st1` — throughput-optimized HDD for logs and analytics

## Dynamic Provisioning

When a pod requests a PersistentVolumeClaim, the storage class dynamically provisions the underlying volume:

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: app-data
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: gp3
  resources:
    requests:
      storage: 50Gi
```

## EBS CSI Driver

The **EBS CSI driver** manages EBS volumes as Kubernetes persistent volumes. It supports snapshots, volume resizing, and encryption.

## Volume Snapshots

Use **VolumeSnapshots** for backup:

```yaml
apiVersion: snapshot.storage.k8s.io/v1
kind: VolumeSnapshot
metadata:
  name: app-data-snapshot
spec:
  volumeSnapshotClassName: ebs-csi
  source:
    persistentVolumeClaimName: app-data
```

Schedule snapshots via CronJobs for regular backups.

## EFS for Shared Storage

When multiple pods need to read/write the same filesystem, use **EFS** (Elastic File System) with the EFS CSI driver. This provides `ReadWriteMany` access mode.
