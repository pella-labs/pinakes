# API Observability: Distributed Tracing

## What Tracing Captures

A **trace** follows a single request through all services it touches. Each service creates a **span** with timing and metadata.

```
[API Gateway] ──→ [Order Service] ──→ [Database]
     │                    │
     │                    └──→ [Payment Service] ──→ [Stripe API]
     │
     └──→ [Auth Service]
```

## OpenTelemetry Setup

```typescript
import { NodeTracerProvider } from '@opentelemetry/sdk-trace-node';
import { HttpInstrumentation } from '@opentelemetry/instrumentation-http';

const provider = new NodeTracerProvider();
provider.register();

const httpInstrumentation = new HttpInstrumentation({
  requestHook: (span, request) => {
    span.setAttribute('http.request_id', request.headers['x-request-id']);
  },
});
```

## Propagation

Trace context propagates via W3C `traceparent` header:

```
traceparent: 00-0af7651916cd43dd8448eb211c80319c-b7ad6b7169203331-01
```

Every service reads and propagates this header. See [[api-041]] for request ID tracing which complements distributed tracing.

## What to Trace

- All HTTP calls (incoming and outgoing)
- Database queries (with sanitized SQL)
- Cache operations
- Queue publish/consume
- Critical business logic spans
