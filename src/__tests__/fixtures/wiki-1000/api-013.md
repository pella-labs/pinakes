# Content Negotiation

## Accept Header Parsing

The client sends `Accept: application/json, text/html;q=0.9, */*;q=0.1` and the server picks the best match.

**Quality factor** (`q`) ranges from 0 to 1. Higher wins. Omitted means `q=1.0`.

## Implementation

```typescript
function negotiate(acceptHeader: string, supported: string[]): string | null {
  const parsed = acceptHeader
    .split(',')
    .map(part => {
      const [type, ...params] = part.trim().split(';');
      const q = params.find(p => p.trim().startsWith('q='));
      return { type: type.trim(), quality: q ? parseFloat(q.split('=')[1]) : 1.0 };
    })
    .sort((a, b) => b.quality - a.quality);

  for (const { type } of parsed) {
    if (type === '*/*') return supported[0];
    if (supported.includes(type)) return type;
  }
  return null;
}
```

If no match, return `406 Not Acceptable`.

## Common Content Types

- `application/json` — default for APIs
- `application/hal+json` — HATEOAS (see [[api-016]])
- `text/event-stream` — SSE (see [[api-020]])
- `application/x-ndjson` — streaming JSON lines
- `application/protobuf` — gRPC binary (see [[api-009]])
