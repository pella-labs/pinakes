# Cloud Networking Fundamentals

## OSI Model in the Cloud

Understanding the **OSI model** helps reason about cloud networking:

- **Layer 3 (Network)**: VPCs, subnets, route tables, NAT gateways
- **Layer 4 (Transport)**: security groups, NACLs, NLBs
- **Layer 7 (Application)**: ALBs, WAF, API Gateway

## VPC Peering vs Transit Gateway

**VPC Peering** connects two VPCs directly. Simple for a few VPCs but doesn't scale — connections grow as N*(N-1)/2.

**Transit Gateway** is a central hub that all VPCs connect to. Scales better and supports transitive routing. Use this for more than 3 VPCs.

## NAT Gateways

Resources in **private subnets** reach the internet via NAT gateways in public subnets. NAT gateways are charged per hour and per GB processed — they're often one of the larger line items on the AWS bill.

Cost optimization: use a single NAT gateway per AZ in dev, and one per AZ in production for HA.

## VPN and Direct Connect

For hybrid connectivity:
- **Site-to-Site VPN**: encrypted tunnel over the internet. Quick to set up, variable performance.
- **Direct Connect**: dedicated physical connection. Consistent performance, higher cost, 2-4 week setup.
