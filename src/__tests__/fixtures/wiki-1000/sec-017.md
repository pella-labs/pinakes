# TLS Configuration

Properly configuring **TLS** (Transport Layer Security) is essential for protecting data in transit. Weak configurations expose traffic to interception and downgrade attacks.

## Minimum Standards

- **TLS 1.2** as the minimum protocol version
- **TLS 1.3** preferred where client support allows
- Disable SSLv3, TLS 1.0, TLS 1.1

## Cipher Suite Selection

For TLS 1.3, only these cipher suites exist:

- `TLS_AES_256_GCM_SHA384`
- `TLS_AES_128_GCM_SHA256`
- `TLS_CHACHA20_POLY1305_SHA256`

For TLS 1.2, prefer **ECDHE** key exchange with **AES-GCM** or **ChaCha20**:

```nginx
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384;
ssl_prefer_server_ciphers on;
```

## HSTS

Always set **HTTP Strict Transport Security**:

```
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
```

This tells browsers to only use HTTPS for your domain. The `preload` directive submits your domain to browser preload lists.

## Certificate Management

Use **Let's Encrypt** or equivalent ACME provider for automated certificate issuance and renewal. Set up monitoring to alert well before expiry. See [[sec-006]] for mutual TLS and [[sec-010]] for certificate pinning.
