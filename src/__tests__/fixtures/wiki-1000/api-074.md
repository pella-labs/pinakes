# API Quota Management

## Quota vs Rate Limit

**Rate limit**: requests per time window (e.g., 100/minute). Protects against burst.
**Quota**: total usage per billing period (e.g., 10,000 API calls/month). Controls cost.

## Quota Tracking

```typescript
interface QuotaRecord {
  tenantId: string;
  period: string;       // "2025-11"
  used: number;
  limit: number;
  plan: string;
}

async function checkQuota(tenantId: string): Promise<{ allowed: boolean; remaining: number }> {
  const record = await redis.get(`quota:${tenantId}:${currentPeriod()}`);
  if (!record) return { allowed: true, remaining: Infinity }; // first request
  
  const quota = JSON.parse(record) as QuotaRecord;
  return {
    allowed: quota.used < quota.limit,
    remaining: Math.max(0, quota.limit - quota.used),
  };
}
```

## Response Headers

```
X-Quota-Limit: 10000
X-Quota-Remaining: 7234
X-Quota-Reset: 2025-12-01T00:00:00Z
```

## Overage Handling

Options when quota is exceeded:
1. Hard block — return 429
2. Soft limit — allow but flag for billing
3. Throttle — reduce rate limit, degrade quality
