# Terraform Data Sources

## Reading Existing Resources

**Data sources** let you reference infrastructure that exists outside your Terraform configuration:

```hcl
data "aws_vpc" "main" {
  filter {
    name   = "tag:Name"
    values = ["production-vpc"]
  }
}

data "aws_subnets" "private" {
  filter {
    name   = "vpc-id"
    values = [data.aws_vpc.main.id]
  }
  filter {
    name   = "tag:Tier"
    values = ["private"]
  }
}

resource "aws_instance" "app" {
  subnet_id = data.aws_subnets.private.ids[0]
  vpc_security_group_ids = [data.aws_security_group.app.id]
}
```

## Common Data Sources

- `aws_caller_identity` — current AWS account ID
- `aws_region` — current region
- `aws_ami` — find the latest AMI matching criteria
- `aws_eks_cluster` — read EKS cluster details
- `aws_secretsmanager_secret_version` — read a secret value

## Best Practices

Use data sources to reference shared infrastructure managed by other teams or Terraform configurations. This avoids duplication and keeps configurations loosely coupled.
