# Container Registry Management

## Private Registry

We run a private **ECR** (Elastic Container Registry) for all production images. Each service has its own repository with lifecycle policies.

## Image Tagging Strategy

- `latest` — never use in production, only for local dev
- `sha-abc1234` — git commit SHA, used in CI/CD
- `v1.2.3` — semantic version for releases
- `main-20250115` — branch + date for nightly builds

## Lifecycle Policies

Configure ECR lifecycle policies to automatically clean up old images:

```json
{
  "rules": [
    {
      "rulePriority": 1,
      "description": "Keep last 20 tagged images",
      "selection": {
        "tagStatus": "tagged",
        "tagPrefixList": ["v"],
        "countType": "imageCountMoreThan",
        "countNumber": 20
      },
      "action": { "type": "expire" }
    },
    {
      "rulePriority": 2,
      "description": "Remove untagged images after 7 days",
      "selection": {
        "tagStatus": "untagged",
        "countType": "sinceImagePushed",
        "countUnit": "days",
        "countNumber": 7
      },
      "action": { "type": "expire" }
    }
  ]
}
```

## Image Scanning

Enable **ECR image scanning** on push. Block deploys if critical vulnerabilities are found. Integrate with [[ops-003]] GitHub Actions for automated scanning.
