# Streaming HTML with React 18

## The Old Way

Traditional SSR renders the entire page to a string, then sends it:

```typescript
// Slow: blocks until everything is ready
const html = renderToString(<App />);
res.send(html);
```

## The Streaming Way

React 18's `renderToPipeableStream` sends HTML in chunks:

```typescript
import { renderToPipeableStream } from 'react-dom/server';

app.get('*', (req, res) => {
  const { pipe, abort } = renderToPipeableStream(<App url={req.url} />, {
    bootstrapScripts: ['/client.js'],
    onShellReady() {
      res.statusCode = 200;
      res.setHeader('Content-Type', 'text/html');
      pipe(res);
    },
    onShellError(error) {
      res.statusCode = 500;
      res.send('<h1>Server Error</h1>');
    },
    onError(error) {
      console.error(error);
    },
  });

  setTimeout(() => abort(), 10000);
});
```

## Shell vs Content

The **shell** is the layout that renders synchronously (navigation, footer, sidebar). Once the shell is ready, it streams to the browser. **Content** behind Suspense boundaries streams later as data resolves.

## Benefits

- Faster Time to First Byte
- Progressive rendering—users see content sooner
- Server doesn't hold the full response in memory
- Selective hydration of streamed sections
