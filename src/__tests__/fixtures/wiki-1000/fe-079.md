# Signals: A New Reactivity Primitive

## What Are Signals

**Signals** are a fine-grained reactivity primitive that updates only the specific DOM nodes that depend on a changed value. Unlike React's component-level re-rendering, signals operate at the value level.

## Implementations

- **Preact Signals**: `@preact/signals-react` for React compatibility
- **SolidJS**: signals are the core reactivity model
- **Angular Signals**: introduced in Angular 16
- **Vue refs**: functionally equivalent to signals

## How They Differ from React State

In React, changing state re-renders the entire component. With signals, only the specific DOM text node or attribute that reads the signal updates:

```typescript
// Preact Signals
import { signal, computed } from '@preact/signals-react';

const count = signal(0);
const doubled = computed(() => count.value * 2);

function Counter() {
  return (
    <div>
      <p>Count: {count}</p>
      <p>Doubled: {doubled}</p>
      <button onClick={() => count.value++}>+</button>
    </div>
  );
}
```

Clicking the button updates only the text nodes showing count and doubled, not the entire component tree.

## Tradeoffs

Signals break React's mental model of pure render functions. The React team has chosen the compiler approach ([[fe-077]]) over signals, arguing that automatic memoization achieves similar performance without changing the programming model.

See [[fe-012]] for Vue's reactivity and [[fe-013]] for Svelte's compile-time approach.
