# Buildkit and Advanced Docker Builds

## BuildKit Features

**BuildKit** is the modern Docker build engine with significant improvements over the legacy builder:

- Parallel build stages
- Better caching
- Secret mounting (don't embed secrets in layers)
- SSH forwarding for private repo access

## Secret Mounting

```dockerfile
# syntax=docker/dockerfile:1
FROM node:24-alpine
RUN --mount=type=secret,id=npmrc,target=/root/.npmrc \
    npm install
```

Build with:
```bash
docker build --secret id=npmrc,src=$HOME/.npmrc .
```

The secret is available during the build but never stored in a layer.

## Cache Mounts

Speed up package installation by caching the package manager's cache directory:

```dockerfile
RUN --mount=type=cache,target=/root/.npm \
    npm ci --prefer-offline
```

## Multi-Platform Builds

Build images for multiple architectures:

```bash
docker buildx create --use
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --tag myapp:latest \
  --push .
```

## CI Integration

Use BuildKit's cache export/import for CI caching:

```bash
docker buildx build \
  --cache-from type=gha \
  --cache-to type=gha,mode=max \
  --tag myapp:$SHA \
  --push .
```
