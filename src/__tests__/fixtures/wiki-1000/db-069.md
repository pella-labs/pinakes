# Handling NULL in SQL

## NULL Is Not a Value

**NULL** represents the absence of a value. It's not zero, not an empty string, not false. Any comparison with NULL returns NULL (not true or false). This catches everyone at some point.

```sql
-- These are all FALSE (or rather, NULL):
SELECT NULL = NULL;      -- NULL
SELECT NULL != NULL;     -- NULL
SELECT NULL > 0;         -- NULL

-- Use IS NULL / IS NOT NULL instead:
SELECT * FROM users WHERE deleted_at IS NULL;
```

## NULL in Aggregates

Most aggregate functions ignore NULLs. `COUNT(*)` counts all rows, but `COUNT(column)` only counts non-NULL values. `AVG` ignores NULLs, which may or may not be what you want.

## COALESCE

Replace NULL with a default:

```sql
SELECT COALESCE(nickname, name, 'Anonymous') AS display_name
FROM users;
```

## NULLIF

Returns NULL if two values are equal. Useful to avoid division by zero:

```sql
SELECT total / NULLIF(count, 0) AS average FROM stats;
```

## NULL and Indexes

In PostgreSQL, NULL values are included in B-tree indexes. Partial indexes can exclude them:

```sql
CREATE INDEX idx_active ON users (email) WHERE deleted_at IS NULL;
```

This produces a smaller, faster index for active user lookups.
