# GraphQL Persisted Queries

## Why

Reduce bandwidth by sending a query hash instead of the full query text. Also prevents arbitrary query execution (allowlist security).

## Automatic Persisted Queries (APQ)

1. Client sends hash of query
2. Server checks cache, misses
3. Server returns `PERSISTED_QUERY_NOT_FOUND`
4. Client resends with full query text + hash
5. Server caches the mapping
6. Subsequent requests use just the hash

```json
{
  "extensions": {
    "persistedQuery": {
      "version": 1,
      "sha256Hash": "abc123..."
    }
  },
  "variables": { "id": "ord_123" }
}
```

## Static Persisted Queries

For maximum security, generate the query map at build time:

```json
{
  "abc123...": "query GetOrder($id: ID!) { order(id: $id) { id status } }",
  "def456...": "mutation CancelOrder($id: ID!) { cancelOrder(input: {orderId: $id}) { ... } }"
}
```

Deploy the map with the server. Reject any query not in the map. This eliminates query injection entirely.
