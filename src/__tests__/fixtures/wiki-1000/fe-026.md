# Internationalization (i18n) in React

## Library Choices

The two dominant i18n libraries for React are **react-intl** (FormatJS) and **next-intl** (for Next.js). Both support message formatting, pluralization, date/number formatting, and ICU message syntax.

## Message Extraction

Define messages as structured objects and extract them for translation:

```typescript
// messages/en.json
{
  "greeting": "Hello, {name}!",
  "items": "{count, plural, =0 {No items} one {# item} other {# items}}",
  "lastUpdated": "Last updated {date, date, medium}"
}
```

## Implementation with next-intl

```tsx
import { useTranslations } from 'next-intl';

function Header() {
  const t = useTranslations();

  return (
    <header>
      <h1>{t('greeting', { name: 'Sebastian' })}</h1>
      <p>{t('items', { count: 42 })}</p>
    </header>
  );
}
```

## RTL Support

Right-to-left languages require CSS logical properties instead of physical ones:

- `margin-inline-start` instead of `margin-left`
- `padding-inline-end` instead of `padding-right`
- `inset-inline-start` instead of `left`

## Best Practices

- Never concatenate translated strings
- Always use ICU message format for plurals and gender
- Extract messages automatically—don't rely on manual processes
- Test with long translations (German) and RTL (Arabic) early

See [[fe-005]] for accessibility considerations that overlap with i18n.
