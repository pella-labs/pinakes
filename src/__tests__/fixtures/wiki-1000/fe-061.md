# React Concurrent Rendering

## What Changed

**Concurrent rendering** allows React to prepare multiple versions of the UI simultaneously. It can start rendering an update, pause to handle a more urgent one, and resume later.

## Key APIs

- `startTransition`: marks a state update as non-urgent
- `useDeferredValue`: returns a deferred version of a value
- `Suspense`: declarative loading states
- `use`: reads promises and context conditionally

## Practical Example

A search interface that doesn't block typing:

```tsx
function SearchPage() {
  const [query, setQuery] = useState('');
  const deferredQuery = useDeferredValue(query);

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Search..."
      />
      <Suspense fallback={<ResultsSkeleton />}>
        <SearchResults query={deferredQuery} />
      </Suspense>
    </div>
  );
}
```

The input remains responsive because `deferredQuery` updates at a lower priority than the input value.

## Automatic Batching

React 18+ batches all state updates by default, even those inside promises, timeouts, and native event handlers. This reduces unnecessary re-renders.
