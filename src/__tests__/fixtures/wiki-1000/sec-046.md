# Container Security

Containers introduce specific security considerations around image supply chain, runtime isolation, and configuration.

## Image Security

- **Use minimal base images**: `distroless`, `alpine`, or `scratch` where possible
- **Pin image versions**: use SHA256 digests, not just tags
- **Scan for vulnerabilities**: use Trivy, Grype, or your registry's built-in scanner
- **Sign images**: use Cosign or Notary for image provenance

## Dockerfile Best Practices

```dockerfile
# Use specific version
FROM node:24-alpine@sha256:abc123...

# Create non-root user
RUN addgroup -S app && adduser -S app -G app

# Copy with appropriate ownership
COPY --chown=app:app . /app

# Drop to non-root
USER app

# Don't run as PID 1 (use tini or dumb-init)
ENTRYPOINT ["tini", "--"]
CMD ["node", "dist/server.js"]
```

## Runtime Security

- **Drop all capabilities**: `--cap-drop=ALL`, add back only what's needed
- **Read-only filesystem**: `--read-only` with tmpfs for writable paths
- **No new privileges**: `--security-opt=no-new-privileges`
- **Resource limits**: CPU and memory limits prevent DoS

## Kubernetes Security Context

```yaml
securityContext:
  runAsNonRoot: true
  runAsUser: 1000
  readOnlyRootFilesystem: true
  allowPrivilegeEscalation: false
  capabilities:
    drop: ["ALL"]
```
