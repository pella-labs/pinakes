# Organizing Playwright Tests at Scale

## Project Structure

```
e2e/
  fixtures/
    auth.fixture.ts        # Authenticated user state
    db.fixture.ts          # Test database seeding
  pages/
    login.page.ts          # Page object model
    dashboard.page.ts
  specs/
    auth/
      login.spec.ts
      logout.spec.ts
    dashboard/
      widgets.spec.ts
      filters.spec.ts
  playwright.config.ts
```

## Page Object Model

```typescript
export class DashboardPage {
  constructor(private page: Page) {}

  async goto() {
    await this.page.goto('/dashboard');
  }

  async selectDateRange(from: string, to: string) {
    await this.page.getByLabel('From').fill(from);
    await this.page.getByLabel('To').fill(to);
    await this.page.getByRole('button', { name: 'Apply' }).click();
  }

  async getWidgetValue(name: string) {
    return this.page
      .getByRole('region', { name })
      .getByTestId('value')
      .textContent();
  }
}
```

## Authentication State

Reuse authentication state across tests:

```typescript
// global-setup.ts
const page = await browser.newPage();
await page.goto('/login');
await page.fill('[name=email]', 'test@example.com');
await page.fill('[name=password]', 'password');
await page.click('button[type=submit]');
await page.context().storageState({ path: 'auth.json' });
```

## Parallel Execution

Playwright runs test files in parallel by default. Use `test.describe.serial` only when tests have true ordering dependencies.

See [[fe-017]] for Playwright component testing and [[fe-068]] for Cypress comparison.
