# State Management in Modern React

## The Landscape

Choosing a state management solution depends on the complexity of your application. For most projects, the built-in **useState** and **useContext** hooks are sufficient. As applications grow, dedicated libraries provide better ergonomics and performance.

## Redux Toolkit

**Redux Toolkit** (RTK) is the official way to write Redux logic. It dramatically reduces boilerplate compared to classic Redux.

```typescript
import { createSlice, PayloadAction } from '@reduxjs/toolkit';

interface CounterState {
  value: number;
}

const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 } as CounterState,
  reducers: {
    increment: (state) => { state.value += 1; },
    decrement: (state) => { state.value -= 1; },
    incrementByAmount: (state, action: PayloadAction<number>) => {
      state.value += action.payload;
    },
  },
});
```

## Zustand

**Zustand** provides a minimal API with no boilerplate. It uses a hook-based approach and doesn't require providers. See [[fe-008]] for a deeper comparison with Jotai.

- No provider wrapper needed
- Built-in middleware for persistence, devtools, and immer
- Tiny bundle size (~1KB)
- Works outside React components

## When to Choose What

- **useState/useReducer**: Local component state
- **Context + useReducer**: Shared state across a small subtree
- **Zustand**: Medium apps, need simplicity
- **Redux Toolkit**: Large apps, need middleware ecosystem, devtools
- **Jotai/Recoil**: Atomic state, fine-grained reactivity
