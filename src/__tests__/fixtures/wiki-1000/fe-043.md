# Refs and Imperative Handles in React

## When to Use Refs

**Refs** provide a way to access DOM nodes or persist mutable values without triggering re-renders. Use them for:

- Managing focus, text selection, or media playback
- Integrating with third-party DOM libraries
- Storing previous values or timers

## forwardRef and useImperativeHandle

Expose specific methods from a child component to its parent:

```tsx
interface InputHandle {
  focus: () => void;
  clear: () => void;
}

const FancyInput = forwardRef<InputHandle, InputProps>((props, ref) => {
  const inputRef = useRef<HTMLInputElement>(null);

  useImperativeHandle(ref, () => ({
    focus: () => inputRef.current?.focus(),
    clear: () => {
      if (inputRef.current) inputRef.current.value = '';
    },
  }));

  return <input ref={inputRef} {...props} />;
});
```

## Callback Refs

For measuring DOM elements or reacting to mount/unmount:

```tsx
function MeasuredBox() {
  const [height, setHeight] = useState(0);

  const measuredRef = useCallback((node: HTMLDivElement | null) => {
    if (node) setHeight(node.getBoundingClientRect().height);
  }, []);

  return <div ref={measuredRef}>Content</div>;
}
```

## React 19 ref as prop

React 19 allows passing `ref` as a regular prop, eliminating the need for `forwardRef` in many cases. This simplifies component APIs considerably.
