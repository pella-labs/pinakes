# API SDK Design Principles

## Auto-Generation

Generate SDKs from the OpenAPI spec ([[api-006]]). Hand-written SDKs drift. Use:
- **openapi-generator** for broad language coverage
- **Stainless** for high-quality TypeScript SDKs
- **smithy** for AWS-style multi-language generation

## Ergonomic Wrappers

Generated code needs polish:

```typescript
// Generated (ugly)
const response = await client.orders.listOrders({ first: 10, after: cursor });

// Wrapper (nice)
const orders = await client.orders.list({ limit: 10, cursor });
for await (const order of client.orders.iterate()) { ... }
```

## Error Handling

SDKs should throw typed errors:

```typescript
try {
  await client.orders.create({ customerId: 'cust_1', items: [] });
} catch (err) {
  if (err instanceof ValidationError) {
    console.log(err.fields); // typed!
  } else if (err instanceof RateLimitError) {
    console.log(err.retryAfter);
  }
}
```

## Built-in Retries

SDKs should retry on 429 and 5xx with exponential backoff by default. The user shouldn't need to implement retry logic. See [[api-024]].

## Pagination Helpers

Provide auto-pagination iterators so users don't deal with cursors manually.
