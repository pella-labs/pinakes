# Kubernetes Pod Scheduling

## Node Affinity

**Node affinity** controls which nodes a pod can be scheduled on. Use it to target specific hardware (GPU nodes, high-memory nodes) or availability zones.

```yaml
affinity:
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
        - matchExpressions:
            - key: node.kubernetes.io/instance-type
              operator: In
              values: ["m5.2xlarge", "m6i.2xlarge"]
```

## Pod Anti-Affinity

Spread replicas across nodes or zones to improve fault tolerance:

```yaml
affinity:
  podAntiAffinity:
    preferredDuringSchedulingIgnoredDuringExecution:
      - weight: 100
        podAffinityTerm:
          labelSelector:
            matchLabels:
              app: api-server
          topologyKey: kubernetes.io/hostname
```

## Taints and Tolerations

**Taints** repel pods from nodes. **Tolerations** allow specific pods to schedule on tainted nodes. Use this for dedicated node pools (GPU, spot, system).

## Topology Spread Constraints

The modern way to distribute pods evenly across failure domains. Prefer this over pod anti-affinity for zone spreading.

See [[ops-004]] for K8s fundamentals and [[ops-024]] for spot instances.
