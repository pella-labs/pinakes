# CDN Configuration

## CloudFront Setup

Our **CDN** layer uses CloudFront to cache and serve static assets globally. This reduces latency for users worldwide and offloads traffic from origin servers.

## Cache Behaviors

Define cache behaviors per path pattern:

- `/static/*` — aggressive caching, 1 year TTL, immutable
- `/api/*` — no caching, forward all headers
- `/*.html` — short TTL (5 min), forward cookies for personalization

## Cache Invalidation

When deploying new assets, use **content hashing** in filenames (e.g., `app.a1b2c3.js`) instead of invalidation. This ensures instant cache busting without waiting for invalidation propagation.

For HTML files that can't be hashed, create CloudFront invalidations:

```bash
aws cloudfront create-invalidation \
  --distribution-id E1234ABCDEF \
  --paths "/index.html" "/about.html"
```

## Origin Shield

Enable **Origin Shield** to add an additional caching layer between edge locations and your origin. This dramatically reduces origin load for popular content.

## Security Headers

Add security headers via CloudFront Functions:

- `Strict-Transport-Security`
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Content-Security-Policy`
