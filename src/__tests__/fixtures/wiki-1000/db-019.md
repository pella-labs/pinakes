# PostgreSQL JSON and JSONB

## When to Use JSONB

PostgreSQL's **JSONB** type stores binary JSON with full indexing support. Use it when:

- Schema varies per row (user preferences, feature flags)
- You're migrating from a document database
- The data is read-heavy and queries target specific keys

Do NOT use JSONB as a replacement for proper relational modeling. If you're always querying the same fields, they should be columns.

## Querying JSONB

```sql
-- Access nested values
SELECT data->>'name' AS name
FROM products
WHERE data->'metadata'->>'category' = 'electronics';

-- Check if key exists
SELECT * FROM products WHERE data ? 'discontinued';

-- Containment operator (uses GIN index)
SELECT * FROM products
WHERE data @> '{"metadata": {"category": "electronics"}}';
```

## Indexing JSONB

A GIN index on the whole column supports containment and existence queries:

```sql
CREATE INDEX idx_products_data ON products USING GIN (data);
```

For specific keys, an expression index is more targeted:

```sql
CREATE INDEX idx_products_category
ON products ((data->'metadata'->>'category'));
```

## JSONB vs JSON

`json` stores the raw text and preserves formatting. `jsonb` parses it into binary format — slower to insert, faster to query. Always use `jsonb` unless you need to preserve the exact JSON text.
