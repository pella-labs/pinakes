# GitHub Actions Matrix Strategy

## Matrix Builds

Run the same job across multiple configurations using a **matrix strategy**:

```yaml
jobs:
  test:
    strategy:
      matrix:
        node-version: [18, 20, 22]
        os: [ubuntu-latest, macos-latest]
        exclude:
          - node-version: 18
            os: macos-latest
        include:
          - node-version: 22
            os: ubuntu-latest
            coverage: true
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
      - run: npm test
      - if: matrix.coverage
        run: npm run test:coverage
```

## Fail-Fast

By default, if one matrix job fails, all others are cancelled. Disable with `fail-fast: false` when you want to see all failures:

```yaml
strategy:
  fail-fast: false
  matrix:
    node-version: [18, 20, 22]
```

## Dynamic Matrix

Generate the matrix from a previous step:

```yaml
jobs:
  determine-matrix:
    outputs:
      matrix: ${{ steps.set-matrix.outputs.matrix }}
    steps:
      - id: set-matrix
        run: echo "matrix=$(cat matrix.json)" >> $GITHUB_OUTPUT
  test:
    needs: determine-matrix
    strategy:
      matrix: ${{ fromJson(needs.determine-matrix.outputs.matrix) }}
```
