# Secure API Gateway Design

An **API gateway** centralizes cross-cutting security concerns, providing a consistent security layer in front of your services.

## Core Security Functions

### Authentication

Validate credentials before requests reach backend services:

```yaml
# Kong gateway config example
plugins:
  - name: jwt
    config:
      claims_to_verify:
        - exp
      key_claim_name: kid
```

### Rate Limiting

Apply rate limits at the gateway level (see [[sec-026]]):

- Global rate limits protect infrastructure
- Per-client rate limits prevent individual abuse
- Per-endpoint rate limits protect expensive operations

### Request Validation

Validate request schemas before forwarding:

- Body schema validation against OpenAPI spec
- Header validation (required headers, format)
- Query parameter validation
- Request size limits

### Response Filtering

Strip internal information from responses:

- Remove internal headers (X-Internal-*, debug headers)
- Enforce response schemas
- Redact sensitive fields for certain client tiers

## Gateway Security

The gateway itself is a high-value target:

- Keep gateway software updated
- Minimize gateway plugins (each is attack surface)
- Monitor gateway logs for anomalies
- Implement gateway-level WAF rules (see [[sec-029]])

## Avoid Gateway Bypass

Ensure backend services reject direct connections:

- Backend services only accept traffic from the gateway IP range
- Backend services validate gateway-injected authentication headers
- Network policies enforce gateway-only access (see [[sec-091]])

See [[sec-044]] for microservices security architecture.
