# Sequences and Identity Columns

## SERIAL (Legacy)

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,  -- creates a sequence automatically
  name TEXT NOT NULL
);
```

`SERIAL` is syntactic sugar for creating a sequence and setting a default. It works but has ownership quirks.

## IDENTITY (Modern)

SQL standard identity columns (PostgreSQL 10+):

```sql
CREATE TABLE users (
  id INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL
);
```

`GENERATED ALWAYS` prevents manual ID assignment. Use `GENERATED BY DEFAULT` if you need to override occasionally.

## Sequence Gaps

Sequences **never** guarantee gap-free numbering. Rolled-back transactions consume sequence values. If you need gap-free numbering (invoices, for legal reasons), use a separate counter in a serializable transaction.

## Alternatives to Sequences

- **UUIDv4**: random, globally unique, but causes B-tree fragmentation
- **UUIDv7**: time-ordered, globally unique, better index performance
- **ULID**: similar to UUIDv7, lexicographically sortable
- **Snowflake IDs**: 64-bit, time-ordered, used by Twitter/Discord

```sql
-- UUIDv7 with pgcrypto
CREATE EXTENSION IF NOT EXISTS pgcrypto;
-- Use application-generated UUIDv7 for best performance
```

For how ID choice affects sharding, see [[db-021]].
