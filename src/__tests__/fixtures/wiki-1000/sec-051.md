# SCIM Provisioning Security

**SCIM** (System for Cross-domain Identity Management) automates user provisioning and deprovisioning between identity providers and service providers. Securing the SCIM endpoint is critical because it controls user access.

## Authentication

The SCIM endpoint must use strong authentication:

- **Bearer tokens** with short expiration
- **OAuth2 client credentials** (see [[sec-002]])
- **mTLS** for highest assurance (see [[sec-006]])

Never use basic authentication for SCIM endpoints.

## Authorization

Restrict SCIM operations by client:

- Some IdPs should only provision users, not modify roles
- Deprovisioning should be audited separately
- Group membership changes should require elevated permissions

## Rate Limiting

SCIM endpoints are targets for enumeration attacks. Apply rate limits per client (see [[sec-026]]) and monitor for unusual patterns like bulk user creation.

## Data Validation

Validate all SCIM payloads against the schema. Reject unexpected attributes. Sanitize display names and email addresses before storage.
