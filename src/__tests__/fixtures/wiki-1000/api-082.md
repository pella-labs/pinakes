# API Design for Mobile Clients

## Mobile-Specific Constraints

- **Bandwidth**: cellular connections are slow and metered
- **Latency**: higher RTT than desktop
- **Battery**: every network request costs energy
- **Connectivity**: intermittent, unpredictable

## Optimization Strategies

### Minimize Round Trips
Use **batch endpoints** ([[api-017]]) or **GraphQL** ([[api-002]]) to fetch everything in one request.

### Compress Responses
Brotli > gzip. See [[api-030]].

### Field Filtering
Only send fields the mobile UI needs. See [[api-057]].

### Delta Sync
Instead of full refreshes, send only changes since the last sync:

```
GET /orders?updated_since=2025-11-03T10:00:00Z
```

### Offline-First
Return `Cache-Control` headers that allow offline reading. Support idempotent retries for offline mutations.

## BFF Pattern

Build a dedicated **Backend for Frontend** for mobile that:
- Aggregates data from multiple services
- Shapes responses for mobile screen layouts
- Handles pagination with mobile-friendly defaults (smaller pages)
- Compresses images server-side

See [[api-025]] for the BFF architecture.
