# Multipart File Upload API

## Endpoint Design

```
POST /files
Content-Type: multipart/form-data

--boundary
Content-Disposition: form-data; name="file"; filename="report.pdf"
Content-Type: application/pdf

<binary data>
--boundary
Content-Disposition: form-data; name="metadata"
Content-Type: application/json

{"folder":"reports","tags":["q4","finance"]}
--boundary--
```

## Server Handling

Stream the upload — don't buffer the entire file in memory.

```typescript
import multer from 'multer';

const upload = multer({
  dest: '/tmp/uploads',
  limits: { fileSize: 50 * 1024 * 1024 }, // 50MB
  fileFilter: (req, file, cb) => {
    const allowed = ['application/pdf', 'image/png', 'image/jpeg'];
    cb(null, allowed.includes(file.mimetype));
  },
});

app.post('/files', upload.single('file'), async (req, res) => {
  const { path, originalname, size } = req.file!;
  const metadata = JSON.parse(req.body.metadata);
  const record = await fileService.store(path, originalname, metadata);
  res.status(201).json(record);
});
```

## Resumable Uploads

For large files, use the **tus protocol** or a chunked upload approach. The client uploads chunks and the server tracks progress. On failure, the client resumes from the last successful chunk.

Presigned URLs (S3) are another option — the server generates a URL, the client uploads directly to storage, then notifies the server.
