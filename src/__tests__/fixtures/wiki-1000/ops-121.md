# Kubernetes Finalizers

## What Are Finalizers

**Finalizers** are keys on Kubernetes resources that tell the API server to wait before completing deletion. Controllers watch for objects with their finalizer and perform cleanup before removing the finalizer.

## How Deletion Works

1. User deletes a resource (`kubectl delete`)
2. API server sets `deletionTimestamp` but doesn't remove the object
3. Controllers see the deletion timestamp and perform cleanup
4. Each controller removes its finalizer when cleanup is done
5. When all finalizers are removed, the API server deletes the object

## Common Finalizer Issues

A resource stuck in `Terminating` state usually has a finalizer that hasn't been removed. Check:

```bash
kubectl get namespace stuck-namespace -o json | jq '.spec.finalizers'
kubectl get pods -n stuck-namespace -o json | jq '.items[].metadata.finalizers'
```

## Force Removing Finalizers

As a last resort, remove finalizers manually:

```bash
kubectl patch namespace stuck-namespace \
  -p '{"metadata":{"finalizers":[]}}' \
  --type=merge
```

Only do this if you understand why the finalizer wasn't removed. Forcing removal may leave orphaned cloud resources.

## Custom Finalizers

Operators use custom finalizers to clean up external resources:

```go
const finalizerName = "example.com/cleanup"

// On create: add finalizer
// On delete: cleanup external resources, then remove finalizer
```

See [[ops-061]] for operators, [[ops-075]] for CRDs.
