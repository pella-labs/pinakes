# SQLite Extensions

## Loading Extensions

SQLite supports loadable extensions for functionality beyond the core:

```typescript
import Database from 'better-sqlite3';

const db = new Database('app.db');
db.loadExtension('/path/to/extension');
```

## Popular Extensions

### sqlite-vec
Vector similarity search for embeddings. Lightweight alternative to pgvector:

```sql
CREATE VIRTUAL TABLE chunks_vec USING vec0(
  embedding float[384]
);
```

### FTS5
Full-text search (built-in but technically an extension). See [[db-035]].

### json1
JSON functions (built-in since 3.38):

```sql
SELECT json_extract(data, '$.name') FROM config;
```

### spatialite
Geospatial queries for SQLite — the equivalent of PostGIS.

### sqlean
A collection of community extensions: crypto, regex, unicode, uuid, and more.

## Building Custom Extensions

SQLite extensions are shared libraries (.so/.dylib/.dll) implementing the `sqlite3_extension_init` entry point. You can write them in C, Rust, or Zig.

## Security

The `SQLITE_DBCONFIG_ENABLE_LOAD_EXTENSION` flag must be explicitly enabled. In production, only load trusted extensions and consider compile-time linking instead of runtime loading.
