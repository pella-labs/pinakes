# Pulumi for TypeScript Teams

## Overview

**Pulumi** lets you define infrastructure using real programming languages. For TypeScript teams, this means type safety, IDE autocomplete, and the ability to use familiar constructs like loops and conditionals.

## Example: S3 Bucket with Lambda

```typescript
import * as aws from "@pulumi/aws";
import * as pulumi from "@pulumi/pulumi";

const bucket = new aws.s3.Bucket("data-bucket", {
  versioning: { enabled: true },
  serverSideEncryptionConfiguration: {
    rule: {
      applyServerSideEncryptionByDefault: {
        sseAlgorithm: "aws:kms",
      },
    },
  },
});

const handler = new aws.lambda.Function("processor", {
  runtime: "nodejs20.x",
  handler: "index.handler",
  code: new pulumi.asset.AssetArchive({
    ".": new pulumi.asset.FileArchive("./dist"),
  }),
  environment: {
    variables: { BUCKET: bucket.id },
  },
});

export const bucketName = bucket.id;
```

## State Backend

Pulumi supports multiple state backends including Pulumi Cloud, S3, and local files. For teams already on AWS, an S3 backend works well.

## When to Use Pulumi vs Terraform

Use Pulumi when the team is already strong in TypeScript and the infrastructure has complex conditional logic. Use Terraform when you need the broadest provider ecosystem or when the team is more ops-focused. See [[ops-006]] and [[ops-007]].
