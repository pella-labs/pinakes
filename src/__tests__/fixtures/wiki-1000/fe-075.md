# CSS Container Queries

## Beyond Media Queries

**Container queries** let components respond to the size of their container, not the viewport. This makes components truly layout-independent and reusable.

## Setup

Declare a containment context on the parent:

```css
.card-container {
  container-type: inline-size;
  container-name: card;
}
```

## Querying the Container

```css
@container card (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
    gap: 1rem;
  }
}

@container card (max-width: 399px) {
  .card {
    display: flex;
    flex-direction: column;
  }
}
```

## Container Query Units

New units relative to the container:
- `cqw` — 1% of container width
- `cqh` — 1% of container height
- `cqi` — 1% of container inline size
- `cqb` — 1% of container block size

```css
.card-title {
  font-size: clamp(1rem, 4cqi, 2rem);
}
```

## Browser Support

Container queries are supported in all modern browsers since early 2023. For older browsers, use a media query fallback with `@supports`.

See [[fe-015]] for responsive design strategies and [[fe-029]] for grid layouts.
