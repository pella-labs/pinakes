# React Compiler (React Forget)

## What It Does

The **React Compiler** (previously called React Forget) analyzes your React components at build time and automatically inserts memoization where it helps. It understands React's rules of hooks and component purity requirements.

## Before and After

Without the compiler, you manually optimize:

```tsx
// Manual memoization
const filtered = useMemo(() => items.filter(predicate), [items, predicate]);
const handleClick = useCallback(() => select(item.id), [item.id]);
const MemoizedChild = React.memo(Child);
```

With the compiler, write straightforward code:

```tsx
// Compiler handles memoization automatically
const filtered = items.filter(predicate);
const handleClick = () => select(item.id);
// No React.memo needed
```

## How It Works

The compiler builds a dependency graph of values in each component. It identifies which values change between renders and wraps expensive computations in automatic memoization. The output is equivalent to expertly hand-optimized code.

## Adoption

The compiler ships as a Babel or SWC plugin. It is opt-in at the file or directory level. Components that violate React's rules (mutating props, side effects in render) will be skipped with a warning.

## Impact

In Meta's testing, the compiler improved re-render performance by 40-70% on complex pages without any manual optimization. It represents a shift from "developers must understand React's rendering model" to "write plain code and the compiler handles the rest."
