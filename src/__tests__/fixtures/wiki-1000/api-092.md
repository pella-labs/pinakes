# Designing Export APIs

## Small Exports (< 10MB)

Synchronous response:

```
GET /orders/export?format=csv&date_from=2025-01-01&date_to=2025-11-01
Accept: text/csv
```

Stream the CSV directly in the response body.

## Large Exports (> 10MB)

Async job pattern:

1. `POST /exports` → returns `{ job_id: "exp_123", status: "queued" }`
2. `GET /exports/exp_123` → returns `{ status: "processing", progress: 45 }`
3. `GET /exports/exp_123` → returns `{ status: "complete", download_url: "...", expires_at: "..." }`
4. `GET /exports/exp_123/download` → returns the file

## Supported Formats

- **CSV** — universal, easy to open in Excel
- **JSON** — machine-readable, preserves types
- **NDJSON** — streamable JSON
- **Parquet** — columnar, best for large datasets in data pipelines

## Considerations

- Exports can be expensive — rate limit them separately
- Include column headers in CSV
- Handle null values consistently
- Respect tenant isolation — never include data from other tenants
- Presign download URLs with short TTL (1 hour)
