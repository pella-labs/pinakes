# DuckDB

## The SQLite of Analytics

**DuckDB** is an in-process analytical database — like SQLite but optimized for OLAP workloads. It runs embedded in your application with no server process.

## Key Features

- Columnar storage engine
- Vectorized query execution
- Reads Parquet, CSV, JSON directly without loading
- Compatible with PostgreSQL SQL syntax
- Parallel query execution on all cores

```sql
-- Query a Parquet file directly
SELECT
  product_category,
  sum(revenue) AS total_revenue,
  count(*) AS num_orders
FROM read_parquet('s3://bucket/orders/*.parquet')
WHERE order_date >= '2024-01-01'
GROUP BY product_category
ORDER BY total_revenue DESC;
```

## Use Cases

- Local data analysis (replacement for pandas in many cases)
- ETL transformation steps
- Embedded analytics in applications
- CI/CD pipeline data validation

## Performance

DuckDB uses columnar compression and vectorized execution. Analytical queries on millions of rows complete in seconds on a laptop. It's not designed for OLTP — no concurrent writes, no replication.

For data warehousing at scale, see [[db-036]]. For embedded OLTP, SQLite remains the right choice (see [[db-028]]).
