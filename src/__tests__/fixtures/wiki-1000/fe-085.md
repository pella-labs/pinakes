# Common Hook Mistakes and Rules

## The Rules

1. Only call hooks at the top level — never inside conditions, loops, or nested functions
2. Only call hooks from React function components or custom hooks
3. Custom hooks must start with `use`

## Stale Closure

The most common hook bug. An effect captures a value at render time, but the value changes before the effect runs:

```tsx
// Bug: count is always 0 in the interval
useEffect(() => {
  const id = setInterval(() => console.log(count), 1000);
  return () => clearInterval(id);
}, []); // empty deps = captures initial count

// Fix: use a ref
const countRef = useRef(count);
countRef.current = count;
useEffect(() => {
  const id = setInterval(() => console.log(countRef.current), 1000);
  return () => clearInterval(id);
}, []);
```

## Infinite Loops

Creating new objects/arrays in render causes effects to re-run infinitely:

```tsx
// Bug: new array every render triggers effect
useEffect(() => {
  fetchItems(filters);
}, [filters]); // if filters = [a, b] created in render

// Fix: memoize or stringify
const stableFilters = useMemo(() => [a, b], [a, b]);
```

## Over-Synchronization

Don't use effects to transform state. Use derived state instead:

```tsx
// Bad: effect to sync state
useEffect(() => {
  setFullName(`${first} ${last}`);
}, [first, last]);

// Good: derive during render
const fullName = `${first} ${last}`;
```
