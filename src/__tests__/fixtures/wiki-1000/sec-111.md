# Command Injection Prevention

**Command injection** occurs when untrusted input is included in operating system commands. It allows attackers to execute arbitrary commands on the server.

## The Vulnerability

```typescript
// VULNERABLE — command injection
const output = execSync(`ping -c 4 ${userInput}`);
// If userInput = "google.com; rm -rf /"
```

## Prevention

### Avoid Shell Commands

Use language-native libraries instead of shelling out:

```typescript
// Instead of: exec('mkdir -p /path/to/dir')
import { mkdirSync } from 'fs';
mkdirSync('/path/to/dir', { recursive: true });
```

### Use Array Arguments

If you must execute external commands, use APIs that accept arguments as arrays:

```typescript
import { execFileSync } from 'child_process';

// SAFE — arguments are not interpreted by the shell
execFileSync('ping', ['-c', '4', userInput]);
```

### Input Validation

If arguments must come from user input, validate against a strict allowlist:

```typescript
const hostname = userInput.match(/^[a-zA-Z0-9.-]+$/)?.[0];
if (!hostname) throw new ValidationError('Invalid hostname');
```

## Environment Variables

Don't forget that environment variables can also be a command injection vector if used in shell commands:

```bash
# VULNERABLE if FILENAME is attacker-controlled
cat "$FILENAME"
```

See [[sec-036]] for general input validation and [[sec-058]] for SSRF, which shares similar root causes.
