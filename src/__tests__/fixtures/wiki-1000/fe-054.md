# React Rendering Optimization

## When React Re-Renders

A component re-renders when:
- Its state changes
- Its parent re-renders (even if props are the same)
- A context it consumes changes

Understanding these triggers is the first step to preventing unnecessary work.

## React.memo

Wrapping a component in `React.memo` skips re-rendering when props are shallowly equal:

```tsx
const ExpensiveList = React.memo(function ExpensiveList({ items }: Props) {
  return items.map((item) => <ListItem key={item.id} {...item} />);
});
```

## useMemo and useCallback

Memoize computed values and callback functions to maintain referential equality:

```tsx
const filteredItems = useMemo(
  () => items.filter((i) => i.category === selected),
  [items, selected]
);

const handleClick = useCallback(
  (id: string) => dispatch({ type: 'SELECT', id }),
  [dispatch]
);
```

## The React Compiler

The **React Compiler** (formerly React Forget) automatically inserts memoization. When adopted, manual `useMemo`, `useCallback`, and `React.memo` become unnecessary. It analyzes component code at build time and optimizes re-renders.

## Measure Before Optimizing

Use React DevTools Profiler to identify actual performance bottlenecks. Premature optimization adds complexity without measurable benefit. Most applications don't need manual memoization. See [[fe-009]] for performance metrics.
