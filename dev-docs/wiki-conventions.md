# Wiki Conventions

## Compiled-Truth + Timeline Page Structure

Every wiki page follows a two-zone pattern (inspired by the Karpathy LLM-Wiki spec and GBrain):

```markdown
# Topic Name

Always-current synthesized knowledge goes here. This section is
**rewritten** whenever evidence changes — it represents what we
know *right now*.

Key facts, decisions, conventions, and current state all live above
the thematic break.

---

## Timeline

### 2026-04-11 — Source: PR #42
Discovered that the auth module uses bcrypt with cost 12.

### 2026-04-10 — Source: architecture review
Initial decision to use JWT with RS256 signing.
```

### Rules

1. **Above `---`**: Compiled truth — always-current, synthesized, rewritten as evidence changes. If you read only this section, you know everything you need.

2. **Below `---`**: Timeline — append-only, reverse-chronological evidence. Never rewritten, only appended. Each entry has a date and source attribution.

3. **One topic per page.** If a page covers two unrelated concepts, split it.

4. **Inline citations.** Every fact in compiled truth should trace to a timeline entry.

5. **Wikilinks.** Use `[[page-name]]` to cross-reference other wiki pages. These are extracted during ingest and populate the knowledge graph edges.

## Page Types (by directory convention)

```
wiki/
  architecture/     # Architecture decisions, system design
  conventions/      # Code conventions, style guides
  concepts/         # Domain concepts, glossary terms
  guides/           # How-to guides, workflows
  log.md            # Append-only event log (use pinakes.project.append())
  contradictions.md # Auto-generated by contradiction-scan
```

## Writing via the MCP tools

```javascript
// Create a new wiki page
pinakes.project.write('concepts/auth.md', `# Authentication

Our system uses bcrypt for password hashing with cost factor 12.
Session tokens are JWT signed with RS256.

---

## Timeline

### 2026-04-11 — Source: initial setup
Documented the authentication architecture.
`);

// Append to the event log
pinakes.project.append('Reviewed auth module — bcrypt cost factor confirmed at 12');
```
